Data Modeling for Qlik Sense

Activity 1: Create new fields for Orders and OrderDetails

Summary:

  • Generate additional data values in the orders table for day, month and year of orders.
  • Create a calculated expression in the OrderDetails table for the exact sales amount of an order line.

Module: Generating data

Qlik Sense app: ABC Sales Data

Challenge

Transform your existing data model by creating new calculated fields:

  • In the Orders table, modify the script and create the following fields to allow for the analysis of trends over year, month and day:
    • Year, that contains the year part of the OrderDate field.
    • Month, that contains the month of the OrderDate field.
    • Day, that contains the day of the OrderDate field.
  • In the OrderDetails table, create the following field that allows for the analysis of the company's orders:
    • LineSalesAmount, a field that contains the total sales amount for every order line.
  • Load your data and check your data model to verify the changes.

Hints

  • The tables where the fields must be added are in the Orders section of the Data load editor.
  • The date functions Year(), Month(), and Day() must be used to create the fields.
  • The expression for the LineSalesAmount field is the following: UnitPrice * Quantity * (1 - Discount).

 

Step-by-step guide
  1. In the Orders section, within the Orders table LOAD, after the OrderDate field, enter a carriage return to make space for new script. Make sure there is a comma right after the previous field.
  2. Add the following three lines of loading script and make sure you use the correct syntax at the end:
    3. YEAR(OrderDate) AS Year, MONTH(OrderDate) AS Month, DAY(OrderDate) AS Day;
  3. In the OrderDetails table LOAD, after UnitPrice, enter a carriage return to make space for new script.
  4. Add the following line of script and make sure you use the correct syntax at the end:

    UnitPrice * Quantity * (1 - Discount) AS LineSalesAmount,

    5. Note: In the preceding line, the spaces on either side of the * and - operators are optional, but are included for readability.

    The Orders script section now looks as follows:

    LIB CONNECT TO 'ABC'; // **** Load Orders table **** [Orders]: LOAD OrderID, // OrderID AS OrderRecordCounter, 1 AS OrderRecordCounter, CustomerID, CustomerID AS ActiveCustomerID, EmployeeID, ShipperID, FreightWeight, OrderDate, YEAR(OrderDate) AS Year, MONTH(OrderDate) AS Month, DAY(OrderDate) AS Day; SELECT OrderID, CustomerID, EmployeeID, ShipperID, FreightWeight, OrderDate FROM "ABC_2025".dbo.orders; // **** Load OrderDetails table **** [OrderDetails]: LOAD OrderID, LineNo, ProductID, UnitPrice, UnitPrice * Quantity * (1 - Discount) AS LineSalesAmount, Quantity, Discount; SELECT OrderID, "LineNo", ProductID, UnitPrice, Quantity, Discount FROM "ABC_2025".dbo.orderdetails;
  5. Load your data after you apply the changes and navigate to the Data model viewer.
  6. Confirm that the new fields have been added. You might need to expand the tables in order to view all the fields.
