Data Modeling for Qlik Sense

Activity 2 - Challenge: Map ShipperName to Orders

Summary:

  • Re-engineer the Shippers dimension to act as a mapping table.
  • Use an ApplyMap() function in the Orders table to incorporate ShipperName into Orders.

Module: Combining tables

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, use a MAPPING LOAD to move the ShippersName field into the Orders table by completing the following tasks:

  • Move the Shippers LOAD statement from the Shipments script section to the MappingTables script section. Relabel the LOAD statement Shippers_Map.
  • Convert the table into a mapping table and remove the alias from the CompanyName field, as it is not needed.
  • The ShipperID is the key field and must always be first in the LOAD statement. Swap the order of the fields if you need to.
  • In the Orders table, use the ApplyMap statement to import the CompanyName field from the Shippers_Map table, based on matching ShipperID values.
  • In the ApplyMap statement, add the word MISSING as the default value in case there are no matching values with the Shippers_Map table and name the new field ShipperName.
  • Comment out the ShipperID field from the Orders table, as it is no longer needed in the data model.
  • Load your data and verify that the values loaded in the ShipperName field are correct.
  • Open the Calendar Flags app sheet and add a Filter pane with the ShipperName dimension. Verify that you see the value MISSING in the field.
Note: The objective of this challenge is to test whether you feel comfortable with the concepts that have been discussed so far in your class. There is no Step-by-Step section in this activity, so you must only rely on the Hints below to understand the tasks you have to execute.
As you are still new to many of these concepts, please don't hesitate to reach out for more guidance (help file, instructor, Qlik Community).

Hints

  • This process eliminates the need for a separate Shippers table and the ShipperID field. It reduces the number of tables in your data model by one and denormalizes your schema.
