Data Modeling for Qlik Sense

Activity 5 - Optional: Create the Link table

Summary:

  • Create concatenated keys within the Quotas and Orders tables, using the fields EmployeeID, Year, and Quarter.
  • Generate a link table between Orders and Quotas to maintain correct associations and provide a bridge to Employees.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In your ABC Sales Data app, create a Link table to resolve the issue in the line chart from the Sales vs. Quota sheet by doing the following:

  • Navigate to the Quota script section and rename the QuotaYear to Year and the QuotaQuarter to Quarter.
  • Load the data and observe the circular reference in your data model. Try to identify the fields and tables that cause the problem.
  • Rename the Quotas table to QuotasTemp, and rename the above fields back to their original values. Alias the field EmployeeID to QuotaEmployeeID.
  • Create the main Quotas table that contains all the fields from QuotasTemp and the composite field named EmpYrQtr_Key.
  • In the Orders section, locate the Orders table and alias EmployeeID as OrdersEmployeeID.
  • Use a preceding LOAD on preceding LOAD to create the composite key as above. Use the same fields and the same key name.
  • Remove the fields, Year and Quarter, from the MasterCalendar table.
  • Create a new script section after Quota and name it LinkTable. Create the script for a Link table according to the steps in the theory section.
  • At the end of the LinkTable script section, drop the fields QuotaYear, QuotaQuarter and QuotaEmployeeID, as you do not need them any more.

Hints

  • A Link table is necessary in order for the Sales vs. Quota sheet to produce correct results.
  • A synthetic key is caused when associating the QuotaYear and QuotaQuarter fields with the Year and Quarter. This synthetic table is resolved with the Link table.

 

Step-by-step guide
  1. A change in the data model is required at this stage. In the Data load editor, locate the Quota section.
  2. The QuotaYear and the QuotaQuarter need to be renamed to make sure the Quotas table associates to the data model correctly. Therefore, replace the QuotaYear and the QuotaQuarter fields in the Quotas table with Year and Quarter, respectively.
  3. Load the data and notice that a circular reference warning appears during the script execution.
  4. Analyze the circular reference in the data model.
    Note: A synthetic table, $Syn 1 Table has been created between MasterCalendar and Quotas, as they have now two fields with the same name, Year and Quarter.
    There is also a circular reference between the Quotas, MasterCalendar, Orders and Employees tables. The Orders table has been disconnected from the data model due to this loop and is now a loosely coupled table.
    In this case, you cannot rename fields or delete them, as they are all required as part of a compound key. A Link table must be created between the Quotas, MasterCalendar, Orders, and Employees tables, with all the common fields and keys to associate them.
  5. In the Data load editor, locate the Quota script section and change the Quotas table name to QuotasTemp.
  6. In the QuotasTemp table, rename the Year and Quarter fields back to QuotaYear and QuotaQuarter. Also, alias EmployeeID as QuotaEmployeeID. This breaks the association between this table and the rest of the data model.
  7. After the QuotasTemp table LOAD segment, add the following script code:
    Quotas: LOAD *, QuotaEmployeeID & '|' & QuotaYear & '|' & QuotaQuarter AS EmpYrQtr_Key RESIDENT QuotasTemp; DROP TABLE QuotasTemp;

    This LOAD...RESIDENT statement creates the key field in the Quotas table.
  8. In the Orders script section, alias the EmployeeID field to OrdersEmployeeID, in the Orders table.
  9. Create the EmpYrQtr_Key field in the Orders table using the following preceding LOAD statement:
    LOAD *, OrdersEmployeeID & '|' & YEAR(OrderDate) & '|' & 'Q' & CEIL(MONTH(OrderDate) / 3) AS EmpYrQtr_Key;
  10. In the MasterCalendar section, comment out the Year and Quarter fields from the MasterCalendar table.
    The fields are not really needed in the MasterCalendar table, as they will be part of the Link table.
  11. Create a new script section named LinkTable, after the Quota section, and add the following script:
    LinkTable: LOAD DISTINCT // **************** From Quotas table EmpYrQtr_Key, QuotaEmployeeID AS EmployeeID, QuotaYear AS Year, QuotaQuarter AS Quarter, 'Quotas' AS FromTable RESIDENT Quotas; CONCATENATE (LinkTable) LOAD DISTINCT // **************** From Orders table EmpYrQtr_Key, OrdersEmployeeID AS EmployeeID, YEAR(OrderDate) AS Year, 'Q' & CEIL(MONTH(OrderDate) / 3) AS Quarter, 'Orders' AS FromTable RESIDENT Orders;

    Note that a Link table is created using a LOAD... RESIDENT statement from the tables with common dimensions. LOAD DISTINCT is used to avoid repetition in the data and the CONCATENATE prefix is used to unify the tables with all the possible combinations of values for the dimensions. There is a new field called FromTable that helps to identify the tables from which each record is located. This field is optional.
  12. Delete unneeded fields by adding this statement at the end of the LinkTable section:
    DROP FIELDS QuotaYear, QuotaQuarter, QuotaEmployeeID, OrdersEmployeeID;
  13. Load the data and verify that your data model resembles the Solution image.
  14. Return to the Sales vs. Quota app sheet and review the line chart. Even though the chart looks better, there is still something you can do to improve it. The chart lines currently show zeros for the data points where data does not exist. In the sheet edit mode, edit the line chart Properties; Replace the Quota measure expression with the following:
    IF(SUM(QuotaAmount)=0, Null(), SUM(QuotaAmount))
  15. Change the line chart style to Area (Appearance > Presentation).
  16. The results should now be correct. Also, notice how the filters display an invalid dimension error message, as these fields were deleted. You can safely remove them from your sheet.
