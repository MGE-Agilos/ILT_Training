Data Modeling for Qlik Sense

Activity 2: Add an Include statement

Summary:

  • Use an Include statement to leverage pre-created, external data load script.
  • Discover a way to amalgamate the resulting table into the existing data model, without altering the external script file.

Module: Developing and Debugging

Qlik Sense app: ABC Sales Data

Challenge

Use an Include statement to load part of the script from an external file.

  • The external file that contains the reusable script is email.txt
  • Create a new script section named Email, following the SalesPerson section.
  • Add an Include statement to create a new table, EmployeeMail, using data from the file noted above.
  • RESIDENT LOAD from the EmployeeMail table and create a JOIN to Employees.
  • Drop the original EmployeeMail table.
  • Check the Data model viewer.

Hints

  • The Include statement is always used inside a dollar-sign expansion syntax.
  • The reason using Include statements is so common is because it allows code reusability, easier maintenance, and version control options.
  • The Email field needs to be folded into the Employees table without altering the original .txt file. Therefore, a RESIDENT LOAD and JOIN are used, and the original table is dropped.

 

Step-by-step guide
  1. Use a text editor like Notepad, to examine the file email.txt. The code in the file contains a LOAD...FROM syntax, geared to using the employees.xlsx file, and creates a table called EmployeeMail.
    The email is generated automatically using the first letter of the first name, and the first and last letters of the last name. It also ensures that there are no capital letters in the signature, only lowercase letters. Foreign (Swedish) letters, such as å, Å, ä, Ä, ö, and Ö, are substituted by ASCII characters. All of these are handled using nestled IF statements.
    Following the signature, @ is appended, followed by the server address. The latter is determined by the employee office.
  2. In the Data load editor, create a new section between SalesPerson and MasterCalendar, named Email.
  3. Using the Select data option on the DataFiles connection, upload the email.txt file from the student collateral, and cancel the Select file dialog when upload is complete.
    4. Qlik Sense on Windows: The email.txt file is already available from the TextFiles connection.
  4. Add an Include variable to load the script as an external file:
    // **** Load emails using include variable **** $(Include=[$(vPathFiles)email.txt]);
    Qlik Sense on Windows: The vPathFiles variable does not exist. Use the vPathText instead, which points to the TextFiles (qse_student) data connection, where the email.txt file resides:
    // **** Load emails using include variable **** $(Include=[$(vPathText)email.txt]);
    5. Note: Do not leave any spaces before or after the equal sign. If you do not use the square brackets, the file still loads, but the text displays in gray, like a comment.
  5. Load the data and verify that the EmployeeMail table is now part of your data model. Preview the table to examine its contents.
  6. To join EmployeeMail to Employees, return to the Email section within the Data load editor.
  7. Underneath the Include statement, add the following RESIDENT LOAD statement from EmployeeMail, which joins all the data to Employees:
    // **** Reload emails as a separate table so we can join it **** LEFT JOIN (Employees) LOAD * RESIDENT EmployeeMail;
    8. Note: We want to join the new EmployeeMail table to Employees, without modifying the original, external script. Therefore, a RESIDENT...LOAD and JOIN are used, and then the original table, EmployeeMail, is dropped.
  8. Drop the original EmployeeMail table by inserting the following script at end of the Email section:
    // **** Drop original email table **** DROP TABLE EmployeeMail;
  9. Load the data and check the Data model viewer to ensure the EmployeeMail table has been joined to Employees.
