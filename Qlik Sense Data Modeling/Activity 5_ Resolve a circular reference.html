Activity 5: Resolve a circular reference

Summary:

  • Load a table of shippers into the data model.
  • Examine the structure of the model and observe the circular reference generated.
  • Resolve the issue by aliasing a field.

Module: Resolving common modeling issues

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, load a table and eliminate the circular reference in your data model by doing the following:

  • Create a new Shipments script section and load the shippers table from the ABC_2025 database.
  • Label the table Shippers and add a comment to describe its contents.
  • Load your data and notice the circular reference in your data model.
  • In the Shippers table, rename the CompanyName field to ShipperName to eliminate the circular reference.
  • Load your data and verify that the circular reference has been eliminated from your data model.

Hints

  • The loops (circular references) must always be eliminated from the data model because they will cause data integrity issues.
  • Because of the circular reference in the data model, the OrderDetails has been converted into a loosely coupled table as it is the longest table.

 

Step-by-step guide
  1. Create a new script section after Employees, and title it Shipments.
  2. Use the ABC database connection to load the shippers table and make sure all the fields are selected.
  3. Relocate the table label [shippers] to above the preceding LOAD statement and capitalize the "S".
  4. Add a comment to describe its contents.
  5. Load the data and notice the error for the circular reference.
  6. Examine your data structure in the Data model viewer. A circular reference is shown because both the Customers and Shippers tables have a CompanyName field. These are not related. So, one can be aliased as another name.
  7. In the Shippers table loading script, alias the field as follows:
    CompanyName AS ShipperName;
  8. Load your data again and verify that there is no circular reference after the transformation.
