Data Modeling for Qlik Sense

Activity 3b: Data reduction with Section Access (QSoW)

Summary:

  • Create an app-level security layer via Section Access security.
  • Investigate the different levels of row and column-level access for users.

Module: Exploring security and advanced concepts

Qlik Sense app: ABC Sales Data / ABC Sales Data_SA2

Challenge

Add SECTION ACCESS with data reduction using the same users as before. To achieve this, complete the following tasks:

  • Examine the DataReductionSA.txt and DataReductionLink.txt files.
  • In the Qlik Sense Hub, open the ABC Sales Data app and add a DivisionName Filter control in the Employees app sheet.
  • In the Qlik Sense Hub, duplicate the app and rename the new app ABC Sales Data_SA2.
  • At the top of the script sections, add a new section named Security. Add the DataReductionSA.txt above the SECTION ACCESS area.
  • Add the DataReductionLink.txt after the SECTION APPLICATION statement, on the same section.
  • Load your data. Publish the app using the Qlik Management Console (QMC) into the Everyone stream.
  • Log on as FRL, then IHX, and then to BMN. Observe the different access levels. See how the data is reduced for the different users.
  • Close all Google Chrome browser windows.

Hints

  • The files that contain the security information are stored in the following location:
    C:\Qlik Sense Data\TextFiles\.
  • The first security file contains the SECTION ACCESS table. This table includes a SA_LINK field, used for the reduction of the available rows. It also includes an OMIT column to remove the JobTitle field. This first file is DataReductionSA.txt.
  • The second file is loaded as part of SECTION APPLICATION. It acts as a link table between the SECTION ACCESS table and the data model, reducing the available rows based on DivisionName. This second file is DataReductionLink.txt.

 

Step-by-step guide
  1. Open and examine the DataReductionSA.txt and DataReductionLink.txt files.
  2. Open the Qlik Sense Hub and make sure you are logged on as Student.
  3. In the ABC Sales Data app, open the Employees sheet and enable the edit mode.
  4. Make space for a new Filter control and add the DivisionName field onto the sheet as the new Filter.
  5. Disable the edit mode and navigate back to the Hub.
  6. Duplicate the ABC Sales Data app and rename it to ABC Sales Data_SA2.
  7. Navigate to the Data load editor. Create a new script section named Security. Move the new section to the top of the section list.
  8. In the Security section, add the following statement:
    SECTION ACCESS;
  9. After inserting the Section Access statement, use the TextFiles data connection to load the DataReductionSA.txt file.
  10. Make sure the Embedded field names option is selected and insert the script. This loads information for the access level, user ID, link to second table, and the OMIT fields column.
  11. After this script, add the following line:
    SECTION APPLICATION;
  12. After inserting the Section Application statement, use the TextFiles data connection again, to load the DataReductionLink.txt file.
  13. Make sure that the Embedded field names option is selected and insert the script. This loads the link table information associating each user to the relevant division they have access to.
  14. Load the data to add the security information and open the QMC.
  15. In the Apps page, select and publish the ABC Sales Data_SA2 app into the Everyone stream.
  16. From the Windows Start button, right-click on the Google Chrome icon, and select Run as a different user.
  17. Log in as Frank Roll, using FRL as the user ID and Training1 as the password.
  18. When Google Chrome opens, make sure you are viewing https://qse/hub/.
  19. When the Qlik Sense Hub opens, confirm the FRL credentials appear on the top bar. Select the Everyone stream.
  20. Open the ABC Sales Data_SA2 app and select the Employees sheet. Frank has access to all fields, but only data associated to North America.
  21. Log on as Ingrid Hendrix this time, with the user name IHX and the password Training1. Open the ABC Sales Data_SA2 app again. Select the Employees sheet. Ingrid only has access to North America and South America. She cannot see JobTitle data because this field has been omitted.
  22. Note that the SalesTitle is still available. Consider how you can also omit this field for Ingrid.
  23. What can you see when you log in as Bing Martin, using BMN as user ID? Why? (see Solution image)
  24. Close all the Google Chrome browser windows.
