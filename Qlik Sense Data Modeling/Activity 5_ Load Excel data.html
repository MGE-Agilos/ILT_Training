Data Modeling for Qlik Sense

Activity 5: Load Excel data

Summary:

  • Import data from two Excel files.
  • Script the loading of two tables, ensuring correct associations.

Module: Sourcing and loading data

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, load two new tables from two Excel files by doing the following:

  • Create a new script section, Employees, after the Customers section.
  • Use the DataFiles connection to import employees.xlsx and team.xlsx into your cloud tenant.
    • Qlik Sense on Windows: Create a new data connection to the ExcelFiles folder, found within C:\Qlik Sense Data, and name it ExcelFiles.
  • Load the following tables in the Employees section:
    • Employees, from the employees sheet in the employees.xlsx file. Rename the field EmpID to EmployeeID to associate the table with your data model.
    • EmployeeTeam, from the employee_team sheet in the team.xlsx file. Rename the field EmpID to EmployeeID for the same reason as above.
    • Team, from the team sheet in the team.xlsx file.
    • Add comments to all the tables.
  • Load your data and verify that the tables have been added and are associated with the existing data model.

Hints

  • The EmpID can be renamed to EmployeeID from either the script editor or the Select data dialog.
  • When the Select data dialog appears, the following must be selected:

    • File format: Excel (.XLSX)

    • Field names: Embedded field names

    • Header size: 0

 

Step-by-step guide
  1. In the Data load editor, create a new script section after the Customers section and name it Employees.
  2. From the Data connections panel, use Select data on the DataFiles connection.
  3. Sourcing the data from the student file collateral, drag the employees.xlsx onto the Select file drop zone. Do the same with the team.xlsx file.
    4. Qlik Sense on Windows: Create a new data connection to the ExcelFiles folder, found within C:\Qlik Sense Data, and name it ExcelFiles.
    Use the Select data option from the new connection, to launch the Select file dialog.
  4. With the Select file dialog still open, select the employees.xlsx file.
  5. In the Select data dialog, check that the following are automatically selected:
    • File format: Excel (XLSX)
    • Field names: Embedded field names
    • Header size: 0
  6. In the Fields area, rename the EmpID field to EmployeeID to associate the new table with the existing data model. You can now see that the EmpID has an alias to EmployeeID using the AS keyword.
  7. Insert the generated script into the Data load editor.
  8. Name the new table Employees: and, as good practice, add a suitable comment before the table label.
  9. With your cursor placed at the end of the script section, add a few carriage returns for easier readability and use the same data connection again to add more tables.
  10. Select the team.xlsx file and, when the Select data dialog appears, select both the employee_team and team tables. Note the numbers on the right of the table name show the number of selected fields.
  11. In the employee_team table, replace the EmpID with EmployeeID and make sure the field is checked.
  12. Insert the script and make sure that the script statement has been inserted.
  13. Name the tables EmployeeTeam and Team, respectively, and add suitable comments before each table label. The code in your Employees section resembles the following:
    14. // **** Load Employees table **** Employees: LOAD EmpID AS EmployeeID, "First Name", "Last Name", DateOfBirth, StartDate, EndDate, OfficeID, JobTitle FROM [lib://DataFiles/employees.xlsx] (ooxml, embedded labels, table is Employees); // **** Load EmployeeTeam table **** EmployeeTeam: LOAD EmpID AS EmployeeID, TeamID FROM [lib://DataFiles/team.xlsx] (ooxml, embedded labels, table is employee_team); // **** Load Team table **** Team: LOAD TeamID, Team FROM [lib://DataFiles/team.xlsx] (ooxml, embedded labels, table is team);
  14. Click the Load data button and check your data model in the Data model viewer.
