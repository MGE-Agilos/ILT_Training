Data Modeling for Qlik Sense

Activity 3: Resident load SalesPerson table

Summary:

  • Use an existing table as a source of data for a new table.
  • Restrict the values in the new table with a Where clause.

Module: Generating data

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, create a new RESIDENT table by reusing existing data from the Employees table. To achieve that, follow the process below:

  • In the Employees script section, locate the Employees table and create a Name field by concatenating the fields First Name and Last Name.
  • Create a new script section after Suppliers and name it SalesPerson.
  • Create a new table using RESIDENT LOAD that retrieves data from the Employees table with the following fields:
    • EmployeeID
    • Name AS SalesPerson
    • JobTitle AS SalesTitle
  • Add a WHERE clause to filter the number of records you are loading in the new table to only those whose title contains the words “Sales” or “Account”.
  • Name the new table SalesPerson and add a comment to describe the table.
  • Load your data and verify that the new table has been created.
  • Create a new Employees sheet to validate your data.
  • In the new sheet, add the fields Name, JobTitle and SalesTitle as Filter panes. Select all the SalesTitle values.
  • Verify that only a part of the JobTitle values is associated to your selections because only a subset of the Employees table is loaded in the SalesPerson table.

Hints

  • Only the Employees records that satisfy the criteria defined with the WHERE clause are added to the SalesPerson table.

 

Step-by-step guide
  1. In the Data load editor, navigate to the Employees section and locate the Employees table.
  2. Replace the First Name and Last Name fields with the following line:
    ... "First Name" & ' ' & "Last Name" AS Name, ...
  3. Create a new section between Suppliers and Shipments. Name it SalesPerson.
  4. Create a new LOAD statement to make a table for SalesPerson based on the Employees table. You cannot do this using Select data from a data connection; you need to type in the code. Add the following statement (with comment and label) to your script.
    5. // **** Load SalesPerson from Employees **** SalesPerson: LOAD EmployeeID, Name AS SalesPerson, JobTitle AS SalesTitle RESIDENT Employees;
  5. Limit the load of all Employees records to just those you can identify as sales people. To do this, add a WHERE condition after the RESIDENT line, but before the semicolon, as follows:
    6. WHERE JobTitle LIKE '*Sales*' OR JobTitle LIKE '*Account*';
    Note: The LIKE operator works with the wildcard (*), searches the field, and finds all values that contains the text Sales or Account.
  6. Load your data and verify that the new table has been created.
  7. Create a new sheet in the app and name it Employees.
  8. Open the new Employees sheet and enable the edit mode.
  9. Add the dimension Name to your sheet as a Filter pane. Resize it so it is 8 columns wide and place it on the left of the sheet.
  10. Create two more Filter panes for JobTitle and SalesTitle dimensions. Make them the same width as Name.
  11. Disable edit mode and select all values from the SalesTitle filter pane to review the results.
    Notice that only specific Name and JobTitle values are shown as possible values associated with the selection. To include a value from the original Employees table in the new logical SalesPersons table, at least one of the defined conditions for the JobTitle, containing either the text “Sales” or “Account”, must be satisfied.
