Data Modeling for Qlik Sense

Activity 1: Order level aggregation

Summary:

  • Create a new table from OrderDetails, using the Order level of granularity.
  • Join the table to Orders.
  • Load the data and review the new field in the model.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, create a new field that represents the total sales value for each order by doing the following:

  • At the end of the Orders script section, add a LOAD statement to create the order-level table. The new table contains:
    • Two fields: OrderID and Sum(LineSalesAmount) AS OrderSalesAmount.
    • A GROUP BY clause at the end of the LOAD statement to aggregate the OrderSalesAmount field by OrderID.
    • Data from the OrderDetails table, by using a RESIDENT LOAD statement.
  • LEFT JOIN the new table to the Orders table to add the OrderSalesAmount field.
  • Load the data and preview the Orders table in the data model to verify that the OrderSalesAmount field exists.

Hints

  • There is no need to add a label for the new table as it is joined to the Orders table.
  • The location of the new table is significant. It must be created after the LOAD statement that creates the LineSalesAmount field, OrderDetails.

 

Step-by-step guide
  1. Navigate to the end of the Orders script section and add the following statement:
    //**** Aggregate LineSalesAmount to order level **** LOAD OrderID, SUM(LineSalesAmount) AS OrderSalesAmount RESIDENT OrderDetails GROUP BY OrderID;

    This statement aggregates the LineSalesAmount value from each line in the OrderDetails table over each occurrence of OrderID. The GROUP BY clause is needed to aggregate, or group, fields other than those included in aggregations. In this case, it totals the LineSalesAmount for each OrderID.
  2. The result in this table needs to be added to the Orders table. To achieve this, LEFT JOIN the table to Orders by typing the following before the LOAD statement of this new table:
    LEFT JOIN (Orders)
  3. Load your data.
  4. In the Data model viewer, preview the Orders table to check that the fields and data exist. The new OrderSalesAmount field simplifies some calculations and measures in sheet objects, such as the % of sales over the total Order sale per LineNo. This is achieved by using the OrderID and LineNo as dimensions and the following measure:
    Sum(LineSalesAmount) / Sum(OrderSalesAmount)
