Data Modeling for Qlik Sense

Activity 9: IntervalMatch function

Summary:

  • Re-engineer the PriceGroup expression to leverage an IntervalMatch construct.
  • Assemble a temporary, inline table to hold the value boundaries.
  • Ensure the final result is the same as prior data classification techniques.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, use the IntervalMatch function to create a price grouping for each product’s unit price. To accomplish this, complete the following tasks:

  • In the Products script section, comment out the field PriceGroup.
  • Create a new script section before LoadComplete and name it IntervalMatch.
  • Create an INLINE table with three columns: Start, Stop, and PriceGroup. Name the table PriceGroup.

  • Create four groups in this table, by defining the Start, Stop, and PriceGroup label value for each group, as below:

    • >= 0 < 100
    • >= 100 < 500
    • >= 500 < 1000
    • >= 1000
  • Create the IntervalMatch table to match the CataloguePrice values from the four groups.
  • LEFT JOIN the new table to the PriceGroups table to eliminate the synthetic key.
  • Drop the fields Start and Stop as they are no longer needed.
  • Load the data and verify there are no issues in the data model.
  • Navigate to the Price Groups sheet to check the results.

Hints

  • The price group values can be kept to an external source, meaning that there is no need to manipulate the script every time someone needs to modify the group limits.
  • When the IntervalMatch function is used, a synthetic key is always created. Joining the table is the usual way of eliminating this synthetic key.

 

Step-by-step guide
  1. Navigate to the Products script and comment out the entire set of nested IF statements relating to the PriceGroup field.
  2. Add a new section before the LoadComplete section named IntervalMatch.
  3. Create an INLINE table to store the price group intervals.
    Note: An Inline table is a table that is created entirely in the script along with its data in the same statement.
    Type the following statement into the IntervalMatch section:
    // **** Load product groups as inline table **** PriceGroup: LOAD * INLINE [ Start, Stop, PriceGroup 0, 100, >= 0 < 100 100, 500, >= 100 < 500 500, 1000, >= 500 < 1000 1000, , >= 1000 ];

    This script creates a new table named PriceGroup, with interval data, using an INLINE LOAD statement. The structure inside the square brackets contains the table information. The first row has three column headings. The subsequent rows are data rows, also with three columns of data.
    Note: Notice that the end of each line does not end with a comma. All row data must be on the same line with each value separated by a comma.
  4. To create an IntervalMatch with CataloguePrice and your INLINE table, type the following script after the PriceGroup table:
    INTERVALMATCH(CataloguePrice) LOAD Start, Stop RESIDENT PriceGroup;
  5. Load the data and preview the data model: The PriceGroup table is your INLINE table. The PriceGroup-10 table is the result of the IntervalMatch statement. The $Syn 1 table is the synthetic table created because both tables contain the Start and Stop fields.
    Note: The IntervalMatch method will always lead to a synthetic key in your data model. The standard process to eliminate this synthetic key is to Left Join the new IntervalMatch table to the table that contains the group definitions.
  6. You must now eliminate the synthetic table by joining PriceGroup-10 into PriceGroup with a LEFT JOIN. Add the LEFT JOIN statement between INTERVALMATCH and LOAD as follows:
    INTERVALMATCH(CataloguePrice) LEFT JOIN (PriceGroup) LOAD Start, Stop RESIDENT PriceGroup;
  7. Load the data and verify that the data model does not contain any synthetic keys.
  8. Remove the Start and Stop fields with a DROP FIELDS statement. Add this statement at the end of the IntervalMatch section as follows:
    DROP FIELDS Start, Stop;
  9. Load the data again. Within Analyze mode, open the Price Groups sheet. Make sure the table is sorted by PriceGroup, in ascending, alphabetical order.
    Note: After it is created, the IntervalMatch solution is the easiest to maintain while still allowing flexibility for varying intervals. The Inline Load from this activity, has been used only for the scope of this module. In reality, the Inline Load would be replaced by data from an external source, meaning that interval ranges can be set by other systems or modified by authorized users.
