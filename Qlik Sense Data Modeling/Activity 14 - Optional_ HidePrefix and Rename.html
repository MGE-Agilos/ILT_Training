Data Modeling for Qlik Sense

Activity 14 - Optional: HidePrefix and Rename

Summary:

  • Set up a prefix character for hiding fields from the data model.
  • Rename key fields to ensure they are not visible to front-end developers.

Module: Applying finishing touches

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, hide the field names that begin with a % symbol by doing the following:

  • In the Main script section, create the HidePrefix variable with the value '%'.
  • In the LoadComplete script section, before the vLoadComplete variable, rename the following fields:
    • OrderID to %OrderKey
    • EmpYrQtr_Key to %EmpYrQtr_Key
    • CustomerID to %CustomerKey
    • EmployeeID to %EmployeeKey
    • ProductID to %ProductKey
  • Open a random app sheet and edit it. In the Assets panel, search for Order. Verify that the %OrderKey does not appear, because it is hidden.
  • Search for more fields you have hidden.

Hints

  • The SET statement can be used to create a variable.
  • At the end of the script, a number of RENAME statements must be used with the addition of a % symbol at the start of field names, hidden from the users.

 

Step-by-step guide
  1. Select the Main script section and add the following line at the end of the SET statements:
    SET HidePrefix = '%';
  2. Navigate to the LoadComplete section and add the following before the vLoadComplete variable.
    // **** Rename key fields to hide **** RENAME FIELD OrderID TO %OrderKey; RENAME FIELD EmpYrQtr_Key TO %EmpYrQtr_Key; RENAME FIELD CustomerID TO %CustomerKey; RENAME FIELD EmployeeID TO %EmployeeKey; RENAME FIELD ProductID TO %ProductKey;
  3. Load the data, switch to Analyze mode, and open a random sheet.
  4. Enable the edit mode on the sheet. Navigate to the Fields in the Assets panel.
  5. Search for Order. Verify that the list of fields displayed from the search does not contain either %OrderKey or OrderID, because OrderID is renamed and hidden.
  6. Search for other fields you have hidden to verify they are not there.
Note: Although you can hide fields from the users, you have to consider if they are needed. %OrderKey may be a key field and is hidden, but it does contain OrderID information, by which the users may want to search. In this case, you would need to add OrderID as a separate non-key field.
Note: As opposed to hiding, it is advisable to Drop unused, non-key fields from the data model.
