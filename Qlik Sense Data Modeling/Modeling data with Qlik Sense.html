Qlik Sense deployment architecture

An app in Qlik Sense:

  • Is a collection of reusable data items (visualizations, dimensions, measures).
  • Is a self-contained entity that includes the data to analyze in a structured data model.
  • Is a binary file or a QVF file.
  • Can contain different types of business information.
  • Allows users to make data discoveries and decisions using data visualizations based on selections.

The building blocks of an app are:

  • Reusable items — visualizations, bookmarks, stories
  • Data load script — connects to a data source and retrieves the data
  • Data model — provides structure for the loaded data
  • Expressions — calculations involving functions, fields, and operators
  • Measures — named calculations for use in visualizations
  • Dimensions — named fields for use in visualizations

Data sources and data structures

A traditional relational database matches data by using common characteristics found within a data set. A data set is a collection of data usually presented in a tabular form.

The relational database is defined by several rules:

  • The database is structured so that information is related across multiple tables, each consisting of rows and columns.
  • The database must support a query language.

One of the most commonly used database query languages is SQL (Structured Query Language). SQL is a special-purpose programming language, designed for managing data held in a RDBMS (Relational Database Management System).

  • A SQL statement can be thought of as the smallest standalone element of a programming language.
  • The most common SQL statement is Select, which queries specific result sets.
  • A program is formed by a sequence of one or more statements.
SQL syntax examples
SELECT * FROM Products WHERE ProductID = 1004005; SELECT ProductName, CategoryID, QuantityPerUnit, UnitPrice FROM Products WHERE ProductID = 1004005 OR ProductID = 1005006;

Qlik Sense data source formats

Qlik Sense can load data from many file formats, including:

  • Database tables and views (SQL query)
  • Text files (fixed width or character delimited), such as csv
  • Microsoft Excel spreadsheets (xls, xlsx)
  • XML files
  • KML files
  • HTML tables
  • QVD files (QlikView & Qlik Sense formats)
  • QVX files (QlikView & Qlik Sense data exchange formats)

Database vs. Qlik Sense data structures

There are key differences between a relational database data structure (like an Access database) and a Qlik Sense data structure.

Relational database data structures

  • The tables have defined relationships and are associated through common (key) fields of the same data type.
  • A key field is either a column or a combination of columns in a database that uniquely identifies a specific entry or record.
Qlik Sense data structures

  • Qlik Sense allows the Data Architect to define the relationships between tables, even if the fields do not have the same name or type in the source.
  • The table relationships or associations are automatically defined based on like-named fields.

Data associations in Qlik Sense

  • Each field loaded into Qlik Sense becomes a field in the associative data structure.
  • Fields that exist in more than one table with identical names are associated and identified as key fields.
  • Each field can be presented in the form of a filter pane in Qlik Sense.
  • Qlik Sense reveals logical connections based on the internal data structure. The values associated with your selection are also identified.

Qlik Sense data modeling

A data model is a conceptual model that describes data relationships and how data is accessed in the system.

Dimensions

A dimension is used to categorize data, such as products, customers, and so on. It consists of one or more tables containing keys and attributes that describe the data values.

Facts

A fact table generally contains foreign keys of dimension tables along with measurement data, typically represented as a numeric value. Each row of the fact table is defined by the set of dimensional keys that describe the measurements. An example of a fact table is sales revenue.

Star schema and snowflake schema

A schema is a way to graphically represent a model through a diagram.

(Click on each image to enlarge)

Star schema
... is a data model with a single fact table or a linked table, surrounded by a set of dimension tables. The single fact table can be the result of concatenating several fact tables.

Snowflake schema
... is a data model where the centralized fact tables (one or more) are connected to multi-level dimension tables.

Working with Qlik Sense

Qlik Sense SaaS offers a Data Integration and Analytics platform where users can get actionable insights from raw data.

In Analytics Services, Qlik Sense works with user data from different data sources or the data catalog.

The Qlik Sense SaaS platform allows users to:

  • Extract, transform, aggregate, combine, and associate data
  • Visualize, analyze, search, and select data
  • Discover and share data insights
  • Collaborate with other in shared development areas
  • Monitor metrics and charts and send alerts

Qlik Sense Hub

The data exploration starts in the Qlik Sense hub where users can create and access apps.

When Qlik Sense launches, the hub is displayed along with all the apps to which a user has access.

Apps in the Qlik Sense hub can be shared and organized into streams or spaces. Depending on the environment, Qlik Sense Enterprise on Windows (QSEoW) or Qlik Cloud (QS SaaS), these options are available:

  • Work area (QSEoW) or Personal space (QS SaaS)— where you edit and upload your existing apps and create new ones.
  • Stream (QSEoW) or Space (QS SaaS)— where published apps are organized or developed collaboratively. In QSEoW, the Everyone stream is the default stream, but additional streams are usually created to satisfy access requirements. In QS SaaS, users and administrators can create their spaces and invite other members.

Search for more info about the Hub F in the Help site.

App overview

Qlik Sense is a web-based platform that has a number of different views to work from.

When you open an app from the hub, the App overview is displayed. From there, you can:

  • Create and edit sheets
  • Apply existing bookmarks to navigate to a specific sheet with selections applied
  • Work with data storytelling and apply bookmarks

The App overview displays a summary of the main components in the app (sheets, bookmarks, and stories). From this area, you can also open the app details section and configure global settings for the app.

In addition, the App overview can be used to navigate to different views, or work functions, categorized as:

  • Prepare: Add, transform, and load data, and manage data associations. Additionally, in Qlik Sense Cloud editions, you can define the business logic to improve the Insight Advisor when providing insights or searches in the data.
  • Analyze: Navigate to existing sheets. Edit and create sheets for analysis, create new bookmarks, and generate insights.
  • Narrate: Create, edit, and play stories.

The view of choice depends on whether you are working with the data and building a data model, exploring and discovering data, or working with data storytelling.

In a new app, only the Prepare drop-down menu is available. Once data has been added into the app, the other toolbar items become available to the user.

Sheet view

In the Sheet view F , you can:

  • Create and edit visualizations (edit mode)
  • Analyze data
  • Take snapshots of visualizations for building stories
  • Work with bookmarks
  • Add thumbnail images to sheets for easy identification
  • Download or export most visualizations as an image, PDF, or data
  • Copy an embed code or link to share a visualization
Insights view

In the Insights view F , you can:

  • Generate and find new insights in your data
  • Search in the app using natural language capabilities
  • Identify existing visualizations or create additional ones based on the generated insights
  • Save and create new measures based on the search results and filters
Storytelling view

In the Storytelling view F , you can:

  • Create stories and share business insights
  • Publish stories to share with other users

  • Allow users to play back a story and act on discoveries

Data model viewer

In the Data model viewer F , you can:

  • See an overview of the data structure of your app
  • Display metadata about the tables and fields
  • Create master dimensions and measures from the fields
Data load editor

In the Data load editor F , you can:

  • Connect your app to data sources
  • Create, edit, and run data load scripts to load data
  • Use the debugging tool to diagnose script issues
Data manager

In the Data manager F , you can:

  • Add your data using the existing Files and other sources option
  • Manage data table associations and make association recommendations with all possible table pairs
  • Auto-generate data load script, so you can use it in your app without learning a scripting language
  • View all data tables in the app, whether you added them using Add data, or loaded them using script in the Data load editor
  • Add, edit, and delete tables, and add calculated fields to perform various table operations
  • Apply changes and reload data
  • Manage automatically concatenated and joined tables
Business logic (Logical model and Vocabulary)

In the Business logic F, you can work on a Logical model and the Vocabulary sections to:

  • Define the business rules and metadata to customize and guide insight generation and interaction.
  • Group fields in data categories and classify each field based on data classes.
  • Define the default aggregation for each field.
  • Decide to include or exclude data values in the Lookup operation at field level.
  • Hide or show fields from the Insight Advisor field list and search.

  • Create vocabularies for Insight Advisor defining terms and values that may be used in queries that are not present in your data model.

In a Qlik Sense app, the following building blocks are:

  • Data load script — connects to a data source and retrieves the data
  • Data model — provides structure for the loaded data
  • Expressions — calculations involving functions, fields, and operators
  • Measures — numerical data fields calculated for use in visualizations
  • Dimensions — named fields for use in visualizations
  • Visualizations — chart objects and other sheet objects that represent data
  • Bookmarks — stored data selections that can be applied by users

Creating a new app

To create a new app within the Qlik Sense hub, use the Add new button from the toolbar, and select New analytics app.

To upload an app, use the same Add new button from the toolbar, but select Upload app. To successfully complete this process, the data connections must be re-created. Ensure that the app has access to the data sources if a reload is needed.

Note: From the Qlik Sense hub, you can also duplicate an existing app. This creates a new app in the hub that can be renamed and edited.

Loading data using drag and drop

The drag and drop data load is a Qlik Sense feature that:

  • Allows users to quickly load local data by dragging and dropping files into their app or by making quick file selections.
  • Does not need to create data source connections or write load scripts.
  • Can load data from different file types, such as Excel, .txt, .csv, or .qvd. It is useful if the data is in a tabular format, cleansed, and has headers.
  • Is supported in Qlik Sense Saas and Qlik Sense on Windows.
  • Is available from the Add data option. When an app is opened for the first time or no data exists, this option is displayed in the App overview. You can also access the Add data option from the Navigation menu.

Data load editor environment

The Data load editor is:

  • An embedded editor to create and edit a data load script for a data model.
  • Used to define Data connections for multiple data sources, extract data from them, and load the data into the app.

You can type a script manually, generate it automatically, or even read it from an external source. After you have loaded the data and the data model is created, it is available within the app for analysis through visualizations.

The Data load editor components are:

Toolbar

The toolbar contains theNavigation menu and other useful commands.

Data connections

In Data connections, commonly used shortcuts to data sources (databases work flows or remote files) can be created and saved.

Text editor

The Text editor is where you edit the script code. Each script line is numbered and the script is color-coded by syntax components.

Sections

Sections contains the names of related script. If you have added data using Add data, you will have a data load script section named Auto-generated section that contains the script code required to load the data.

Output

Output displays all messages that are generated during data load.

Script sections

When you create a new app, only one script section exists. However, you may create new sections to divide the load script and organize the structure, making it easier to work with and maintain.

Sections:

  • Are executed sequentially in top to bottom order from the Sections panel.
  • Can be renamed and moved to change the order of script execution, if required.

The default section is called Main and contains a number of initial SET statements to establish the default setting for the environment. It is best practice to leave these SET statements at the beginning of the script.

When using the Data manager to add data, locked Auto-generated sections are created.

These sections:

  • Are created when using the Add data option. Qlik Sense uses the Data manager to generate its own script sections.
  • Have a symbol on the right, to indicate they are read-only.
  • Can be viewed by clicking on them.
  • Display a warning in the script area to indicate that the section was auto-generated by the Data manager.
  • Can be automatically updated by the Data manager.
  • Can be modified by clicking the Unlock button. This detaches the auto-generated section from the Data manager and the section is not an auto-generated section anymore.
Note: If you unlock the auto-generated script, you will not be able to edit the data with the Data manager. Unlocking cannot be undone.
