Data Modeling for Qlik Sense

Activity 8 - Optional: If statements

Summary:

  • Re-work the previous PriceGroup expression to utilize nested IF statements.
  • Analyze the resulting data classification field within the existing table object.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, use the IF function to create groupings, like CLASS, but with varying sizes of buckets. You can achieve that by doing the following:

  • In the Products script section, comment out the existing PriceGroup field.
  • Create a new field called PriceGroup. This time, use nested IF statements.
  • Make sure the IF statements create four groups based on the UnitPrice value:
    • >= 0 < 100
    • >= 100 < 500
    • >= 500 < 1000
    • >= 1000
  • Load the data and check your data model to verify the field has been created.
  • Navigate to the Price Groups sheet and check the results.

Hints

  • The IF statement refers to the UnitPrice, and not the CataloguePrice field, because it is created in the same LOAD where CataloguePrice is defined.
  • This approach can prove to be complex and difficult to maintain if there are many different group ranges.

 

Step-by-step guide
  1. In the Data load editor, navigate to the Products script section and locate the Products table.
  2. Comment out the PriceGroup field you created earlier. Add the following script to a new line, after the commented field:
    IF(UnitPrice>=0 AND UnitPrice<100,'>= 0 < 100',
    This is the first part of the IF statement. If the product meets the condition (UnitPrice between 0 and 100), the then part of the expression is used and the result is the value of >= 0 < 100. If the product does not meet the condition, the else portion is used.
  3. For the else portion, start a new nested IF statement with a new condition. The script continues with several nested IF statements. Complete the statement so it reads as follows:
    4. IF(UnitPrice>=0 AND UnitPrice<100,'>= 0 < 100', IF(UnitPrice>=100 AND UnitPrice<500,'>= 100 < 500', IF(UnitPrice>=500 AND UnitPrice<1000,'>= 500 < 1000', IF(UnitPrice>=1000,'>= 1000')))) AS PriceGroup,
  4. Load the data and preview the Products table in the data model to see the new intervals.
  5. In the App overview, open the Price Groups sheet and make sure that the table is sorted alphabetically, by PriceGroup ascending.
