Data Modeling for Qlik Sense

Activity 12: Data islands

Summary:

  • Load exchange rate data as a data island in the data model.
  • Leverage expressions and field settings to utilize the exchange rate data values within analysis mode.

Module: Applying finishing touches

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, include currency exchange data. Do the following to create a multi-currency functionality in the app:

  • Load the ExchangeRates data set from a local text file.
  • Keep the new table as a data island and use expressions in the sheets to display the difference.
  • Create a new sheet, Currencies, and add the QuoteCurrency as a Filter pane.
  • Insert a Table object with:
    • CompanyName as the dimension
    • Sum(LineSalesAmount) as the first measure with the name Sales
    • Sum(LineSalesAmount) * [ExchangeRate] as a second measure, with Number formatting as Money and the following dynamic Label:
      = 'Sales in' & Only([QuoteCurrency])
  • Change the QuoteCurrency properties to only allow one selected value. Select a value and observe the results.

Hints

  • The new ExchangeRate table is not associated with the existing data model. This means that the association must be created at the expression level.
  • The table does not work if more than one value or no values are selected in the QuoteCurrency field.

 

Step-by-step guide
  1. In the Data load editor, create a new section after MappingTables named Islands.
  2. From the Data connections panel, use Select data on the DataFiles connection.
  3. Sourcing the data from the student file collateral, drag the ExchangeRates.txt file onto the drop zone to import it.
    4. Qlik Sense on Windows: Use the TextFiles connection you created previously.
  4. Once imported, select ExchangeRates.txt to open it.
  5. Ensure the following settings are being used:
    • File format: Delimited
    • Field names: Embedded field names
    • Delimiter: Comma
    • Header size: 0
    • Quoting: MSQ
  6. Insert the script and label the table ExchangeRates. Provide suitable commentary.
  7. Update the file source with a variable, if available.
  8. Execute the data load script and examine your data model. Note that the new table is a data island, as it is not associated with the rest of the data model.
  9. Select the ExchangeRates table and expand the Preview. This table will be used to convert some measures, like Sales amount, into the corresponding selected QuoteCurrency.
  10. Within Analyze mode, create a new sheet, Currencies.
  11. Add the QuoteCurrency field onto the sheet as a Filter pane.
  12. Insert a Table object with CompanyName as the dimension and the Master Measure of Sales as the first measure.
  13. Add a second measure with the following properties:

    14. Label = 'Sales in ' & Only([QuoteCurrency])
    Number formatting Money
    Measure Sum(LineSalesAmount) * [ExchangeRate]

    Note: When using Money as the Number formatting, Qlik Sense utilizes the settings from the data load script to determine the format pattern. If required, it can be overridden.
  14. In order for the above expression to work, only one selected value must be allowed in the QuoteCurrency field. From the Assets panel, under the Fields section, right-click on the QuoteCurrency field. Choose Field settings and enable the Always one selected value check box. This prevents the user from selecting multiple values in the field.
  15. Disable the edit mode.
  16. Change the currency selected for Pound sterling in the QuoteCurrency Filter pane. Observe the results.
  17. Return to edit mode for the Currencies sheet.
  18. From the Assets panel, under the Fields section, right-click on the QuoteCurrency field again. Choose Field settings and disable the Always one selected value check box. This returns the field to the normal mode of operation.
  19. Disable edit mode and clear all selections.
