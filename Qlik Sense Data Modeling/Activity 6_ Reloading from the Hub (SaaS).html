Data Modeling for Qlik Sense

Activity 6: Reloading from the Hub (SaaS)

Summary:

  • Initiate app reloads from the Hub.
  • Analyze reload logs.
  • Set up reload schedules.

Module: Exploring security and advanced concepts

Qlik Sense app: ABC Sales Data

Challenge

Investigate how reloads can be initiated from the Hub:

  • Perform an interactive reload from the Data load editor.
  • Observe the app reload History.
  • Set up a reload schedule for the ABC Sales Data app.
  • Alter a reload schedule and analyze a reload log.

Hints

  • Do not unintentionally leave a recurring reload schedule running on an app, as it consumes resources.

 

Step-by-step guide
  1. Ensure you are logged in to Qlik Cloud with your primary Qlik ID.
  2. Navigate to the Personal space and use the Types filter to limit the display to just Apps.
  3. Open the ABC Sales Data app and visit the Data load editor.
  4. Use the Load data button to execute an interactive reload of the app data model.
  5. When finished, close the Data load progress dialog box, then return to the Hub.
  6. Use the More actions menu for the ABC Sales Data app to open the menu, and select Details.
  7. Observe the Last reload date for the app as being the current date and time.
  8. Switch to the Reload history tab.
  9. Note that the history is empty.
    10. Note: Interactive reloads from the Data load editor do not get logged in the Reload history.
  10. Use the More actions ellipsis to open the menu, and select Reload now.
    A notification appears at the top to indicate reload of the app has started. Once executed, another notification appears to state it is complete.
    An entry appears in the Reload history log with the relevant details.
  11. Click the View button to review the new entry in the Reload history log.
  12. Observe that while most of the log is an echo of what can be witnessed when performing an interactive reload, there is a single line at the top which indicates the ReloadID associated with this occurrence of a reload for the app.
  13. Close the Reload summary dialog box.
  14. Use the More actions ellipsis again, to open the menu. This time select Reload schedule.
  15. Click Add schedule.
  16. Change the Reload this app drop-down to Yearly.
  17. Analyze the various options for setting repetition.
  18. Select the prior month, and deselect this month, so that the next scheduled execution will take place 11 months from now.
  19. Save the Reload schedule to set it, and return to the App Reload history page.
    20. Note: Tenant owners and admins can display all scheduled reloads for the tenant, as well as manage them to some degree.
  20. While you have no access to the Management Console in the training environment, to edit the Reload schedule of an app, locate the ABC Sales Data app, and use the ellipsis actions button.
  21. Select Reload schedule.
  22. Because the app already has a reload schedule set, the current settings are displayed in the dialog box and can be altered. Use the Delete option to cancel the reload schedule.
  23. Click Save to update the app reload schedule.
  24. Use the Back button to return to the Catalog.
