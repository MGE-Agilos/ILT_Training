Data Modeling for Qlik Sense

Activity 6: Develop a master calendar

Summary:

  • Evaluate the required calendar end points.
  • Generate a temporary table of all dates from start to end, with no gaps.
  • Create and verify the final master calendar.

Module: Generating data

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, create a temporary table that gathers the starting and ending points for the calendar by using the earliest and latest order dates from the Orders table. To achieve this, complete the following tasks:

  • Create a new script section after SalesPerson and before Shipments, named MasterCalendar.
  • In the new section, create the table MinMaxTemp that holds the minimum and maximum dates from the OrderDate field.
  • In the Orders table, apply the FLOOR function to the OrderDate field to keep only the integer part of the date value.
  • Use the PEEK function to store the max date in the vToday variable. For this course, assume that today's date is the maximum date in the OrderDate field.

In the ABC Sales Data app, generate a temporary calendar table to store all the dates between the minimum and maximum dates, inclusive of both dates. To achieve this, complete the following tasks:

  • In the MasterCalendar section, create the MasterCalendar_Temp table.
  • In the new table, create a field with all the dates between the minimum and the maximum order date.
  • Load your data and verify that MasterCalendar_Temp is now part of your data model and it contains the expected values.

In the ABC Sales Data app, create the MasterCalendar table and then validate it by doing the following:

  • Navigate to the MasterCalendar section. At the bottom of the script section, copy and paste the contents of the master calendar script from the file in the path below.
  • Load your data and verify that the MasterCalendar table exists in your data model with the correct fields.
  • Drop the table MasterCalendar_Temp as it is no longer needed.
  • Create a new app sheet named Calendar, and add a Table with the following fields:
    • OrderDate
    • sum(OrderRecordCounter) as Orders
  • Add the dimensions Year, Month and Day as Filter panes , and make selections to check that the data filters correctly.

Hints

  • The FLOOR function removes the decimal part of the value in the OrderDate field. The decimal part represents the time.
  • The vToday variable has been added to use the maximum value of the OrderDate field as the current date. This allows consistent calculations with the calendar flag fields. (The use of variables in development is covered within another module.)
  • The LOAD...WHILE is used to generate data in the script.
  • The IterNo() F is a function that is very commonly used to generate incremental values together with a WHILE clause.
  • Qlik Sense date and time functions are used to create all the date-related fields.
  • The MasterCalendar table needs to associate with the OrderDate field in the Orders table.

 

Part 1 - Acquire the starting and ending points for the calendar

Step-by-step guide
  1. Create a new script section between SalesPerson and Shipments and name it MasterCalendar.
  2. In the new section, type the following:
    // **** Retrieve Min and Max dates from OrderDate field **** MinMaxTemp: LOAD Date(Min(FieldValue('OrderDate', RecNo()))) AS MinDate, Date(Max(FieldValue('OrderDate', RecNo()))) AS MaxDate AUTOGENERATE FieldValueCount('OrderDate');

    The DATE function used in the above script is not needed to create the master calendar. It is only added to help you verify the values in the Data model viewer and it will be removed shortly.

  3. In the Orders section, locate the Orders table and alter the line for OrderDate to read:
    DATE(FLOOR(OrderDate)) AS OrderDate,
    This ensures that OrderDate does not have a time component and associates with the MasterCalendar. The Floor F function converts the date number into an integer.
  4. In the Orders table, comment out the Year, Month, and Day fields, as they will not be needed after you create the master calendar. Ensure the last field in the preceding LOAD has no comma after it.
  5. Load the data and look for the new table MinMaxTemp in your data model. Preview its data and verify that it contains two values, the minimum and the maximum of the OrderDate field.
  6. In the Data load editor, navigate to the end of the MasterCalendar section and add a couple carriage returns.
  7. Use the Peek F function to get the value from the MaxDate field in the MinMaxTemp table and store it into the variable vToday. The MinMaxTemp table contains only one row. So, in the PEEK function, set the row number to 0. Create the new variable for the today's date by typing the following script:
    LET vToday = NUM(PEEK('MaxDate', 0,'MinMaxTemp'));
    Note: For the purpose of this activity and the supporting data, today’s date is assumed to be the same as the latest order date. In a real example, you would use the Today() function.
    8. Note: The development and use of variables within data load scripts are covered in another module.

 

Part 2 - Generate a temporary table of gapless dates

Step-by-step guide
  1. In the MasterCalendar section, remove the two DATE functions from the MinMaxTemp table. The script should look like the following:
    // **** Retrieve Min and Max dates from OrderDate field **** MinMaxTemp: LOAD Min(FieldValue('OrderDate', RecNo())) AS MinDate, Max(FieldValue('OrderDate', RecNo())) AS MaxDate AUTOGENERATE FieldValueCount('OrderDate');
  2. Use the two values from the MinMaxTemp table to create a new table with one field, OrderDate. This field is associated with the Orders table and it contains all the dates from the minimum to the maximum. This table will act as the base for the master calendar table. At the bottom of the MasterCalendar section, add a couple carriage returns and the following code:
    //**** Create the OrderDate field **** MasterCalendar_Temp: LOAD Date(MinDate + IterNo() - 1) AS OrderDate RESIDENT MinMaxTemp WHILE MinDate + IterNo() - 1 <= MaxDate;
    • The WHILE clause controls the iteration.
    • The loop will stop when the MinDate + IterNo() - 1 is bigger than the MaxDate.
    • The IterNo() function returns an integer that indicates for which time a record is evaluated in a LOAD statement with a WHILE clause.
      • The first iteration has number 1. This number is used to increase the start date MinDate in each record 1 more day.
      • To also include the MinDate value, you need to use the -1 in the expression.
    • The OrderDate field is generated with all the dates between MinDate and MaxDate.
    • The DATE function is applied to the final value to format the value into a date format.
  3. The MasterCalendar_Temp table will be used in the next step as the base for the main calendar table. The MinMaxTemp table has served its purpose, so it is no longer needed. Safely remove it by using the Drop Table statement at the end of the script as follows:
    DROP TABLE MinMaxTemp;
    The DROP TABLE statement entirely removes a table from the data model when the script is executed.
  4. Load your data and check the data model. Make sure the new table MasterCalendar_Temp has been added and preview the data to check there are no missing dates.

Part 3 - Create and validate the master calendar table

Step-by-step guide
  1. Navigate to the MasterCalendar section and locate the MasterCalendar_Temp table script and add a couple carriage returns.
  2. Add a new LOAD statement for the MasterCalendar table by copying the script below and pasting it into the editor. This script creates a MasterCalendar table with the fields from the preceding LOAD:
    //**** Create the Master Calendar **** MasterCalendar: LOAD OrderDate, Week(OrderDate) AS Week, Year(OrderDate) AS Year, Month(OrderDate) AS Month, Day(OrderDate) AS Day, WeekDay(OrderDate) AS WeekDay, Dual('Q' & Ceil(Month(OrderDate) / 3), Ceil(Month(OrderDate) / 3)) AS Quarter, Date(MonthStart(OrderDate),'MMM-YYYY') as MonthYear, Week(OrderDate) & '-' & If(Week(OrderDate)=1 AND Month(OrderDate)=12,Year(OrderDate)+1, If(Match(Week(OrderDate), 52, 53) AND Month(OrderDate)=1, Year(OrderDate)-1, Year(OrderDate))) AS WeekYear, InYearToDate(OrderDate, $(vToday), 0) * -1 AS CYTDFlag, InYearToDate(OrderDate, $(vToday), -1) * -1 as LYTDFlag, If(DayNumberOfYear(OrderDate) <= DayNumberOfYear($(vToday)), 1, 0 ) AS IsInYTD, If(DayNumberOfQuarter(OrderDate) <= DayNumberOfQuarter($(vToday)), 1, 0) AS IsInQTD, If(Day(OrderDate) <= Day($(vToday)), 1, 0) AS IsInMTD, If(Month(OrderDate) = Month($(vToday)), 1, 0) AS IsCurrentMonth, If(Month(AddMonths(OrderDate,1)) = Month($(vToday)), 1, 0) AS IsLastMonth RESIDENT MasterCalendar_Temp;
  3. The final step is to add a couple carriage returns at the end of the new script and drop the MasterCalendar_Temp table, as below:
    4. DROP TABLE MasterCalendar_Temp;
  4. Load your data and use Analyze mode to create a new sheet named Calendar.
  5. Enable edit mode and insert a Table that covers all but the top row of the sheet.
  6. Add the field OrderDate to the table, as a dimension.
  7. Add the OrderRecordCounter field in the table as a measure. Select Sum(OrderRecordCounter) from the aggregation list.
  8. In the table properties, expand the Sum(OrderRecordCounter) measure and rename it to Orders.
  9. Add the dimensions Year, Month, and Day as Filter panes, in the space at the top of the sheet.
  10. Disable the edit mode and make selections in these fields to check that the data filters correctly.
