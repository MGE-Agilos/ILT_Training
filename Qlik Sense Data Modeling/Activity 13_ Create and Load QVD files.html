Data Modeling for Qlik Sense

Activity 13: Create and Load QVD files

Summary:

  • Write data load script to store tables and fields into QVD files.
  • Write data load script to load table data from QVD files.

Module: Applying finishing touches

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, create .qvd files based on the MasterCalendar and Customers tables. Achieve that by completing the following tasks:

  • Create a new script section before LoadComplete and name it ExportToQVD.
    • Qlik Sense on Windows: Create a new data folder connection, QVDFiles, that points to the QVDFiles folder in C:\Qlik Sense Data.
  • Navigate to the ExportToQVD script section and create script that stores the MasterCalendar table in the MCal.qvd.
  • Do the same for the Customers table storing only the CustomerID as CustID, CompanyName as CustName and DivisionName as Division. Name the file Cust.qvd.
  • Load the data and check the destination storage location to verify that two new files have been created.
  • Comment out the entire script in the MasterCalendar section and create a new section right below, named MasterCalendarQVD.
  • Load the MCal.qvd using the data connection.
  • At the end of the same section, load the Cust.qvd file.
  • Load the data again and be prepared to explain why the QVDCustomers table is not QVD optimized.
  • In the ExportToQVD section, create a loop that will export all the loaded tables into .qvd files with the same name.
  • Examine the destination storage location to verify that all the tables have been created.

Hints

  • The STORE command needs to be used to create the .qvd files.
  • A FOR...NEXT loop can be used to export all the tables into .qvd files.

 

Step-by-step guide
  1. Add a new script section before LoadComplete and name it ExportToQVD.
    2. Qlik Sense on Windows: Create a new data folder connection, QVDFiles, that points to the QVDFiles folder in C:\Qlik Sense Data.
  2. In the ExportToQVD section, add the following STORE statements to create the .qvd files:
    STORE MasterCalendar INTO 'lib://DataFiles/MCal.qvd' (qvd); STORE CustomerID AS CustID, CompanyName AS CustName, DivisionName AS Division FROM Customers INTO 'lib://DataFiles/Cust.qvd' (qvd);
    3. Qlik Sense on Windows: Use lib://QVDFiles (qse_student)/ as the connection destination.
  3. Load the data and verify that the two files exist in the specified location. Any subsequent data loads overwrite the .qvd files.
  4. Comment out the STORE statements because the data required is now stored in the .qvd files.
  5. In the MasterCalendar section, comment out all of the script it contains. (Tip: You can use shortcut keys CTRL+A and then CTRL+K).
  6. Create a new section after MasterCalendar and name it MasterCalendarQVD.
  7. Navigate to the MasterCalendarQVD script section and use the data connection to load all the fields from MCal.qvd.
  8. With no table label, the table created from this LOAD statement is automatically named MCal, after the file name. Add the label, MasterCalendar, immediately before the LOAD statement.
  9. At the end of the same section, use the same data connection again to load all the fields from Cust.qvd.
  10. Add the label QVDCustomers. Add the following WHERE clause in the end of the LOAD statement, right before the semi-colon (;):
    WHERE Division = 'EMEA';
  11. Load the data to add the MasterCalendar and QVDCustomer tables from the .qvd files to your data model.
  12. Examine the messages in the Data load progress dialog and consider why the QVDCustomers table is not QVD optimized:
    MasterCalendar << MCal
    (QVD (row-based) optimized)
    Lines fetched: 1,186
    QVDCustomers << Cust
    Lines fetched: 55
  13. Go back to the ExportToQVD section and add the following script:
    FOR i = 0 TO NoOfTables() - 1 LET d = TableName(i); STORE [$(d)] INTO [lib://DataFiles/$(d)].qvd; NEXT;
    14. Qlik Sense on Windows: Use lib://QVDFiles (qse_student)/ as the connection destination.
  14. Load the data and check the destination connection location. Verify that all the tables exist in that storage area as .qvd files.
