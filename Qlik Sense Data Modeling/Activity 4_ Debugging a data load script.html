Data Modeling for Qlik Sense

Activity 4: Debugging a data load script

Summary:

  • Use the debugger to place breakpoints and step through a data load script.
  • Track and monitor variable values as script execution progresses.

Module: Developing and Debugging

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, familiarize yourself with the data load process and debug your script by doing the following:

  • Use the debugger to control your data load progress by doing the following:
    • Add two breakpoints in the Orders section of your script and use the Debug toolbar to control the script execution. The two breakpoints must be added next to the LOAD statement for the Orders table and next to the OrderDetails table name. Observe the load behavior after you execute the script in the debugger.
    • Add breakpoints to the Include statement in the Email section and the For Each statement in the Quota section. Observe the debugger behavior as you run and step through script execution.
    • Use the Variables section to verify that the value of the vWorkSheet variable changes during script execution.

Hints

  • The debugger can be found in the Data load editor.
  • At the end of the activity, all the changes must be reverted back to their initial state.

 

Step-by-step guide
  1. Close any open applications and return to the hub.
  2. Open the ABC Sales Data app and navigate to the Data load editor.
  3. Open the script debugger from the Data load editor toolbar and analyze the Variables section. Underneath All variables, there are currently no variables listed. Until one load is executed, there are no variables initialized.
  4. Use the Run button on the Debug toolbar to execute a data load.
  5. Observe how variables are initialized during script execution within the Variables section.
  6. From the list of script sections, select the Orders section and set two breakpoints by clicking on the line numbers to the left of the script:

    Q on the line with the LOAD for Orders.

    Q on the line with the OrderDetails table label for OrderDetails.

  7. Run the script in debug mode again. Notice how the script execution ignores the first breakpoint and stops at the second breakpoint.
  8. Run the script again, to complete execution to the end of the script.
  9. Use the drop-down in the Breakpoints section and select Delete all to remove both the breakpoints.
  10. Navigate to the Email script section.
  11. Set a breakpoint on the line with the Include statement which loads the employee emails via the external email.txt script.
  12. Navigate to the Quota script section.
  13. Set a breakpoint on the line allowing FOR EACH to start the loop which loads in the Excel sheets.
  14. Use the debug Run button to execute the script in debug mode.
  15. Processing stops at the Include statement breakpoint: Observe how the contents of this external script are displayed within the orange-bordered, statement preview box. Scrolling down the preview box reveals the entire set of statements within the email.txt file.
  16. Use the Step button on the Debug panel to proceed one statement.
    17. Note: In reviewing code, it is easier to step through and analyze script lines using breakpoints.
  17. Observe that while we can see the content of the external script file within the debugger, moving one step only reflects the script we have in the Data load editor. We cannot step through external script, one statement at a time.
  18. Use the debug Run button to continue script processing.
  19. Execution pauses at the next breakpoint, at the start of the FOR EACH loop. We want to monitor the vWorkSheet variable; scroll down the list of All variables and click the Mark as favorite icon ï next to vWorkSheet.
  20. Scroll to the top of the Variables section and note that the vWorkSheet variable now appears within the Favorites area, making it easier to observe as we step through the script. Note that it is currently set to the string Quota 2023. This is due to it being the last value vWorkSheet was set to during the previous script execution.
  21. Press the Step button once. Notice that vWorkSheet has now been freshly-assigned the new value of Quota 2022 for this script execution.
  22. Use Step a couple more times and execution returns to the start of the loop, with vWorkSheet now set to Quota 2023.
  23. Use the debug Run button again to complete script execution.
  24. Use the drop-down in the Breakpoints section again, and Delete all the breakpoints in the script.
  25. Close the debugger to return to the Data load editor.
