Data Modeling for Qlik Sense

Activity 3: Using Sum() vs Count()

Summary:

  • Analyze current record numbers for Orders and OrderDetails.
  • Create visualizations to confirm data model records.
  • Modify script for the Orders table to provide counter fields.

Module: Resolving common modeling issues

Qlik Sense app: ABC Sales Data

Challenge

Count the number of orders in your current data model. Be prepared to explain why using Count is potentially risky and why using Sum is more efficient by doing the following:

  • Verify the number of orders using the Preview panel in the Data model viewer.
  • In the App overview, create a new sheet named Basic data transformations.
  • In sheet edit mode, toggle Advanced options on.
  • Create a Text & image object that contains the text Count(OrderID), and the same as an expression.
  • In the same sheet, create a second Text & image object that contains the text Count(Distinct OrderID), and the same as an expression.
  • In the Orders script section, create a new field in the Orders table: OrderID AS OrderRecordCounter, and reload your data.
  • In the sheet, copy and paste the first object you created for a third Text & image object that contains the text Count(OrderRecordCounter), and the same as an expression.
  • In your script, modify the new field in the Orders table as follows: 1 AS OrderRecordCounter, and reload your data.
  • Modify the text and the expression in the last Text & image object to: Sum(OrderRecordCounter), and observe the results.

Hints

  • Counting key fields in your data model using the Count() function should be avoided.
  • Using Sum() instead of Count() is recommended for counting records.

 

Step-by-step guide
  1. Open your Data model viewer and use the Preview panel to check the number of rows in the Orders table. This is the number of orders you have in your data set. Verify that the number is 6571 rows (orders).
  2. Use the Analyze Sheet option to edit a blank sheet and change the title to Basic data transformations.

Use Count(OrderID)

  1. With Advanced options toggled on, remove any existing objects from the new sheet, then open the Charts in the Assets panel. Drag a Text & image object onto the sheet and resize it to be 16 columns x 1 row. Place it at the top left corner of your sheet.
  2. Enter the text Count(OrderID): in the Text & image object.
  3. From the object Properties window on the right, add a new measure. Use the OrderID field and select Count(OrderID) from the aggregation options. The result of this calculation is 20246. It is different than the result you verified in Step 1.
    Note: Count() is performed over all the possible values of the OrderID field from all the tables that contain it. So in this case, the Count() function counts the records in the OrderDetails table, which is the longest of the two tables that contain the field.

Use Count(Distinct OrderID)

  1. Select the Text & image object you created and make a copy of it on the same sheet.
  2. Resize the new object to match the previous one. Place it under the first object.
  3. Edit the text of the object to modify it as follows: Count(Distinct OrderID):
  4. From the Properties window, expand the measure properties for Count(OrderID) and modify the expression to: Count(Distinct OrderID).
  5. Disable the edit mode in your sheet and check the results. The new expression that contains the Distinct qualifier shows 6571, which is the correct and expected number of rows.
    Note: The Count(Distinct...) syntax will give you the correct result. However, according to best practices, Count(Distinct...) is a heavy function and, therefore, many occurrences should be avoided in the front-end of your app, as it will affect the performance.

Use OrderID AS OrderRecordCounter

  1. In the Data load editor, navigate to the Orders script section and add the following line in the Orders table, under the existing OrderID field:
    OrderID AS OrderRecordCounter,
    This will create a counter field to count the number of orders.
    2. Note: This field is related to orders at an order level, so it is added to the Orders table. It is not at an order-line level, so you do not add it to OrderDetails.
  2. Load your data to add the new field OrderRecordCounter into the data model.
  3. In the Data model viewer, verify that the new field, OrderRecordCounter, exists in the data model within the Orders table. Preview the table to verify that the OrderRecordCounter field contains the exact same values as the OrderID field.
  4. In Analyze mode, open the Basic data transformations sheet and enable the edit mode, if required.
  5. Copy the first Text & image object you added into the field. Resize it to be exactly the same and place it under the second object.
  6. Edit the text in the object and modify it as follows: Count(OrderRecordCounter):
  7. From the Properties window, expand the measure properties for Count(OrderID) and modify the expression to: Count(OrderRecordCounter).
  8. Disable the edit mode in your sheet and check the results. The new expression is 6571 which, again, is the correct and expected number of rows.
    Note: Despite the fact that the Count(...) function is more efficient than the Count(Distinct ...) function (simpler algorithm), whenever possible, you should avoid using Count() on the front-end. The Sum() function, which is much faster, can be used instead.

Use 1 AS OrderRecordCounter

  1. In the Data load editor, navigate to the Orders section. Comment out the existing counter field in the Orders table and create a new counter with 1 instead of OrderID, as seen below:
    // OrderID AS OrderRecordCounter, 1 AS OrderRecordCounter,
  2. Load your data to update the values of the new field, OrderRecordCounter, into the data model.
  3. In the Data model viewer, verify that the field, OrderRecordCounter, exists and now contains the value 1.
  4. From the App overview window, open the Basic data transformations sheet and enable the edit mode again, if required.
  5. Edit the third Text & image object you added into the field, and modify its text as follows: Sum(OrderRecordCounter):
  6. From the Properties window, expand the measure properties for Count(OrderRecordCounter) and modify the expression to: Sum(OrderRecordCounter).
  7. Disable the edit mode in your sheet and check the results. The result of the new expression is 6571 which, again, is the correct and expected number of rows.
    Note: According to best practices, using Sum() is the recommended way to add counter fields into your tables.
