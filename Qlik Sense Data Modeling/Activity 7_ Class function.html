Data Modeling for Qlik Sense

Activity 7: Class function

Summary:

  • Create a new derived field to hold price groups of product UnitPrice.
  • Analyze the new field in a table object.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, use the CLASS function to create intervals based on a field or an expression. You can achieve this by doing the following:

  • In the Products script section, add a CLASS statement to create the PriceGroup field after the CataloguePrice field.
  • The CLASS function creates intervals of 100, called CataloguePrice, using the UnitPrice values.
  • Load the data and preview the Products table in the data model. Verify that the new field has been added.
  • Create a new app sheet named Price Groups and add a Table with the following columns:
    • PriceGroup as the dimension.
    • Count(ProductName) as Products for the measure.
  • Use the REPLACE function to create a shorter dimension label that resembles the following: >= 100 < 200
  • Load the data and navigate to the Price Groups sheet to see the result.

Hints

  • The CLASS function only creates intervals of the same size.
  • CLASS is both a script and a chart function.

 

Step-by-step guide
  1. Select the Products script section and add the following line to the LOAD statement for Products:
    CLASS(UnitPrice, 100, 'CataloguePrice') AS PriceGroup,
  2. Load the data and, in the data model, preview the Products table. A new field for PriceGroup has appeared containing intervals of 100 in the form:
    100 <= CataloguePrice < 200
  3. Navigate to the Analyze mode, and create a new sheet named Price Groups.
  4. In the sheet edit mode, add a Table to the new sheet, spanning 4 rows and 8 columns, with the field PriceGroup as the dimension and the expression Count(ProductName) as the measure. Label the measure Products.
  5. Disable the sheet edit mode and analyze the data. The table shows the price groups and the corresponding numbers of products in each group.
    6. Note: A visible problem here is that the price group labels are long text strings which uses too much space when displayed in some charts. Using the Replace F function will help shorten this label.
    The syntax of the Replace function is the following:
    Replace (string, 'old substring' , 'new substring' )
  6. The dimension label that the original class expression returns is: 100 <= CataloguePrice < 200. To make it shorter, replace the following part of the string '<= CataloguePrice <'.
    In addition, the bucket label 100 – 200 is not quite correct due to the <= and < operators. The bucket, for this example, actually contains values from 100 to 199.99. Use the REPLACE function to change the substring for a < symbol:
    100 < 200
    You can also concatenate using a ‘>=’ substring to clarify that the bucket contains values from 100 inclusive:  
    >= 100 < 200
    Therefore, in the Products script section, edit the script for PriceGroup as follows:
    '>= ' & REPLACE(CLASS(UnitPrice, 100, 'CataloguePrice'), '<= CataloguePrice <', ' < ') AS PriceGroup,
  7. Load the data and review the Products table in the data model. The PriceGroup now contains intervals in the form:
    >= 100 < 200
  8. In the App overview, open the Price Groups sheet and verify that it shows the price groups with their updated and shorter labels.
