Data Modeling for Qlik Sense

Activity 2: Load quota data

Summary:

  • Load quota data from an Excel source.
  • Transform the table with the addition of a Crosstable prefix.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, load a new table as a cross table by doing the following:

  • Examine the Quota.xls data file from the student collateral.
  • Create a new script section after MasterCalendar and name it Quota.
  • In the Quota section, load a new table, Quotas, using the Quota.xls data. Select only one of the sheets, Quota 2022. Make sure that Embedded field names is selected and the Header size is 1 row.
  • In the Quotas table, comment out the SalesPerson field and add a new field:
    '2022' AS QuotaYear,
  • Use the CROSSTABLE prefix to load the table:
    CROSSTABLE(QuotaQuarter, QuotaAmount, 2)
  • Load the data and, in the data model, verify that the table has been loaded correctly.

Hints

  • The quota data is held by employees and quarters for different years. There are two Excel worksheets in the Quota.xls, one per year, with the same table structure in each. The table is built as a cross table using EmployeeID, SalesPerson, and Quarter.

 

Step-by-step guide
  1. Open and examine the Quota.xls file. This Excel file contains two worksheets, one for 2022 quotas and one for 2023 quotas. In each worksheet, there is a title and a table:
    • The title is not considered data. Therefore, do not load the first row of this worksheet in the app.
    • The table is a cross table because the quarter values are across the header. Qlik Sense will identify them as field names. The first two columns are presented as normal columns with the fields EmployeeID and SalesPerson. In this case, SalesPerson is the employee name and is not required as you have EmployeeID that associates with the Employees table.
  2. Add a new script section following MasterCalendar named Quota.
  3. Using the Select data option on the DataFiles connection, upload the Quota.xlsx file from the student collateral, and select it when the upload is complete.
    4. Qlik Sense on Windows: Use Select data on the ExcelFiles connection and select the Quota.xls file.
  4. Select only one of the two tables, Quota 2022, and make sure all columns are selected.
  5. The file contains one header row. Change the Header size to 1. This updates the headers to the appropriate names.
  6. The file also has embedded labels. Ensure that Embedded field names is selected in the Select data window.
  7. Insert the script and comment out the field SalesPerson.
  8. Create a new field below this one to insert the QuotaYear, as follows:
    // SalesPerson,
    '2022' AS QuotaYear,
  9. Convert the table using the CROSSTABLE prefix: Add a new line above the LOAD statement, as follows:
    CROSSTABLE(QuotaQuarter, QuotaAmount, 2)
    In the table, QuotaQuarter is the attribute field containing the Quarter column heading converted. QuotaAmount is the data field containing the quota data converted. The number of qualifier fields is set to 2, in this case, for EmployeeID and QuotaYear.
    Note: The number of qualifier fields is 2 and not 3 because the SalesPerson field has been commented out. The order of the fields in a cross table is important; the qualifier fields are defined as the first fields in the LOAD statement.
  10. Label the new table Quotas. Your script should resemble the following:
    Quotas: CROSSTABLE(QuotaQuarter, QuotaAmount, 2) LOAD EmployeeID, // SalesPerson, '2022' AS QuotaYear, Q1, Q2, Q3, Q4 FROM [lib://<ConnectionName>/Quota.xls] (biff, embedded labels, header is 1 lines, table is [Quota 2022$]);
  11. Load your data and check your data model. Verify that the new table Quotas exists and contains 4 fields.
