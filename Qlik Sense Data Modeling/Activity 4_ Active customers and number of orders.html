Data Modeling for Qlik Sense

Activity 4: Active customers and number of orders

Summary:

  • Add a counter field to the Orders table to track active customers.
  • Create a new table visualization to track number of active customers and orders, by division.

Module: Resolving common modeling issues

Qlik Sense app: ABC Sales

Challenge

Create a new visualization in the ABC Sales Data app to show how many active customers there are and the number of orders, by division. To achieve this, do the following:

  • Create a new field in the Orders table named ActiveCustomerID.
  • Load your data to add it into the data model. The new field contains the CustomerID values.
  • In the Basic data transformations app sheet, insert a Table and add three columns:
    • DivisionName as dimension.
    • Count(Distinct ActiveCustomerID) as measure, with Active Customers as the label.
    • Sum(OrderRecordCounter) as measure, with Orders as the label.

Hints

  • The field ActiveCustomerID refers to the active customers who have placed an order. This is why it is added to the Orders table and not the Customers table.

 

Step-by-step guide
  1. In the Data load editor, navigate to the Orders section and create a new counter field in the Orders table for the number of active customers:
    2. // **** Load Orders table **** [Orders]: LOAD OrderID, // OrderID AS OrderRecordCounter, 1 AS OrderRecordCounter, CustomerID, CustomerID AS ActiveCustomerID, ...
  2. Load your data to add the new field into the data model.
  3. Navigate to the Analyze mode and open the Basic data transformations sheet, if applicable.
  4. Enable the edit mode on the sheet, if required.
  5. From the Charts tab in the Assets panel, drag a Table object onto the sheet, aligned with the Text & image objects.
  6. Use the Fields tab in the Assets panel to add the DivisionName field to the table as a dimension.
  7. Locate and add the ActiveCustomerID field to the table as a measure. From the aggregations list, select the Count(ActiveCustomerID).
  8. From the Properties panel, modify the new expression to Count(Distinct ActiveCustomerID), and change the label to Active Customers.
  9. In the Assets panel, locate and add the OrderRecordCounter field to the table as another measure. In the aggregation list, select Sum(OrderRecordCounter). In the Properties panel, change the label to Orders.
  10. Disable the edit mode to finish editing and verify that the table you just created shows the breakdown of how many Active Customers and Orders there are for each DivisionName.
