Data Modeling for Qlik Sense

Activity 6: Load XML data

Summary:

  • Load data for suppliers from an .xml file.
  • Observe the synthetic table and circular reference caused by the field naming convention clash.

Module: Resolving common modeling issues

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, load a new table from an .xml file by doing the following:

  • Create a new script section, Suppliers, between the Employees and Shipments sections.
  • Use the DataFiles connection to import suppliers.xml into your cloud tenant.
    • Qlik Sense on Windows: Create a new data connection, named XMLFiles, to the XMLFiles folder, under C:\Qlik Sense Data.
  • Load the following tables in the Suppliers section:
    • root/suppliers table from the suppliers.xml file and label the table Suppliers.
    • root/supplier_band table in the same file and label the table SupplierBand.
    • Add comments for both tables.
  • Load your data and notice that a synthetic key and a circular reference appear in your data model. Explain why.

Hints

  • Suppliers and Customers tables have multiple fields in common.

  • When the Select data dialog appears, the .xml file format must be selected.

 

Step-by-step guide
  1. In the Data load editor, create a new section and name it Suppliers. Place Suppliers between the Employees section and the Shipments section.
  2. From the Data connections panel, use Select data under the DataFiles connection.
  3. Sourcing the data from the student file collateral, drag the suppliers.xml file onto the drop zone to import it.
    4. Qlik Sense on Windows: Create a new data folder connection, named XMLFiles, to the XMLFiles folder, under C:\Qlik Sense Data.
  4. Use the data connection to select the suppliers.xml as your data file.
  5. In the Select data dialog, notice that there are only two tables:
    • root/suppliers
    • root/supplier_band
  6. Ensure that all the fields from both tables are checked and insert the script.
  7. Label the new tables Suppliers and SupplierBand, respectively, and add a suitable comment above each table.
  8. Load your data and check your data model in the Data model viewer. Notice that a synthetic key and a circular reference have been created.
  9. Identify the fields that cause the synthetic key by selecting the $Syn 1 Table.

    10. The synthetic key and circular reference are created because the file suppliers.xml has several fields in common with the Customers table loaded from the database. These fields must not be associated with each other. Some of the field names in the Suppliers table must be changed.

    Instead of aliasing each field using the AS keyword or other renaming statements, the simplest solution is to have the table name precede the field name (which is commonly seen in most database utilities). You can easily accomplish this by using a special statement in Qlik Sense that qualifies all the field names with the name of the table when they are loaded. This strategy is used in Activity 7 in this Module.
