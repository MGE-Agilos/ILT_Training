Data Modeling for Qlik Sense

Activity 5: TOTAL versus ALL versus {1}

Summary:

  • Develop a bar chart with DivisionName as the dimension and various LineSalesAmount measures.
  • Analyze differences in the measure columns when using various qualifier prefixes.

Module: Applying finishing touches

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, create a Bar chart to better demonstrate the difference between Total, All and {1} qualifiers by doing the following:

  • Create a new sheet named Qualifiers and add the DivisionName field as a Filter pane on the top of the sheet.
  • Insert a Bar chart with Year as the dimension and Sum(LineSalesAmount) as the measure.
  • Modify the above measure expression to Sum(TOTAL LineSalesAmount) and observe the results.
  • Create a new measure on the same chart with the All qualifier: Sum(ALL LineSalesAmount)
  • Select a value in the DivisionName field and observe the different behavior.
  • Create a third measure using the {1} set analysis identifier: Sum({1} LineSalesAmount)
  • Make selections and observe the differences.

Hints

  • The first measure uses the Total qualifier. This expression takes into account the user selections, but ignores the dimension values.
  • The second measure uses the All qualifier. This expression will ignore both the dimension values and the user selections.
  • The third measure uses the set analysis identifier {1}. This expression aggregates the data by dimension values but ignores the user selections.

 

Step-by-step guide
  1. Using Analyze mode, create a new sheet named Qualifiers.
  2. In the new sheet, clear the selections, if any, and enable the edit mode.
  3. From the Assets panel, drag the DivisionName field to the top row. Make sure you use the Shift key when you drag the field to add it as a Filter pane.
  4. Insert a Bar chart onto the sheet with Year as the dimension and Sum(LineSalesAmount) as the measure.
  5. Label the measure TOTAL Sales and modify the expression as follows:
    Sum(TOTAL LineSalesAmount)

  6. Disable the edit mode. From the DivisionName filter pane, select North America and confirm your selection.

    NOTE: After modifying the expression, you see that all the bars in your bar chart have the same height. This is because the TOTAL qualifier forces the expression to ignore the dimensional values of the chart.
    Selecting a DivisionName value affects the TOTAL Sales expression and changes the height of the bars. However, the dimensional values are still being ignored. As a result, all the bars still have the same height.
  7. Clear your selections and enable the edit mode again for the sheet. Add a new measure to the bar chart, using the LineSalesAmount and the Sum(LineSalesAmount) aggregation option, just like before.
  8. Label the new measure ALL Sales and modify the expression as follows:
    Sum(ALL LineSalesAmount)
  9. Disable the edit mode and from the DivisionName filter pane, select North America again and confirm your selection.
    NOTE: After modifying the expression, you see that all the bars in your bar chart have the same height. This is because the ALL qualifier forces the expression to ignore the dimensional values of the chart.
    Selecting a DivisionName value does not affect the ALL Sales expression. The height of the bars are unchanged even after making selections. This is because the ALL qualifier also ignores the user selections.
  10. Clear your selections and enable the edit mode for a third time. Add a third measure, using the same aggregation, Sum(LineSalesAmount).
  11. Label the third measure {1} Sales and modify the expression as follows:
    Sum({1} LineSalesAmount)
  12. Disable the edit mode. From the DivisionName filter pane, select EMEA and confirm your selection.
    NOTE: After modifying the expression, you see that a new set of bars appears in your bar chart. These bars take into account the dimensional value for each bar, showing correct results.
    Selecting a DivisionName value does not affect the {1} Sales expression. The height of the bars are unchanged even after making selections. This is because the {1} set analysis identifier always disregards the user selections.
  13. Clear all your selections.
