Data Modeling for Qlik Sense

Activity 3a: Data reduction with Section Access (SaaS)

Summary:

  • Create an app-level security layer via Section Access security.
  • Investigate the different levels of row and column-level access for users.

Module: Exploring security and advanced concepts

Qlik Sense app: ABC Sales Data / ABC Sales Data_SA2_<user>

Challenge

Add Section Access with data reduction using the same users as before. To achieve this, complete the following tasks:

  • Examine the DataReductionSA.txt and DataReductionLink.txt files. Update DataReductionSA.txt to use your QlikID tenant email.
  • In the Hub, open the original ABC Sales Data app and add a DivisionName Filter control in the Employees app sheet, then make the sheet public.
  • In the Hub, duplicate the app and rename the new app ABC Sales Data_SA2 and suffix it with _<yourname>
  • At the top of the script sections, add a new section, named Security. Load the DataReductionSA.txt above the SECTION ACCESS area.
  • Add the DataReductionLink.txt after the SECTION APPLICATION statement, on the same section.
  • Load your data. Move the app into the DMQS space.
  • Log on as qlik.frank.roll@protonmail.com, then qlik.ingrid.hendrix@protonmail.com, and then to qlik.bing.martin@protonmail.com. Observe the different access levels. See how the data is reduced for the different users.

Hints

  • Ensure the DataReductionSA.txt ADMIN user correctly reflects.your own Qlik ID email.
  • The first security file contains the SECTION ACCESS table. This table includes a SA_LINK field, used for the reduction of the available rows. It also includes an OMIT column to remove the JobTitle field. This first file is DataReductionSA.txt.
  • The second file is loaded as part of SECTION APPLICATION. It acts as a link table between the SECTION ACCESS table and the data model, reducing the available rows based on DivisionName. This second file is DataReductionLink.txt.

 

Step-by-step guide
  1. From the student collateral, use a text editor to open the DataReductionSA.txt file.
  2. Replace the USER.EMAIL field value for the ADMIN user (your.email@address), with your own Qlik ID login email:
    3. ACCESS, USER.EMAIL, SA_LINK, OMIT ADMIN, your.email@address, *, USER, qlik.joan.callins@protonmail.com, EMEA, USER, qlik.frank.roll@protonmail.com, NA, USER, qlik.ingrid.hendrix@protonmail.com, SA, JobTitle USER, qlik.ingrid.hendrix@protonmail.com, NA, JobTitle USER, qlik.bing.martin@protonmail.com, *,
  3. Save the file.
  4. Examine both the DataReductionSA.txt and DataReductionLink.txt files, then close them.
  5. In Qlik Sense, navigate to the Personal space and make sure you are logged on as your main Qlik ID user.
  6. In the ABC Sales Data app, open the Employees sheet and enable edit mode.
  7. Make space for a new Filter control and add the DivisionName field onto the sheet as the new Filter.
  8. Disable the edit mode and make the Employees sheet public.
  9. Navigate back to the Personal space in the Hub.
  10. Duplicate the ABC Sales Data app and rename this new copy to ABC Sales Data_SA2_ and your name as a suffix.
  11. Open ABC Sales Data_SA2 and navigate to the Data load editor. Create a new script section named Security. Move the new section to the top of the section list.
  12. In the Security section, add the following statement:
    SECTION ACCESS;
  13. Add a few carriage returns, then use the Select data option for the DataFiles connection. Drag the DataReductionSA.txt file onto the dialog to upload it.
  14. Select the file from the dialog box to open it.
  15. Make sure the Embedded field names option is selected and insert the script. This loads information for the access level, user ID, link to second table, and the OMIT fields column. Insert the script.
  16. After this script, add the following line:
    SECTION APPLICATION;
  17. Add a few more carriage returns, then use the Select data option for the DataFiles connection again. Drag the DataReductionLink.txt file onto the dialog to upload it.
  18. Select the file from the dialog box to use it.
  19. Make sure that the Embedded field names option is selected and insert the script. This loads the link table information associating each user to the relevant division they have access to.
  20. Load the data to add the security information and close the app to return to the Personal space in the Hub.
  21. For the new ABC Sales Data_SA2 app, use the More actions option to open the menu, and select Move.
  22. Select the Space: DMQS and proceed with the Move, navigating to the space, as required.
    23. Note: The remaining steps of this activity involve logging in-and-out of Qlik Sense Enterprise SaaS, from a single browser. If a second browser is available, feel free to leverage it, i.e. remain logged in as your main account in one browser, but utilize an additional browser to log in as the test users.
  23. Click the User profile menu to open it, and select Log out.
  24. Sign back into the cloud tenant as Frank Roll, using Qlik.Frank.Roll@protonmail.com as the user ID and Training1 as the password.
  25. Ensure the FR credentials appear as the profile on the Hub navigation bar.
  26. Navigate to the DMQS space, via the Explore option.
  27. Open the ABC Sales Data_SA2 app and select the Employees sheet. Frank has access to all fields, but only data associated to North America.
  28. Return to the Hub and log Frank out.
  29. Log back on as Qlik.Ingrid.Hendrix@protonmail.com with the password Training1. Open the ABC Sales Data_SA2 app again. Select the Employees sheet. Ingrid only has access to North America and South America. She cannot see JobTitle data because this field has been omitted.
  30. Note that the SalesTitle is still available. Consider how you can also omit this field for Ingrid.
  31. What can you see when you log in as Bing Martin, using Qlik.Bing.Martin@protonmail.com as the user ID? Why? (see Solution image)
  32. Use the browser Back button to return to the Hub, and log Bing out.
