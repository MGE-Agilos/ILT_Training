Data connections

Qlik Sense contains various dialogs to simplify the process of creating connections. Data connections:

  • Are created in the Data load editor, the Data manager, and the hub.
  • Utilize a connector to allow the software to attach to the external source.
  • Need to be declared before retrieving data from an external source.
  • Can be shared with other users.
  • Are owned by the user who creates them.

Qlik Sense has a wide variety of connectors that can be used to connect to several common data sources:

  • Folder — connections that define a path for local or network file folders
  • ODBC — database connections
  • Web file — connections used to select data located on a website
Qlik Sense SaaS: Depending on the cloud edition of Qlik Sense that is available, the data connections differ. For a complete list of the available data connections, search for Connecting to data sources F.

In this lesson, a database connector to create a connection to an Azure SQL database will be used. This database is the main source of data for the data model you will create.

Note: During data load script execution, Qlik Sense only maintains one open database connection at a time. If data is required from more than one database, you need to ensure that the correct database connection is open before extracting the data tables from it and then, subsequently, connecting to the next database.

During script execution, a connection script instructs Qlik Sense to connect to the data source, using the credentials and settings in the connection.

Connection script example
LIB CONNECT TO 'ABC (abc_student)';
Note: Deleting a data connection from the Data connections panel, using Qlik Sense Enterprise, deletes it from the server. This can affect all other apps using the same data connection.

More Qlik connectors

Additional connectors are available depending on the product editions. Some of them are mentioned below:

Database

The database connectors use ODBC (Open Database Connectivity) drivers to connect directly to databases such as Apache Hive, IBM DB2, and Oracle without requiring DNS connections.

Search for ODBC Connector Package F.

Salesforce

The Qlik Salesforce Connector enables Salesforce.com account holders to efficiently load their Salesforce data into Qlik Sense.

Search for Salesforce connector F.

SAP

The Qlik Connector for use with SAP Netweaver enables Qlik Sense to load data from a number of different repositories within the SAP architecture: SAP BI/BW, a SAP ERP, and a CRM system.

Search for Qlik Connector for use with SAP NetWeaver F.

REST

The Qlik REST Connector enables Qlik Sense to efficiently load data into a Qlik Sense app from a REST data source. The Qlik REST Connector provides access to JSON, XML, and CSV data through a REST API.

Search for REST F.

Essbase

The Qlik Essbase Connector enables Qlik Sense to efficiently load data into a Qlik Sense application from an Essbase dataset. The Essbase Connector uses an XMLA service to retrieve data from Essbase cubes.

Search for Essbase connector F.

There are other connectors for Qlik Sense that need to be installed separately.

Extracting data from a database

After the database connection is configured, extract the data.

When the data source is a database:

  • Qlik Sense utilizes standard SQL syntax to extract the data.
  • The SQL SELECT statement defines the fields and tables to load.
  • ODBC and OLE DB drivers impose limitations for some connections.
SQL Syntax example
SQL SELECT DISTINCT I.AddressID, A.Name, A.Address, A.PostalCode FROM [Invoice] I, [Address] A
WHERE I.InvoiceType IS NOT NULL ANDI.InvoiceDate >= '2014-01-01' ANDI.AddressID = A.AddressID;

Database drivers limitations

Some of the limitations imposed by the database drivers are:

  • SQL syntax deviations may occur. The SELECT statement is interpreted by the selected driver, so the syntax can vary depending on the connection. For example, some drivers do not accept some types of quotation marks.
  • The AS statement is sometimes not allowed. For example, the aliasname must follow immediately after the fieldname. AS is sometimes compulsory if an aliasname is used.
  • Distinct, As, Where, Group by, Order by, or Union are sometimes not supported.
  • Field names and table names must be bracketed by quotes or square brackets if they contain spaces or special characters.
  • Qlik Sense functions cannot be used within the SQL SELECT statements.

Scripting constructs

The script, which must be written using the Qlik Sense script syntax, is color-coded to make it easy to distinguish the different elements. For example, comments are highlighted in gray, Qlik Sense syntax keywords are highlighted in blue, and so on. Each script line is numbered.

The main script constructs are preceding LOAD, table naming, and indentation.

Preceding Load

The preceding LOAD allows the use of various internal Qlik Sense functions when loading the data and is defined on top of the SQL SELECT statement.

Table naming

  • Qlik Sense sets the table names based on the table names in the original data source.
  • It is best practice to explicitly name tables by manually defining a name above the LOAD statement. The table name must be followed by a colon.
  • Although spaces or special characters can be used in table names (by using square brackets or single-quote delimiters), it is generally considered good practice to avoid doing so.

Visit the Help site to find out more about Table names F and Table labels F.

Script comments

A comment is an annotation added to the data load script for the purpose of making it easier to understand and delineating components more visually. It is part of the script, but is not executed when loading data. Comments are identified in a gray color and italics text in the script.

Qlik Sense allows three comment types:

  • REM preceding a statement comments that statement up to its ending semi-colon.
  • // comments the text following it on a single line, also known as inline comments.
  • /* … */ comments all text between the delimiters.
Note: With lines of script highlighted, you can also use the … Comment / uncomment button on the toolbar or use CTRL+K.

Visit the Help site and search for Scripting in the Data load editor F.

Indentation

To make the script easier to read, you can indent lines of text. The script generated by ± Select data does this for you automatically, but you can adjust indent levels with the À indent and Ã outdent buttons on the text editor toolbar.

Data model viewer

The Data model viewer provides an overview of the internal and source data structures for your app. Access the Data model viewer from the Prepare drop-down menu in the top toolbar.

 

The Data model viewer consists of the following components:

1. Data model viewer toolbar

In addition to the top toolbar options that have previously been discussed, the following tools are available from the Data model viewer toolbar:

t - Collapse all collapses all tables and shows the table names only.

å - Show linked fields reduces the size of all tables to show the table name and all fields that are linked to other tables.

s - Expand all expands all tables to show all fields and table names. This can impact readability with more complex data models.

+ - Internal table view shows the internal data structure, including any synthetic keys (shows how Qlik Sense stores data).

7 - Source table view shows the structure of all data source tables (shows how Qlik Sense reads data).

ì - Layout contains different options for changing the layout. These options are: ì Grid layout, ó Auto layout, and õ Restore layout.

2. Zoom and lock controls
  • The zoom controls allow you to zoom in or zoom out to increase or decrease the size of the tables displayed by using the slider or by clicking on either Y or Z.
  • Clicking the home button ü will restore the zoom level to an original 1:1 aspect. Use lock [ to disable or unlock \ to enable changes in the data model layout.
3. Data model structure

You can select a table by clicking on the table name. When a table is selected:

  • Its caption heading changes to orange to indicate that it is the table with the current focus. Any tables that are immediately joined to the selected table change to a lighter orange. All other tables are displayed in a very light gray.
  • If the data model is not locked \, the table can be dragged to a new position to make the structure more readable.
  • In addition to the automatic display size options provided on the toolbar, the table can be resized by clicking and dragging the bottom right corner of it.

You can also click on an individual field to select it. When you select a field:

  • It is highlighted with an orange background and its table also changes to orange because it is the selected table.
  • Other fields that are linked to the selected field (in various tables) are also highlighted.

Changes you make in the Data model viewer are automatically saved, unless you are using Qlik Sense Desktop, in which case you need to click the Save button.

4. Preview panel

The Preview panel can be displayed by clicking Preview at the bottom left of the window. When it is displayed, it can be hidden by clicking Preview again.

The panel displays a preview of data. This is useful for the Data Architect to help ensure that the type of data and formats are as expected.

When a table is selected, the following information is displayed in this panel:

  • Rows — number of data rows in table
  • Fields — number of fields in table
  • Keys — number of key fields in table
  • Tags — tag types that exist for the table

When a field is selected, the following information is displayed:

  • Density — percentage of rows that contain non-null values. Key fields normally have 100% density.
  • Subset ratio — percentage of distinct values for the field in the current table compared to the distinct values across all occurrences. This is calculated as:
    Present distinct values in table / Total distinct values present.
    If a field exists only in one table, this value will always be 100%. If a field exists in more than one table, and this value is less than 100%, there are other values of this field in tables that are not present in this table.
  • Has duplicates — displays true if the values are repeated in the table.
  • Total distinct values — total number of distinct values that exist for this field, across all tables in the data model.
  • Present distinct values — number of distinct values that exist for this field, in this table only, in the data model.
  • Non-null values — number of rows that are not null.
  • Tags — tag types that exist for the field.

Add as dimension and Add as measure also adds items to the Master Library of items.

Loading file data

If you need to load data from tabular format files (Excel, text files, XML files, etc.), use the Data load editor:

Select data

To select data from the folder connection, use the ± Select data icon on the relevant file data connection, and choose the file that contains the data.

The Select data dialog contains the following options when loading a file:

  • The File format drop-down menu, which allows you to select the format of the file you want to load.
  • The Field names allows you to use the column names embedded in the file.
  • The Header size indicates how many rows at the beginning of the table are not data and should not be loaded.

Each worksheet with data in an Excel file appears in the Select data dialog and you can check several sheets at once.
