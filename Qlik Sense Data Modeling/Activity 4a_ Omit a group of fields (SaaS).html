Data Modeling for Qlik Sense

Activity 4a: Omit a group of fields (SaaS)

Summary:

  • Modify app-level security to omit groups of fields.
  • Analyze sheet filter objects to verify the results.

Module: Exploring security and advanced concepts

Qlik Sense app: ABC Sales Data_SA2 / ABC Sales Data_SA3_<user>

Challenge

In the ABC Sales Data_SA3 app, omit a group of fields by doing the following:

  • Create a copy of the DataReductionSA.txt and rename it to DataReductionSA_OPT.txt.
  • In the new file, replace OMIT with OMIT_GROUP and all the JobTitle instances with OMIT_GROUP1.
  • In the Hub, open the DMQS space and duplicate the ABC Sales Data_SA2 app.
  • Change the name to ABC Sales Data_SA3, suffixed with _<your name>.
  • In the Security script section, modify the SECTION ACCESS LOAD statement to load from the new file.

  • Add a second LOAD statement containing table columns OMIT_GROUP and OMIT, and fields JobTitle, SalesTitle, and EmploymentType, associated in the same group, OMIT_GROUP1.

  • Load your data.
  • Log on as IHX. Observe the fields the user has access to in the Employees sheet.

Hints

  • Ensure the DataReductionSA_OPT.txt ADMIN user correctly reflects.your own Qlik ID email.

 

Step-by-step guide
  1. Examine the DataReductionSA.txt you loaded in the previous activity.
  2. Duplicate this file and rename the copy as DataReductionSA_OPT.txt. Open the file to edit it.
  3. In the file, replace OMIT with OMIT_GROUP. Also, replace all the JobTitle instances with OMIT_GROUP1. Then, Save and close the file.
  4. Return to the Hub and make sure you are logged on as your primary Qlik ID. Go to the DMQS space.
  5. Duplicate the ABC Sales Data_SA2 app which contains your name. Rename the copy that appears in the space to ABC Sales Data_SA3_<yourname>.
  6. Open the new app, navigate to the Data load editor and locate the Security section.
  7. In the Data connections panel, expand the Personal space, if required, and use the Select data option for the DataFiles connection. Drag the DataReductionSA_OPT.txt file onto the dialog to upload it.
  8. Click Cancel to close the dialog box.
  9. Within the script, replace the name of the file DataReductionSA.txt with DataReductionSA_OPT.txt in the first FROM clause.
  10. In the SECTION ACCESS LOAD, replace the OMIT field with OMIT_GROUP.
  11. Between the SECTION ACCESS table and the SECTION APPLICATION statement, add the following script:
    LOAD * INLINE [ OMIT_GROUP, OMIT OMIT_GROUP1, JobTitle OMIT_GROUP1, SalesTitle OMIT_GROUP1, EmploymentType ];
  12. Load the data to add the security information to the app and return to the Hub, displaying the DMQS space
    13. Note: The remaining steps of this activity involve logging in-and-out of Qlik Sense Enterprise SaaS, from a single browser. If a second browser is available, feel free to leverage it, i.e. remain logged in as your main account in one browser, but utilize an additional browser to log in as the test users.
  13. Click the User profile menu to open it, and select Log out.
  14. Sign back into the cloud tenant as Ingrid Hendrix, using Qlik.Ingrid.Hendrix@protonmail.com as the user ID and Training1 as the password.
  15. Ensure the IH credentials appear as the profile on the Hub navigation bar.
  16. Navigate to the DMQS space, via the Explore option, if required.
  17. Click the ABC Sales Data_SA3 app and select the Employees sheet. What can you see? Consider why.
  18. Navigate back to the Hub and log Ingrid out.
