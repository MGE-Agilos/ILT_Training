Data Modeling for Qlik Sense

Activity 1: Causing and resolving a synthetic key

Summary:

  • Load another table of data from the ABC connection.
  • Observe and analyze the resulting synthetic key.
  • Resolve the synthetic key by aliasing a field.

Module: Resolving common modeling issues

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, load the table products in the data model, by doing the following:

  • Navigate to the Products script section of the Data load editor.
  • Use the data connection to load all the fields from the table.
  • In the script, name the new table Products and add a description.
  • Load your data, and identify the synthetic key in your data model.
  • In the Data model viewer, change data model views to see how Qlik Sense reads and stores data.
  • In the Data load editor, resolve the synthetic key by aliasing UnitPrice as CataloguePrice.
  • Reload the script and, using the Data model viewer, confirm there are no more data modeling issues.

Hints

  • The Select data dialog in the data connection can be used to load the table.
  • The Source table view can be used to see how Qlik Sense reads the tables.
  • The Internal table view can be used to see how Qlik Sense stores data internally.
  • The Data load editor should be used to alias fields.

 

Part 1 - Causing a synthetic key

Step-by-step guide
  1. In the Data load editor, navigate to the Products script section.
  2. Use the ABC database connection to load the products table and make sure all the fields are selected.
  3. Relocate the table label [products]: from above the SELECT statement, to above the preceding LOAD statement and capitalize the "P".
  4. Add a comment to describe the table.
    Your script should appear as follows:
    5. // **** Load Products table **** [Products]: LOAD CategoryID, ...
  5. Load your data and check the issues in the load script. Qlik Sense warns of a synthetic key being generated because both ProductID and UnitPrice appear in more than one table.
  6. Open the Data model viewer and expand the tables. The data model shows the $Syn 1 Table that has been introduced.
  7. Select the $Syn 1 field to see the tables associated with the synthetic key.
  8. Select one of the tables with a $Syn 1 field. View the details of that table by expanding the Preview panel. Notice how the synthetic fields are not displayed in the preview of data.
  9. The Data model viewer allows you to view how a data structure looks from the original data source. Click Source table view and note the difference. The source shows two links between the tables, which is not allowed in Qlik Sense.
  10. Switch back to the Internal table view mode.

Progress check

Click on the Progress check image to open as a pop-up, or right-click on the image for more options. To close, click on the image again.

 

Part 2 - Resolving a synthetic key

Step-by-step guide
  1. In the Data load editor, select the Products script section.
  2. Edit the line for the UnitPrice field in the preceding LOAD part of the Products table to alias the field as CataloguePrice using AS. Make sure you use the correct syntax at the end of the line.
    The script should appear as the following:
    3. ... UnitCost, UnitPrice AS CataloguePrice; ...
  3. Load your data and in the Data load progress dialog, confirm that the synthetic key is no longer present in the data model.
  4. Check the data model and verify that the synthetic key has been removed from the model and only the correct association, using ProductID between OrderDetails and Products, remains.
