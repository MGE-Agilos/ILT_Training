Data Modeling for Qlik Sense

Activity 3 - Optional: Load the Bonus table

Summary:

  • Analyze the structure of the Bonus.xls file.
  • Write a data load script to load and transform the Bonus data into a table.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, load the Bonus table from its Excel file by completing the following tasks:

  • Examine the Bonus.xls file. Identify what needs to be done to load the table.
  • At the end of the Quota section, load the Bonus table as a cross table and make sure it is associated with the data model.
  • Use the load parameters and filters option for further transformation of your table. Perform the following:
    • Avoid loading the first row, as it is not part of the data.
    • Fill in the blank cells in the first column with the year value noted above them.
    • Avoid loading the third column, Quarter, as it is blank and does not contain any information.
  • Load the data and check the new table in the data model. Preview the table to verify that the values it contains are correct.

Hints

  • The Bonus table represents the bonus data by year, quarter, and employee.
  • In order for the Bonus table to be associated with the existing data model, the EMPLOYEE_ID must be renamed to EmployeeID.

 

Step-by-step guide
  1. Open and examine the Bonus.xls file and review the table structure. Identify the changes needed to load this table and complete the load:
    • The first row in the Excel file is just a title and not part of the table.
    • This is a cross table. The quarter data is spread across columns.
    • The first column is incomplete. This needs addressing in the data load.
    • The third column is empty and contains no pertinent data for the table.
  2. Using the Select data option on the DataFiles connection, upload the Bonus.xlsx file from the student collateral, and cancel the dialog when the upload is complete.
    3. Qlik Sense on Windows: The ExcelFiles connection already has access to the Bonus.xls file.
  3. In the Data load editor, navigate to the end of the Quota section and add the following script (Do not forget to update the [lib://...] statement.):
    Bonus: CROSSTABLE(BonusQuarter, BonusAmount, 2) LOAD YEAR AS BonusYear, EMPLOYEE_ID AS EmployeeID, Q1, Q2, Q3, Q4 FROM [lib://<ConnectionName>/Bonus.xls] (biff, embedded labels, header is 1 lines, table is [Bonus$], filters(Replace(1, top, StrCnd(null))));
    • The CROSSTABLE prefix combines the quarter data in one single column called BonusQuarter.
    • The EMPLOYEE_ID field has been renamed to EmployeeID to associate the Bonus table with the data model.
    • The last parameter in the FROM clause specifies the filters applied to the data using a filters function. In this case, the only filter applied is Replace. It replaces the first column (1), YEAR, with the previous value (in the row above), top, when the cell is blank or contains a null (...,StrCnd(null)).

    Note: Another way of completing the YEAR field without using the filters function in the FROM clause is the following:

    1. Create the Bonus table, as it is specified above. But, use BonusTemp for the table name and do not rename YEAR.

    2. After the BonusTemp table, create a new table called Bonus. This table is created based on BonusTemp using a Resident Load.

    3. In this new table, complete the YEAR field in a new BonusYear field using the IsNull F and Peek F functions in an If F statement.

    4. Delete the BonusTemp table.

    Using this method, the final script should appear as follows:

    BonusTemp: CROSSTABLE(BonusQuarter, BonusAmount, 2) LOAD YEAR, EMPLOYEE_ID AS EmployeeID, Q1, Q2, Q3, Q4 FROM [lib://<ConnectionName>/Bonus.xls] (biff, embedded labels, header is 1 lines, table is [Bonus$]); Bonus: LOAD EmployeeID, BonusQuarter, BonusAmount, IF(ISNULL(YEAR),PEEK(BonusYear), YEAR) AS BonusYear RESIDENT BonusTemp; DROP TABLE BonusTemp;
  4. Load the data.
  5. In your data model, select the Bonus table and preview its data to verify that the values are correct.
