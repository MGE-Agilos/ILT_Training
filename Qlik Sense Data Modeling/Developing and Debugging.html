Script Variables

Variables are entities that can be assigned any text or numeric value.

Variables:

  • Are created in the script using a SET or a LET statement. SET assigns the text/numeric value and LET evaluates the expression/calculation.
  • Are replaced with their value, when used.
  • Are used in the script, in chart expressions, and in other sheet objects.

Some reserved variables are set by default at the start of the script and contain information to handle formatting, such as time and number values.

Note: Both script and user interface variable names are case-sensitive.

To create a variable in the script:

Set

Use the SET statement for defining script variables, which you can use for substituting strings and reserved variables.

SET y=3+2; // value of y is “3+2”
Let

The LET statement is used to evaluate the expression on the right side of the equals sign (=) before it is assigned to the variable.

LET y=3+2; // value of y is “5”

User interface variables

Variables can be created and used on the front-end of an app.

  • When you use the variable in an app, any change made to the variable is applied everywhere that variable is used.
  • Variables are defined using the Variables overview. In the sheet editor, open the Variables overview by clicking Ò, located at the bottom of the Assets panel. Advanced options should be active for the Variables Ò, to be visible.
  • The Variables overview lists all the possible variables you can use in the expressions for making calculations, based on the data in that app. When used, the variable is substituted by its value.

Reusing script

Dollar-sign expansion

Dollar-sign expansions are definitions of text replacements used in the script or in expressions. The replacement is made just before the script statement or when the expression is evaluated.

When using a variable for text replacement, use the following syntax: 

$(variablename)

$(variablename) expands to the value in the variablename. If variablename does not exist, the expansion is the empty string.

Dollar-sign expansions can be used with:

  • variables
  • parameters
  • expressions

Include files

The Include variable is a user-defined variable that specifies 'a file containing text to be included in the script'. The entire script can thus be put in a file. Include files can be used to:

  • Manage scripts so that a single file can contain sections of script or variable definitions. This can be updated by Data Architects without changing anything in the app.
  • Reuse the same code in several applications, keeping logic and definitions consistent between them.
  • Control the code version when the scripts are stored in external files.

The Include variable can be typed as follows:

$(Include=[lib://<ConnectionName>/filename])

Example:

$(Include=[lib://MyDataFiles/abc.txt]);

If you don’t already have a data connection to the folder where your text file is located, then you need to create one. Use lib:// and your data connection name before the text file reference that contains your script.

Scenarios for using Include statements

Storing code outside the Data load editor can be useful in different scenarios. The following are some examples:

  • Variable definitions sharing between applications. Some examples of these are:
    • Paths to data sources and data connections
    • Color-coding in the user interface
    • Formula/Expression definitions
    • Language Dictionary
    • Default variables for formatting
  • Generic script code that can be used in any app, e.g.:
    • Scripts that store all the tables of any generic app into .qvd files for backup purposes.
    • Scripts that generate app statistics from any application, such as table names and number of records, etc.
  • Procedures and function definitions
  • Security and Access tables

Controlling script execution

The For loop

Qlik Sense accepts code loops in the Data load editor to help with data loading. Using loops allows for a more automated approach when loading data and makes maintenance easier.

The FOR loop is a script iteration that executes the piece of code inside the loop as many times as a condition is met. There are different variations:

  • FOR...NEXT loop uses a counter to compare it with a limit and determine the number of code executions. After each execution, the counter is increased/decreased automatically (based on the step parameter).

    • Example:

    FOR a=1 TO 9 STEP 1
    LOAD * FROM file$(a).csv;
    NEXT

    The above example loads all the fields from all the files with filename file1.csv ...file9.csv. Nine files are loaded in total.

  • FOR EACH...NEXT loop uses a list of values and executes the code for each of the items in the list.

    • Example:

    FOR EACH a IN 1,3,7,'xyz'
    LOAD * FROM file$(a).csv;
    NEXT

    The above example loads all the fields from the following files: file1.csv, file3.csv, file7.csv, filexyz.csv. Four files are loaded in total.

Debugging script

The Debugger environment

Qlik Sense has a Debug panel which:

  • Is used for identifying and isolating errors in the script.
  • Is accessed by clicking Show debug panel on the Data load editor toolbar.
  • Contains a toolbar to choose which panes to display from Output, Variables, and Breakpoints. Once activated, the Debug panel is displayed at the bottom of the Data load editor.

Debugger components:

Debug toolbar

The Debug toolbar provides debugging utilities to aid development:

  • Step through the execution of your script by using breakpoints.
  • Stop execution, continue until the next breakpoint is reached, or step to the next line of code.
  • Inspect variable values and output from the script execution.
  • Limit the load to a specific number of rows per table.
Output

The Output button:

  • Is located in the bottom-left corner of the Data load editor.
  • Displays the last messages from the Data load progress dialog.
  • Toggles to hide and show the Output dialog.

There are two buttons on the right of the Output toolbar.

    \ Scroll lock: Locks the output from scrolling when new messages are displayed.
    ¨ Output menu: Displays options to Clear the text, Select all text so it can be copied, or Scroll to bottom for the last output message.
Variables

Variables list all favorite variables, system variables, reserved variables, and user defined variables in the script, and displays their current values during script execution.

Breakpoints
  • You can use breakpoints in your script to halt debug execution on specific lines of code and inspect variable values and output messages.
  • When you have reached a breakpoint, you can choose to End execution, continue to Run until the next breakpoint is reached, or Step to the next line of code.
  • All breakpoints in the script are listed with a reference to script section and line number.
