Data Modeling for Qlik Sense

Activity 2a: Apply basic Section Access (SaaS)

Summary:

  • Implement a Section Access security to an app.
  • Validate basic security is working by logging in as different users.

Module: Exploring security and advanced concepts

Qlik Sense app: ABC Sales Data / ABC Sales Data_SA1_<user>

Challenge

In the ABC Sales Data app, add SECTION ACCESS to provide extra application level security. To achieve that, use a number of pre-existing users as per the table below:

USERID Name
Qlik.Bing.Martin@protonmail.com Bing Martin
Qlik.Frank.Roll@protonmail.com Frank Roll
Qlik.Ingrid.Hendrix@protonmail.com Ingrid Hendrix
Qlik.Joan.Callins@protonmail.com Joan Callins
Note: The password for all users is Training1.

To add SECTION ACCESS to your app, complete the following tasks:

  • Open and update the file SectionAccess.txt from the student collateral, adding your own email address.
  • Within the Personal space, duplicate the ABC Sales Data app and rename the copy ABC Sales Data_SA1 and suffix it with _<your name>.
  • In the beginning of your script, add a new script section named Security. Add a script to load the above file into the SECTION ACCESS area.
  • Load your data.
  • Move the app to the DMQS space.
  • Login as FRL and then as IHX. Observe the different access levels.

Hints

  • Ensure the SectionAccess.txt ADMIN user correctly reflects your own Qlik ID email.

 

Step-by-step guide
  1. From the student collateral, use a text editor to open the SectionAccess.txt file.
  2. Replace the your.email@address, with your own Qlik ID login email:
    3. ACCESS, USER.EMAIL ADMIN, your.email@address USER, qlik.bing.martin@protonmail.com USER, qlik.frank.roll@protonmail.com
  3. Save and close the file.
  4. Within the Personal space, open the actions menu for the ABC Sales Data app and use the Duplicate option. After a moment, a new app named Copy - ABC Sales Data is created.
Note: It is recommended to Duplicate the app before applying Section Access to avoid making logical errors while developing the security.
  1. Rename this new copy to ABC Sales Data_SA1_ and your name as a suffix.
  2. Open the app and navigate to the Data load editor. Create a new script section named Security. Move the new section to the top of the section list.
  3. In the Security section, add the following statement:
    SECTION ACCESS;
  4. Add a few carriage returns, then use the Select data option for the DataFiles connection. Drag the SectionAccess.txt file onto the dialog to upload it.
  5. Select the file from the dialog box to open it.
  6. Make sure that the Embedded field names option is selected for the table and insert the script. This loads information for the access level and user ID.
  7. After this script, add the following line:
    SECTION APPLICATION;
  8. Load the data to add the security information, then close the Data load progress dialog box.
  9. Return to the Personal space in the hub.
  10. For the new ABC Sales Data_SA1 app, use the actions option to open the menu, and select Move.
  11. Select the Space: DMQS and proceed with the Move, navigating to the space, as required.
    12. Note: The remaining steps of this activity involve logging in-and-out of Qlik Sense Enterprise SaaS, from a single browser. If a second browser is available, feel free to leverage it, i.e. remain logged in as your main account in one browser, but utilize an additional browser to log in as the test users. If you do so, ensure you use the URL provided by the instructor to access the specific cloud tenant you are using for training.
  12. Click the User profile menu to open it, and select Log out.
  13. Sign back into the cloud tenant as Frank Roll, using Qlik.Frank.Roll@protonmail.com as the user ID and Training1 as the password.
  14. Ensure the FR credentials appear as the profile on the Hub navigation bar.
  15. Navigate to the DMQS space, via the Catalog.
  16. Click the ABC Sales Data_SA1 app. Can you open it?
  17. Close the app and log Frank Roll out of the tenant.
  18. Repeat the previous steps to log on as a different user: Qlik.Ingrid.Hendrix@protonmail.com. This user, Ingrid Hendrix, receives an Access is denied message when trying to open the app. She has no user access in the security file.
  19. Use the browser Back button to return to the Hub.
  20. Log Ingrid out of the tenant.
