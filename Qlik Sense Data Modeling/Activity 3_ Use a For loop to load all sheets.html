Data Modeling for Qlik Sense

Activity 3: Use a For loop to load all sheets

Summary:

  • Re-engineer the script to load all sheets of Quota data from Excel.
  • Create and analyze the resulting data in a table.

Module: Developing and Debugging

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, load all the sheets from the Quota.xls file using a FOR EACH...NEXT loop. You can achieve that by doing the following:

  • Create a new app sheet named Quotas and insert a Table with the following dimension fields:
    • Year
    • Quarter
    • EmployeeID
    • Name
    • QuotaAmount
  • Insert a Filter pane from the Year field.
  • When analyzing the table by year, notice that only one year contains quota data. You need to load all the tables from the file.
  • In the Quota section, add a FOR EACH...NEXT loop to load data from the tables in the Excel file.
  • Use the Subfield F function in the Quotas table to only keep the numerical part of the QuotaYear value.

Hints

  • The vWorkSheet variable can be used in the FOR EACH...NEXT loop to navigate and load all the tables from the Excel file.
  • All the existing references to the Excel worksheet name must be replaced with the above variable. The variable must be included in a dollar-sign expansion syntax.

 

Step-by-step guide
  1. Create a new app sheet and name it Quotas.
  2. In sheet edit mode, insert a Table object from the Assets panel that fills all but the first three columns on the sheet.
  3. Add the following fields as dimensions: Year, Quarter, EmployeeID, Name, and QuotaAmount.
  4. Add the Year field as a Filter pane in the top left of the sheet and stop editing. Select different years in the filter while checking the table. Notice there is only Quota data for a single year. You need to load the second Excel worksheet to complete the Quotas table.
  5. Navigate to the Data load editor. In the Quota section, add the following FOR loop before the QuotasTemp table:
    FOR EACH vWorkSheet IN 'Quota 2022','Quota 2023'
    Note: In the FOR loop, define a variable, vWorkSheet. Then, use the dollar-sign expansion to evaluate the value of this variable: $(vWorkSheet).
  6. In the QuotasTemp table, change the table is part of the FROM clause to connect [$(vWorkSheet)$].
    table is [$(vWorkSheet)$]);
  7. Change the QuotaYear field to use the variable vWorkSheet. Using the SUBFIELD function keeps part of the string in the variable.
    SUBFIELD('$(vWorkSheet)', ' ', 2) AS QuotaYear,
  8. Add the NEXT statement after the QuotasTemp table to close the FOR loop. Feel free to indent the inner statements of the FOR loop, to aid readability. Your script in the Quota section looks as follows:
    FOR EACH vWorkSheet IN 'Quota 2022','Quota 2023' QuotasTemp: CROSSTABLE(QuotaQuarter, QuotaAmount, 2) LOAD EmployeeID AS QuotaEmployeeID, // SalesPerson, SUBFIELD('$(vWorkSheet)', ' ', 2) AS QuotaYear, Q1, Q2, Q3, Q4 FROM [$(vPathFiles)Quota.xls] (biff, embedded labels, header is 1 lines, table is [$(vWorkSheet)$]); NEXT;
    9. Qlik Sense on Windows: The variable used on the Windows versions will be $(vPathExcel), as opposed to $(vPathFiles).
  9. Load the data and open the Quotas sheet again to review the results. The table now contains values for years 2022 and 2023.
Note: In the above example, the file and sheet names are known and have been hard-coded in the script.
