Data Modeling for Qlik Sense

Activity 5: Join OrderDetails and Shipments

Summary:

  • Analyze the association between OrderDetails and Shipments.
  • Use the correct JOIN type between the tables to fold Shipments into OrderDetails.

Module: Combining tables

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, combine the OrderDetails and Shipments tables by completing the following:

  • Analyze the OrderDetails and Shipments tables. Review the number of Rows, Density and Subset ratio for the key fields.
  • Create a new app sheet, Shipments, and insert two tables onto the new sheet.
    • The first table contains four dimension columns: OrderLineKey, OrderID, LineNo and ShipmentDate.
    • The second table contains three dimension columns: OrderLineKey, OrderID, and LineNo, and Count(Distinct ShipmentDate) as the measure.
  • In the Shipments script section, locate the Shipments table. Comment out the name of the table and add the JOIN prefix above the LOAD statement: Left Join (OrderDetails).
  • Comment out the WHERE clause from the Shipments table, as it is no longer needed.
  • Load your data. Check the results in the new sheet and in your data model.

Hints

  • The Shipments table only contains a date field (ShipmentDate) and the key field (OrderLineKey). In this case, OrderDetails is the main table, as it is one of the fact tables in the data model. You can keep OrderDetails as the main table and include the information from the Shipments table in it.

 

Step-by-step guide
  1. In your data model, analyze the table information for OrderDetails and Shipments. Similar to the previous activity, review the number of Rows, Fields, information Density, and Subset ratio of the key field.
  2. Create a new app sheet named Shipments and insert a new Table onto it. Expand the Table to cover the top half of the sheet.
  3. Add the field OrderLineKey as the first dimension. Use the Properties panel to add three more dimensions into your table: OrderID, LineNo, and ShipmentDate.
  4. Insert a second Table onto the bottom half of the sheet.
  5. Add the field OrderLineKey as the first dimension. Add two more dimensions: OrderID and LineNo.
  6. Use the Properties panel again to add the Count(ShipmentDate) as the measure.
  7. Modify the measure expression and add the Distinct qualifier before the ShipmentDate field as below:
    Count(Distinct ShipmentDate)
  8. Sort the table by the Count(Distinct ShipmentDate) measure.
  9. Disable the edit mode. Notice that there is only one ShipmentDate maximum per each OrderLineKey based on the count above. You also see that not all the OrderLineKey values have a ShipmentDate. Scroll down to verify that.
  10. There is no risk of duplicating OrderDetails records if you join the Shipments table to them. In the Data load editor, navigate to the Shipments script section and locate the Shipments table. Note that this table is loaded after the OrderDetails table. The order of the tables is relevant for the type of JOIN you use for the LEFT and RIGHT JOIN.
  11. What type of JOIN are you going to apply? In this case, the main table is OrderDetails and you only want to add the ShipmentDate field to it. Analyze the results using the different types of JOIN:
    • INNER JOIN removes all the OrderLineKey values with no ShipmentDate, and only leaves the common OrderLineKey values in both tables. This is not the desired result.
    • RIGHT JOIN uses the second table (Shipments table) as the main table, and removes those OrderLineKey values with no shipment dates. This is not the desired result.
    • LEFT JOIN uses the first table (OrderDetails table) as the main table, and removes those OrderLineKey values in the Shipments table that do not exist in the OrderDetails table. This is what you need.
    • OUTER JOIN/JOIN keeps all the records from both tables. This type of JOIN works, but the WHERE clause in Shipments is required.
    12. Comment out the [Shipments]: table label and add a LEFT JOIN in front of the LOAD statement for this table.
    //Shipments:
    LEFT JOIN (OrderDetails)
    LOAD
    ...
  12. The WHERE clause in the Shipments table is no longer required due to the nature of the LEFT JOIN. Comment out the WHERE clause in the Shipments table. Make sure you have a semi-colon at the end of the statement.
    ...
    //WHERE EXISTS(OrderID, OrderID)
    ;
  13. Load the data and observe the changes in the data model. One more table, Shipments, has been removed and its fields have been added to OrderDetails.
