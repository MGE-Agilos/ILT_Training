Data Modeling for Qlik Sense

Activity 2: Where clause in Orders table

Summary:

  • Alter the script which loads the Orders table to restrict the data extracted, based on the order date.
  • Analyze the resulting year values within the sheet objects.

Module: Generating data

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, use a condition to determine specific records for loading from the source by doing the following:

  • In the Orders table, add a WHERE clause to load records of specific date range.
  • In the OrderDetails table, add a WHERE EXISTS clause to only load records related to the above amount of years.
  • Load your data and check the Year values in the Basic data transformations sheet.

Hints

  • The WHERE clause in both cases is added at the end of the preceding LOAD statement.

 

Step-by-step guide
  1. In the Data load editor, navigate to the Orders section.
  2. In the Orders table only, limit the number of orders till the latest date: Add a WHERE clause in the LOAD statement of the Orders table, before the semicolon:
    3. WHERE OrderDate < NOW();
    Note 1: One can argue that only the existing orders would be displayed and not any future order, even without the WHERE clause. But, by design, this is training material that has included future order dates. Hence, for capping the date range, the WHERE clause is used.
  3. In the OrderDetails table, there is no OrderDate field. So, in the OrderDetails table, add a WHERE EXISTS clause with OrderID in the LOAD statement, before the semicolon, to limit the order records that exist in the Orders table. This will retrieve OrderDetails records that match the loaded date-restricted Orders:
    4. WHERE EXISTS(OrderID, OrderID);
  4. Be careful with punctuation. The final fields in the preceding LOAD lists should not have a comma or semicolon after them and lead straight into the WHERE statements.
    5. Note 2:The first OrderID is the existing field in the data model. This field exists in the Orders table. The second OrderID is the new field being loading from the OrderDetails table. During the load, every value of the OrderID field in the OrderDetails table, is cross-checked against all the values in the existing OrderID field in the Orders table. If the same value exists in both fields, the function (Boolean) returns True and the record is imported into the OrderDetails table. If the same OrderID value does not exist, the record is omitted.
    Note 3: The load order is important and matters when using the Where Exists clause. In this case, the Orders table needs to be loaded before the OrderDetails table in the script for this Where clause to work.
    Note 4: The Where Exists clause in this case uses two parameters. Although this is best practice, one parameter can be used instead.
  5. Load your data and navigate to the Basic data transformations sheet.
  6. To more easily verify that the table now contains the data of the specified date range, edit the sheet and select the Table object to open up the Properties panel. Within the Data section, change the DivisionName field, to use the Year field instead.
  7. Exit edit mode and evaluate the data table.
