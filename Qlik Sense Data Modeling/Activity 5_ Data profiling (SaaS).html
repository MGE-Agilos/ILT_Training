Data Modeling for Qlik Sense

Activity 5: Data profiling (SaaS)

Summary:

  • Use built-in tools to profile and analyze data files.
  • Verify data files and apps are in-sync.
  • Add metadata to data files.

Module: Exploring security and advanced concepts

Qlik Sense app: ABC Sales Data

Challenge

Use the tools available in the Hub to profile data and apply metadata properties:

  • Analyze the Employees.qvd file, using the Profile data option.
  • Verify aggregate calculations.
  • Add and remove metadata properties to the file.
  • Review the data lineage of the Employees table.
  • Analyze the OrderDetails.qvd file, using the Profile data option.
  • Ensure a statistical match between the profile results and the ABC Sales Data app.

Hints

  • Employees.qvd and OrderDetails.qvd are generated by running the data load script of the ABC Sales Data app.

 

Step-by-step guide
  1. Ensure you are logged in to Qlik Cloud with your primary Qlik ID.
  2. Navigate to the Personal space and select the Data from the Types filter.
  3. Select the Employees.qvd file to open it. The page updates to display the Overview details of the data file.
  4. Use the profile refresh icon, next to the button labeled Create analytics app, to ensure the profile data is up-to-date.
  5. Observe and analyze the headline information for the table the file contains.
  6. Click View fields to expand the display to show all fields in the table.
  7. Locate the field of Team and observe that the four most frequent values are: Sales, IT,Marketing, and Delivery.
  8. Click on the tile for the DateOfBirth field to display an additional Details panel.
    9. Note: If the fields are not displayed in Tile view, use the view controls on the right to change it.
  9. Click the check box for Personal Information to mark the DateOfBirth field as containing such, then close the panel.
  10. Navigate from the Fields tab to the Overview tab to return to the headline profile information for the Employees.qvd file.
  11. Observe in the summary information at the top; the Metadata modified indicator has updated to reflect the recent change.
  12. Click the More actions ellipsis button in the top right and select Lineage.
  13. Move the view around by clicking and dragging, or zooming in and out as required to display the full lineage of this table. Observe that there were sources of data from Qlik Data Files (Personal) loaded into the ABC Sales Data app, which subsequently generated this downstream Employees.qvd file, residing within the same File storage of the Personal space.
  14. Use the drop-down arrow on the File storage box on the left, to expand the sources.
  15. Verify that there were at least four data file sources which were ingested by the ABC Sales Data app, in order to store the Employees.qvd file.
  16. To return to the Analytics Services area, click on the Qlik logo in the top left.
  17. As before, navigate to the Personal space and select the Data from the Type filter.
  18. Select the OrderDetails.qvd file to open it.
  19. Make sure you are viewing the Overview information.
  20. Click View fields.
  21. Ensure the view selector is set to List view.
  22. Use the Column picker button in the top right of the table, and ensure the Sum aggregation is selected.
  23. Check the Sum calculation for the LineSalesAmount field, and make a note of it.
  24. Use the Qlik logo again to return to the Analytics Services and navigate back to the Personal space.
  25. Locate the open the ABC Sales Data app.
  26. Navigate to the Bonus sheet.
  27. Ensure no selections are currently applied, and verify that the total Sales figure matches up with the figure being recently exported to the OrderDetails.qvd.
