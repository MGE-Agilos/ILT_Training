Data Modeling for Qlik Sense

Activity 4: Add the Sales vs. Quota sheet

Summary:

  • Create a new sheet to analyze Sales and Quotas measures.
  • Identify issues between the known source data and the data as the model presents it.

Module: Handling advanced modeling challenges

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, create a new sheet to show the Sales vs. Quota trend by Quarter and Year and observe the wrong results by completing the following tasks:

  • In the data model, examine the tables Quotas and Orders.
  • Create a new app sheet named Sales vs. Quota and add a Line chart with:
    • Year as the dimension.
    • Sum(LineSalesAmount) as Sales, for the measure.
    • Sum(QuotaAmount) as Quota, for the second measure.
  • Add the fields Year, Quarter, QuotaYear, QuotaQuarter, Name, and EmployeeID as Filter panes.
  • Analyze the Sales vs. Quota sheet and make your observations.

Hints

  • The issues identified in this activity make a Link table solution necessary.

 

Step-by-step guide
  1. In the data model, analyze the fields in the Quotas and Orders tables.
    Note: The Quotas table contains the QuotaAmount per EmployeeID over the QuotaYear and QuotaQuarter. The EmployeeID field is the key field in this table and relates the table with the Orders table. The Orders table is associated with the OrderDetails table that contains the LineSalesAmount field used to calculate the Sales measure.
    The Orders table also contains the OrderDate field that associates this table with the MasterCalendar and the Year and Quarter fields.
  2. Within Analyze mode, create a new sheet and name it Sales vs. Quota.
  3. In sheet edit mode, insert a Line chart to fill the top half of the sheet.
  4. Add the Year field as the dimension and the Sum(LineSalesAmount) as the measure. Name the new measure Sales.
  5. From Fields in the Assets Panel, add a second measure by dragging the QuotaAmount onto the chart. Select Sum(QuotaAmount) for the aggregation and name the new measure Quota.
  6. In the free space of the sheet, add the following fields as Filter panes: Year, Quarter, QuotaYear, QuotaQuarter, Name, and EmployeeID.
  7. Disable the edit mode and analyze the sheet.
    Note that the line chart is not correct:
    • The line representing the Quota. is displayed across all the years in the chart, but remember the Quotas table just has two QuotaYear values.
    • There is a QuotaAmount with no year (null value) in the chart.
    • Select Bob Park in the Name field and note that the line representing the Quota goes flat across the timeline.
    Note: The problem lies in the associative data model. The Quota table associates with the rest of the tables in the data model using only EmployeeID. The Year field, used as a dimension in the chart, comes from the MasterCalendar table that is associated with the rest of the data model using the Orders table by the OrderDate field.
    Based on this, the red line representing the Quota is:
    • Summing up the quotas over all the quota years, per each year, in the chart. The difference between years is due to the different employees' sales activities.
    • Displaying a total quota amount, not associated with any year (null value) in the chart dimension. There are employees with Quota, but no Sales, like Erik Fredriksen.
  8. Clear your selections, if there are any.
  9. Consider potential solutions.
