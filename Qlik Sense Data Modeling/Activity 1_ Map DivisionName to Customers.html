Data Modeling for Qlik Sense

Activity 1: Map DivisionName to Customers

Summary:

  • Re-engineer the Divisions table as a mapping table.
  • Apply the Divisions mapping table to the Customers table to fold the Divisions dimension into Customers.

Module: Combining tables

Qlik Sense app: ABC Sales Data

Challenge

In the ABC Sales Data app, apply a MAPPING LOAD to the Divisions table. Move the DivisionName field into the Customers table by doing the following:

  • Create a new script section after Main and name it MappingTables.
  • Cut the Divisions LOAD statement and paste it into the MappingTables section.
  • Change the name to Divisions_Map and add the MAPPING prefix in front of the LOAD statement.
  • Use the ApplyMap function in the Customers table and comment out the DivisionID field, as it is no longer needed.
  • Load your data and verify that the Divisions_Map does not exist in the data model and the DivisionName field has been added to the Customers table.

Hints

  • The MAPPING LOAD technique eliminates the need for a separate Divisions table and the DivisionID field. It reduces the number of tables in your data model by one and denormalizes your schema.
  • It is always recommended to verify that the values in the new field, added with the ApplyMap function, are correct.

 

Step-by-step guide
  1. Create a new section immediately after Main. Name it MappingTables.
  2. Position the cursor at the top of the MappingTables section and use the Insert Connection String option on the ABC data connection, to insert a LIB CONNECT statement.
  3. From the Customers section, cut the entire LOAD statement for the Divisions table and paste it beneath the LIB CONNECT on the MappingTables section.
  4. In the MappingTables section, change the table name from [Divisions]: to [Divisions_Map]:
  5. Add the prefix MAPPING in front of the LOAD statement in the Divisions_Map table. The new script resembles the following:
    // **** Load Divisions table **** [Divisions_Map]: MAPPING LOAD DivisionID, DivisionName; SELECT DivisionID, DivisionName FROM "ABC_2025".dbo.divisions;
  6. In the Customers section, add this script to the Customers preceding LOAD statement, beneath the DivisionID line, to generate a new field within the table:
    APPLYMAP('Divisions_Map', DivisionID) AS DivisionName,
  7. Comment out DivisionID in the Customers table, as it is no longer needed.
  8. Load your data and observe the differences in your data model: There is no longer a Divisions table, the DivisionID is gone, and the DivisionName field now exists in the Customers table.
  9. Preview the data for Customers to check that DivisionName has been mapped correctly.
