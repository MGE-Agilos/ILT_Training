Adding data

There are different ways to add data to a Qlik Sense app:

  • Data manager — Available from the Prepare section. This option allows users to load and transform data without learning a scripting language.
  • Data load editor — Available from the Prepare section. This option launches the Script editor that allows more technical users to load and transform data using load statements and script syntax.
    • Note: The tables that have been loaded using this method are called scripted tables. Initially these tables cannot be edited in the same way as the tables added using the Add data option, but you can Synchronize scripted tables using Data manager to unlock all the tools found within the Data manager.

When you create a new app, you have different options:

  • Files and other sources, so you can select the field you want to load from those files, and transform them using the Data manager, ideally for users with little or no experience with a scripting language. Using this option, you can also enter data manually with Manual entry, or add data from a data connection.
  • Data catalog, that allows you to load data from files available in the spaces which you have access to or loaded by you previously.

    • Note: When using these two options, you can define whether the Data profiling feature is on or off when selecting the table fields. Use the menu ¥ at the bottom right of the screen.

    Note: This option is only available in Qlik Cloud editions. If you are working with Qlik Sense Client-managed versions, use one of the other options to load data to your app.

  • Open the Script editor, to create the data load syntax to load the data and connect to the data sources.

When loading data into the app using either method, you can also determine what tables, columns (fields), and rows (applying filters) are loaded into the app. Also, you can apply filters to tables to limit the records being loaded in the app using conditions.

Managing data

You can use the Data manager F to add and manage data from your own data sources using a graphical user interface, so that you can use it in your app without learning a scripting language.

Use Data manager to:

  • Get an overview of all data tables in the app
  • Add, edit, and delete data tables
  • Manage data table associations
  • Load data and apply filters to the tables
  • Review data source and change any existing filters
  • Manage automatically concatenated tables (tables that contain the same structure)
  • Concatenate tables that do not have the same structure (field names)
  • Automatically generate fields for geo data and dates
  • Associate tables and data based on recommended associations
  • View data profiling cards that include a preview of the field values and their distribution
  • Enter data manually by typing the data into a dialog (Adding data manually F)
  • Unpivot a cross table
  • Create calculated fields and transform the existing fields in tables, some of the field operations include:
    • Replacing field values F
    • Splitting a field F
    • Customizing the order of dimension values F
    • Grouping measure data into ranges F
    • Setting field values as NULL F
    • Changing field types F

All of these operations can be performed sequentially in tables and fields. This allows you to use the data profiling cards, concatenation, calculated fields, and unpivoting data together in the same table.

You can view the operations and transformations performed in tables and fields in the Data manager using the View details dialog. Select the table and then select the menu ¥ at the bottom to view table and field transformation details in Data manager F.

Data review

After data has been loaded in the app, you can review the data tables and the associations using the Data model viewer F available under the Prepare section.

  • It shows all the existing tables loaded into the app and how they are associated.
  • It also allows you to preview the tables and create master dimensions and measures.

Data associations

When loading tables with the Data manager, you can also manage the associations between tables.

  • It is important to associate your data in order to make all of the data set react to your selections when performing the data analysis.
  • Data manager provides some help associating the data from different tables based on a data analysis that is performed while the data is loaded.

The Associations view in Data manager can aid you in:

  • Managing data associations F
  • Concatenating tables in Data manager F - See also this video
  • Synchronizing scripted tables F

There are some limitations where the association recommendations are not provided:

  • Many-to-many relationships

  • Field pairs with data that do not match well in both directions

  • Compound key associations

You can also manage associations in Data manager from the Tables view (Associating data in the Table editor F).

Defining the business logic

Business logic is an optional part of data preparation that defines the logical model used by some Insight Advisor features to interpret your data model. Business logic comprises two main features:

  • Logical model: defines the data model of the app used when generating visualizations.
  • Vocabulary: includes alternative terms for natural language queries.

Insight Advisor uses business logic in the following Insight Advisor features:

  • Insight Advisor Search

  • Insight Advisor Analysis Types

  • Insight Advisor Chat

  • Associative insights

Logical model

Logical models are conceptual models that can help Qlik understand the data based on the business requirements and how the users will look at and query it.

Each app has a single logical model that the Insight Advisor uses to understand data queries when creating suggested visualizations in searches or generating insights.

The logical model in an app:

  • Is built from the underlying data model of an app.

  • Influences how Insight Advisor functions and how the system chooses between fields in natural language queries.

  • Has fields and master items as core components that can be organized into groups, representing conceptual associations or relationships between fields or master items.

Operations with groups include:

  • Packages F define collections of groups that can be use together.
  • Hierarchies F define drill-down analytical relationships between groups.
  • Behaviors F define preferred or deny relationships between fields.
  • Calendar periods F create default periods of analysis used by the Insight Advisor.

You can customize the default Logical model F under Business logic in the Prepare tab. Qlik Sense provides an overview of the model allowing you to modify definitions in fields, groups, and the group operations.

Vocabulary

Qlik allows you to define the business user vocabulary in the app using terms and values that are not present in the data. This improves the natural language queries success as Qlik can understand better what users are asking for. Additionally, you can define the response for specific terms and control the analyses returned by Insight Advisor.

You can create vocabularies in the Vocabulary F section under Business logic in the Prepare tab.

Vocabularies are created by language for an app. Each language's vocabulary is separate. The Insight Advisor supports multiple different languages. Please search Adding custom analyses to Insight Advisor F in the Help site for the current list.

Vocabularies are not supported in associative insights.

This feature supports three kinds of vocabulary:

  • Synonyms: allow you to add alternate terms.

    In the Synonyms table, each entry contains:

    • Terms— A list of names or values that do not exist in the data model but can be used by business users when performing queries.
    • Field — The field in the data model that can be associated with those terms.
    • Condition — (optional) Specify the condition in the data when the terms will apply to the field. The availability of conditions for fields varies depending on the values in the fields.
    • Values — (optional) The data values that are part of the condition.

  • Custom analysis: allow you to define what analysis should be used with select terms.

    When you create one, you add terms to the analysis and then select the analysis type. You can then specify default dimensions or measures. Optionally, you can set constraints on your analysis type, to limit the scope.

    You can optionally choose the chart type to use in your analysis from those available to your analysis type. If you select a chart type, the custom analysis will always use that chart type unless it determines another chart is more appropriate for the selected data.

  • Example questions: give you the ability to define default questions that will appear under the search bar in the Insight Advisor, as suggestions for users.

    When you register a question, you will see the result preview before saving it.

    After adding it, users will see the example question under the Insight Advisor Search bar.

    New example questions become available in Insight Advisor Chat after the app reloads.

Benefits of using Business logic in the app

The benefits of customizing Business logic in an app are:

  • Improve Insight Advisor functionality with responses more aligned to the business.
  • Allow Qlik to understand business terms or common acronyms or abbreviations users might use when searching.
  • Use your own custom periods for analysis with calendar periods.
  • Predefine behaviors when generating insights or searching based on selections or fields.
  • Guide and help users' search in the app data.

Reusable items and object library

After the data is loaded in the app, you can define frequently used fields and calculations and store them as master dimensions and measures. There are also other objects that can be created and be reused in the app, such app charts or visualizations and selection states.

  • These dimensions and measures can be used in several visualizations and objects across the app but they are maintained or modified from one single entry point, in the Master items area.
  • Master items and sheet assets are listed in the app library, also known as Assets library. In edit mode, the Assets panel displays these assets on the left.

Master items

Reusable fields, calculations, and visualizations can be set or defined in the app. They are called Master items and allow the users to apply global changes to visualizations, dimensions, and measures in one app.

Once these items are created, they can later be used as building blocks by anyone who has access to the app.

The items can be:

  • Dimensions F
  • Measures F
  • Visualizations F
  • Alternate states F

The benefits of using Master items are:

  • They provide centralized maintenance.
  • They can be reused multiple times in an app, while maintaining them in just one place.
  • When updating them, changes are applied everywhere they are used.
  • They allow the analysis of data over relative time ranges (calendar measures).
  • They create other selection states, apart from the default state, to analyze the data and compare different subsets together.

Asset panel library

When editing an app, access the Assets panel. The Assets panel is considered a library of items which allow the users to further develop their apps.

Note: You may need to use the Advanced options toggle to access the Assets panel in Edit mode.

Some of the library items are:

  • Fields — A list of data fields available in the app. Users can also reload or refresh the data from this list.
  • Master items — Reusable dimensions, measures, and visualizations created in the app.
  • Charts — The list of standard chart types and objects available in Qlik Sense.
  • Custom objects — A list of different custom extensions and standard extension bundles that you can use in Qlik Sense to create custom visualizations and functionality.

Search and select items in the Assets panel F. From this work space, create, delete, or modify items when the app is not published. When it is published, library items are available for business users, but they cannot be modified.
