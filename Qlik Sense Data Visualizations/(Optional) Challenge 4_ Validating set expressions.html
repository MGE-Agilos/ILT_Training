(Optional) Challenge 4: Validating set expressions

Summary:

  • Understand set analysis syntax and be able to fix incorrect syntax.

Module: Controlling data selections

Qlik Sense app: ABC Sales Dashboard v1

Challenge

Review the expressions using set analysis, identify possible errors, and suggest corrections.

Note that these expressions are using the data fields and values from the ABC Sales Dashboard v1 app.

Note: There is a hidden sheet in the app with these formulas. Set the vSetAnalysis variable to 1 to make the sheet visible. Then use the button to show or hide the solution.

Hints

  • Review the table below and check whether the set analysis syntax in these expressions could be used in the app.
  • Consider the answers to these questions as they relate to each expression:
  • Is the expression correct if it is used in the app? If not, could you edit it so it is a correct expression for the app?
  • Please be aware that not all the expressions have syntax errors.
ID Expression
1 Sum({1<country={USA}>} Sales)
2 Count({<Year={2021}, Month=>}Distinct OrderID)
3 Avg({$<OrderDate.autoCalendar.InYTD={1}>} Sales)
4 Count(${<Division={America}> distinct Customer)
5 Sum({$<Division=EMEA, APAC>} Sales)
6 Sum({<Month={"<3"}>} Margin)
7 Sum({$<Product category={“Footwear”}+{“Baby”}>} Sales)
8 $<categoryName={"swimwear"},Country={Austria}> Sum(margin)
