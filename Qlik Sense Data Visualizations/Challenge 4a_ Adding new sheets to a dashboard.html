Create Visualizations with Qlik Sense

Challenge 4a: Adding new sheets to a dashboard

Summary:

  • Explore other Qlik Sense visualizations.
  • Create sheets and sheet objects in a Qlik Sense app to create a well designed and structured app.
  • Work with the Properties panel to customize visualizations.

Module: Creating visual analytics

Qlik Sense app: ABC Sales Analysis v1

Challenge

Create two new sheets focused on product analysis in the ABC Sales Analysis v1 app and move them after to the dashboard sheet.

Name the sheets: Product overview and Product analysis and include the following information:

  • Include filters: Year, Quarter, Division, Product Category, and Product
  • Add meaningful descriptions for the visualizations using titles, subtitles, and footnotes

Product overview should contain these visualizations:

  • KPI - Num Orders and Num Orders CYTD (as secondary measure)
  • KPI - Number of distinct Product
  • Histograms - based on UnitPrice and UnitCost defining a bar width of 100 in each histograms
  • Product Ranking bar chart - showing Product by Sales with alternative measures: COGS, Margin%, Num Orders and include these in the tooltip
    • Color-code the visualization based on Product Category and sort the bars descending based on the selected measure
    • Highlight with a reference line the most expensive product: Runner Shoes
  • Pie chart (Donut) - Sales shared and YTD comparison - top 10 product categories or products based on Sales, with Sales CYTD - LYTD as second measure (for the radius of the segments)
  • Treemap - Sales by Product category and Product limiting the number of products to the top 10 in each category and displaying the values in the blocks

The Product analysis sheet should include:

  • Scatter plot Product correlation - where each product is represented with a bubble (with the bubble size based on Sales, the bubble color based on Discount, and the bubble location in the X and Y axes based on two measures that the user can select between the following: Margin%, Num Orders, COGS, Discount, Avg Price).
    • Highlight with a regression line the dependency relation between Num Orders and COGS.
  • Distribution plot Product distribution by category - where the distribution is based on Margin% and the user can change this measure to any of the following alternative measures: Num Orders, Sales, COGS, Discount. Use Jitter points if the points overlap.

Use the solution images as a reference to include all the titles and extra information.

Note: Measures and data shown depend on today's date. If you reload your app data, your results might differ from the solution image results.

Hints

  • Drag and drop the visualizations and objects onto the sheet using the Assets panel > Charts section. Then configure each visualization using the properties panel on the right.
  • Alternative dimensions and measures can be added in some visualizations using the properties panel.
  • The filter pane object can contain several fields and allows the user to filter the data. By resizing the filter panel to one row at the top of the sheet, the sheet can contain more visualizations.
  • You can add titles, subtitles, and footnotes to a visualization using the properties panel (under Appearance > General).
  • You can include dimension reference lines and regression lines using the Add-ons section in the properties panel.
  • For the Avg Price measure, use average function with the UnitPrice field: Avg(UnitPrice).
Note: The objective of this challenge is to test whether you feel comfortable with the concepts that have been discussed so far in your class. There is no Step-by-Step section in this activity, so you must only rely on the Hints below to understand the tasks you have to execute.
As you are still new to many of these concepts, please don't hesitate to reach out for more guidance (help file, instructor, Qlik Community).
