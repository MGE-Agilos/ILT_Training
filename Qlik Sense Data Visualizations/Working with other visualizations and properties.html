Maps, properties and operations

The location of the data can be a key point of knowledge to gain insights. Maps can visually represent data based on their related geo data or location.

These are some examples of representation that can achieved using maps in Qlik Sense:

Players' statistics

Product parts

Seating types

Country or City locations

Floor plans

Healthcare

Qlik Sense maps

In Qlik Sense, the , Map chart allows you to visualize location data which can be represented by different layers.

When you add data with Data manager, Qlik automatically recognizes geodata (fields containing cities, countries, postal codes, etc.) and creates new fields containing the geographical information required to visualize the maps automatically (geo points and area data). These fields are named <data field>_GeoInfo and can be used in maps to visualize data. You can also define and load your own locations or areas if you are creating custom maps or you are manually writing data load script with the Data load editor.

As shown in the examples above, custom maps can be created in Qlik defining locations or areas and using images as background layer.

Area layer

  • Used to display geographical areas, such as countries or regions. Areas can be defined in KML format.
  • Can differentiate measure values with color.
  • Does not need a background map, but can be complemented with one, if required.

Point layer

  • Used to mark places of interest, using longitude and latitude coordinates.
  • Each location is represented as a point (including different symbols or images).
  • Shows measure values by changing point size or color.
  • Background is used for context. Interactive background maps can be applied to allow zoom.
  • Several Point layers can be combined together on the same map. They can also be combined with an Area layer map.

Line layer

  • Represents transit or flow from a source point to a destination point using two geo fields as dimensions.
  • Line width can be defined based on a measure. You can also define the line curve and add directional arrows.
  • Line color can also be set based on a dimension, measure, or expression.

Density layer

  • Used to visualize the density of points in an area using a color ramp.
  • Each point influences a circular area, with the highest influence at the center and declining towards its outer perimeter.
  • Density layers can be used to show hotspots of activity, population densities, and more.

Chart layer

  • Displays small pie charts or bar charts over locations in your map.

  • One dimension is used to identify the locations in the map and a second dimension to create the mini charts.

Background layer

  • Displays the background image in your map. This image can be customized to represent any area, not specifically a map. Examples: building floor plans, structures defined within areas where location is important, image representation of objects or parts of the body, etc.

  • You can overlay several background layers if the images or the base maps allow transparency.

You can choose a background map and then add layers (you can combine several ones) to visualize map data.

See more info about Maps F in the help site.

Map properties

In the Map Properties (missing or bad snippet), you can:

  • Choose a background map.
  • Add layers. The layers in the map can be hidden or shown based on conditions and you can disable selections in specific layers.
  • Select a field for each layer, if the field is not a geo field, you can specify the location separately. Geo fields are marked with ,.
  • Choose the point symbol for each point layer - specific layers have different settings.
  • Color the areas, points, density, charts, or lines in the map based on a measure, dimension, or expression.
  • Resize the points or the line widths in the map based on a measure.
  • Define map settings like map language, activate navigation, activate location finder, etc.
  • Include addtional information, measures, charts or images in tooltips in each layer.

End users are able to toggle layers, define size objects, and change other map settings from the Exploration menu.

Other visualizations and extensions

Extend the visualization capabilities of Qlik Sense with custom visualizations and objects, also known as extensions, that can be included as add-ons in the product.

The concept of extensions, in general, can also be seen as a plugin mechanism which allows developers to combine the power of Qlik Sense APIs with the nearly unlimited capabilities of the web. These extensions:

  • Extend the standard set of visualizations.
  • Create new ways to visualize data and enhance user input and interaction.
  • Integrate with other systems and visual libraries, such as D3, add new selection objects and productivity components.
  • Can be developed with standard web technologies (HTML5, JavaScript, and CSS).
  • Once installed, are visible from the Assets panel, under the Custom objects section, and can be used as standard Qlik objects.

Qlik Sense extensions

Although you can install the extensions of your choice, Qlik provides some extensions already installed and ready to use. You can find them under the Custom objects > Extensions section.

Includes: Reload Button and Traffic-Light.

Qlik Sense extension bundles

Qlik also provides two bundles of extensions that allow to classify them into two main purposes.

  • These are automatically available in Qlik Sense cloud editions and are also installed with Qlik Sense Desktop and Qlik Sense Enterprise on Windows (opting out is possible).
  • These bundles are fully supported, develop and verified by Qlik.

The two Qlik extension bundles available are:

  • Visualization bundle F - Enhances and increases your Qlik Sense app's charting capacity.

    Includes: Funnel chart, Multi KPI, Network chart, P&L pivot, Radar chart, Trellis container, Sankey chart, Variance Waterfall, and Word cloud chart.

    Funnel chart

    Similar to a vertical bar chart with descending sorting.

    Four rendering modes are supported for this chart: Area, Height, Width, and Ordering.

    Grid chart

    Allows "at-a-glance" comparisons of complex data using symbols of varying size and color sorted in a grid.

    • The dimension values are the x and y axes.
    • The measure is the metric that determines the size of the symbol in each crossing.

    Multi KPI

    Extended version of the basic KPI object. This extension offers customizations such as:

    • Font styling with size, position, and color.
    • Multiple KPIs with dimensions or measures.
    • Embedded mini chart with master visualization.
    • Styling options with CSS and icons.

    Network chart

    This chart represents links and nodes. It is used to represent hierarchies.

    The following dimensions are expected:

    • Node ID - unique numeric ID for each node, the ID is sequential and starts with 0

    • Node label - node name or description
    • Parent ID - refers to the Node ID of the parent node in the hierarchy
    • Node group

     

    Org chart

    This extension provides a way to visualize hierarchies in your data.

    In this chart, each entity of the hierarchy is represented by a card that can be customized with more information and color background.

    A plus sign (+) in a card means that there are more cards under that card that are not visible. Once the card is opened, the cards under that entity are shown and the plus sign turns into a minus sign (-).

    P&L pivot

    Profit and Loss table with font and color customization suitable for financial reporting.

    Radar chart

    Radar charts let you compare and measure data with an arbitrary number of variables.

    Radar charts require two dimensions and one measure.

    They are helpful when comparing features or performance over several metrics.

     

    Sankey chart

    This is a type of flow chart, where the width of the arrows is shown proportionally to the flow quantity.

    Straight table

    This table has a wide variety of possibilities that differ from the normal table object: many fields can be added simultaneously, it is possible to customize the table at the column-level, add alternative dimensions and measures, set column width, apply pagination and turn on chart exploration. This last feature allows users select dynamically the fields they want to see at any moment.

    Trellis container

    Container object that displays multiple versions of a visualization based on a master visualization and up to two dimensions.

    Variance waterfall

    This extension is a type of waterfall chart that provides variance analysis (the difference) between two measures across the different values of a dimension.

    It is used primarily in financial Actual vs Forecast analysis.

    You can make selections of data on this chart unlink the standard waterfall chart.

    Word cloud chart

    This extension is a visual representation of text data.

    The tags are usually single words shown with different font size and color based on a measure.

     

  • Dashboard bundle F - Enhances navigation and selection in your Qlik Sense app.

    Includes: Animator, Date picker, NL Insights, Variable input, and Video player.

    Animator

    This control adds a button and progress slider to the sheet. When pressed, animator cycles through the values of its dimension.

    Date picker

    It lets you select a single date or a range of dates from a calendar.

    NL Insights

    It displays natural language insights and information about selected dimensions and measures using the Insight Advisor feature. It is updated when new selections are made in the app.

    Variable input

    This allows user to set the value of a variable and get a more dynamic app. Can be presented in different formats: slider, input box, drop down and buttons.

    Video player

    It gives users he ability to play videos in different formats from external resources. It supports MPAR, OGG, WEBM ansd YouTube.

    • Qlik Sense Client-Managed: There is an additional extension in this bundle, the On-Demand reporting control, that creates a Qlik NPrinting report using the current selections in the app. It is possible that you don't see this extension, as it is only available for users that meet the requirements.

Although these objects can be used as other standard Qlik Sense visualizations, there are some limitations. The objects are fully functional but are somewhat limited compared to native charts (Limitations of visualization extension bundles supplied by Qlik F).

Visual aspects: color, order, and shape

Visual aspects in data representation such as color properties, order, and shape representation can help to:

  • Focus user attention, highlighting specific data points.
  • Make it easier to identify and interpret and analyze data.
  • Represent complex data or provide guidance using well-known signs or symbols.
  • Provide additional information that can complement the data in the visualization and provide more context.

These aspects can be customized and defined in visualizations in the Qlik Sense app using the Properties panel:

Color

Using color is a great technique to focus the user's attention to key information.

 

Most visualizations can include color properties based on dimension or measure values.

  • Apply dynamic color on a visualization by dragging and dropping any field, dimension, or measure, from the Assets Panel panel onto the visualization.
  • Alternatively, you can use the option Color by expression or by measure in the Properties panel to:
    • Decide how to color the measure values.
    • Define what colors to use.
    • Define background and text color, as well as the coloring for different measures.
    • Accentuate or differentiate values.

In the colors section, selecting the Expression is a color code option allows the use of an expression to define a color. If this option is cleared, the expression evaluates to a number, which in turn is plotted against one of the chart gradients. Examples of supported color expression formats:

  • RGB function — rgb(255,0,0)
  • Pre-defined color function — magenta()
  • Hexadecimal — ‘#0000ff’ (quotations required)
  • Color keyword — ‘cornflowerblue’ (quotations required)

See more info about Coloring a visualization F in the help site.

Dimension and measure colors

Master measures and master dimensions can be set with specific colors when editing or creating the master items.

This feature is great for keeping the color consistent across the app:

  • For master dimension, define a color for each dimension value using the Value colors tab (see this video for more information about this feature). See Creating a master dimension from a field F.
  • For master measures, specify a color segment range defined for the measure using the Segment colors tab. See Assigning colors to master measure values F.

The color settings will be stored in the color library:

  • When you use a master measure in a visualization (KPI, gauge, etc.), the color you set for the measure will be used by default in the visualization, unless you change the default color settings.
  • When you use the master dimension in a visualization (chart), the default colors you set will be used.
  • When using the Color by dimension property in a visualization, you will be able to use the colors specified for the values in the library.

Order

When working with data, sorting is a common method used for visualizing data into a form that makes it easier to comprehend it. Sorting can be done with raw data or at an aggregated level (in a table, chart, or some other aggregated or summarized output).

  • Sorting items in a particular order is another way to focus the user's attention.

  • You can order dimensions by an expression in the Sorting section of the properties panel. This will help you compare and identify the items you want to highlight more quickly.
  • Sorting by a different measure can have some meaning in the right context.

    Example:

    These two tables are displaying the same information (Country and Population), but the left one is sorted by sales YTD descending and the other is sort by Population.

    Question 1: Which country has the highest population? Which one has the lowest population?

    In the left table, the countries are not sorted by population and it is difficult to answer the question. In this case, it is better to use the the table on the right.

    Question 2: Which country is generating the largest sales YTD? Provide country name and population.

    We are displaying the dual info country-population, but in this case, we want to sort the table based on sales amount descending. So we can answer the question: Canada, with a population of 36M.

However, in some cases, visualizations should not be sorted by measure, but instead, by the logical order of the dimension values (examples: ranking scales, ages, date related fields, time related fields).

Shapes

Shapes and symbols are used everywhere and we recognize them as icons or signs.

Here are some examples:

Shapes and icons in Technology and Software:

Traffic icons and symbols:

The use of these icons and other visual cues is beneficial for:

  • Helping the user perceive complex information quickly and accurately.
  • Helping users to communicate globally. For example, triangle is universally understood as a decrease or increase, depending on where the triangle is pointed.
  • Colorblind users that cannot differentiate colors but can interpret shapes.

  • Using shapes and icons in reduced spaces can provide space for navigation menus or tool menus, without taking up unnecessary space. A typical example is how the tool menu view changes in a mobile app or a small screen. For example, regarding the button to edit the sheet, only the pen icon is displayed when we view our app in a mobile application.

It is possible to add shapes to the following Qlik objects:

  • Image and text objects - a dynamic image (or shape) can be configure in this object.
  • Tables - a trend indicator, a shape or an image can be included per each measure in the table.
  • KPI objects - a shape can be included when segments are defined for the measure.
  • Maps - the area layer in a map shows continent, country, or other defined shapes. The point layer in a map allows representation by shapes and images.
  • Buttons - a symbol can be included in a button object.
  • Stories - you can add shapes in the slides.

Additional properties

Apart from the basic objects properties, additional options can be highlighted, as they can enrich the existing visualizations created to get a more advanced version of our sheets:

Measure modifiers

In some charts it is possible to perform advanced calculation modifiers over an existing measure in the chart or table.

These modifiers change the calculation in the measure to transform it into an advanced expression. Some of these modifiers include:

  • Accumulation over a specific number of periods or the full range
  • Moving average over the specific number of values or over the full range
  • Difference versus previous period
  • Relative numbers as percentage of a total
  • Time series decomposition

    • Qlik Sense Client-Managed: Times series decomposition modifier is not available in this product version.

Apply calculation modifiers to measures in charts or tables by expanding the measure in the Data section of the chart or table's properties panel and selecting Modifier. You can choose the modifier calculation to be applied and the setting for that modifier.

See more info about these modifiers in the help site (Modifiers F).

Trend lines and Reference lines

In some charts it is possible to display trend lines and reference lines. These lines provide more insights and information about the data.

  • Trend lines are a set of predefined functions by Qlik Sense (Average, Linear, Polynomial, Exponential, Logarithmic, etc.) that perform the operation based on a measure used in the visualization. Use these lines in Bar charts or Line charts to:
    • Smooth out fluctuations in data
    • Show any trends more clearly

    • Compare different types of trends

  • Reference lines are lines indicating a certain level in the chart. These lines:
    • Are defined using values or calculations created by the business analyst
    • Can represent custom boundaries, targets, limits, etc.
    • Can be applied to a wide range of visualizations: Bar charts, Line charts, Combo charts, Box Plots, Distribution plots, Scatter plots, Waterfall charts, Gauges, etc.

Other related topics:

  • Trend lines F
  • Reference lines F
Line forecast

This option provides the ability to estimate how the historical sequence of observations will continue into the future in a line chart, what is really good in terms of decision-making.

  • Depending on the estimation method employed, a shaded area representing the confidence interval, displays possible values with a configurable level of confidence.

  • Predicted values are clearly differentiated from historical values.

  • The Forecast toggle only appears in the measure properties if:

    • The dimension is numerical or time based.

    • There is only one dimension and one measure.

    • The dimension axis is continuous.

    • Qlik Sense Client-Managed: Times series forecasting option is not available in this product version.

Machine learning features are not covered in this course. However, in case you are interested on go deeper inside the data predictions and pattern findings with Qlik Sense, please ask the instructor for more info about AutoML training.

Custom tootilps

Tooltips add extra information to an object and are displayed when you hover over a visualization.

They can apply to most of the charts, but also to KPI objects and maps.

  • A tooltip can contain measures, a chart or an image.

  • If you want to include a char, note that must be saved as master visualization.

  • Images can be added from the media library or by using a URL.

Other related topics:

  • Creating custom tooltips F
Background color or image

It is possible to configure the background of a sheet or a visualization in order to customize it with a color or an image. Image formats supported are .png, .jpg, .jpeg, and .gif.

  • This option can be found under Styling > General, both in sheets and objects.

  • Color can be set as single color or by expression.

  • If you use an image, this must be saved in the Media library previously.

  • It is important to adapt the size of the image to the sheet, using the size dropdown and the Position options.

Dynamic calculations and conditions

In a Qlik Sense app, business users can analyze data in different ways and it is very important to provide the relevant insights based on the user actions.

Defining dynamic behaviors or displaying different views based on conditions or calculations can guide business users through their analysis.

Guiding users in their analysis can be achieved by applying some of the following actions:

  • Providing specific context using dynamic text in sheet names, titles, column names in tables, button labels, etc.
  • Making them focus on specific visualizations by displaying or hiding them or by limiting calculations based on conditions.
  • Getting their attention on specific data points using dynamic color in visualizations, buttons, KPI segments, etc.

These actions can be based on the result of a dynamic calculation that also takes in consideration user selections or user actions.

Dynamic calculations or expressions:

  • Are not limited only to visualizations. There are many other places in the app where you can add expressions (identified by the 3 symbol next to the edit field).
  • Can use Qlik functions to calculate values in the data (by default, based on the current selection state defined by the user).

Examples:

  • Dynamic label expressions
  • Dynamic sheet names
  • Dynamic text in text & image objects
  • Dynamic text, URLs, measures and charts in tooltips
  • KPI objects color segment limits
  • Color functions
  • Calculation condition to evaluate a chart or table

  • Calculation condition to enable or disable a button

  • Calculation conditions in map layers
  • Condition message (in a calculation condition to evaluate a chart)
  • Show condition in sheets
  • Show condition in charts and containers
  • Show condition in tables to show or hide specific columns
  • Show condition in reference lines and dimension reference lines
  • Variable definitions

In a Qlik app, use the following features to define dynamic behaviors:

Calculation conditions

Providing many insights in one single sheet can also distract user's attention. For performance reasons, user focus attention, or providing specific context at the right time, we can use calculation conditions.

Calculation conditions can:

  • Determine if a chart or a table is calculated
  • Determine if a map layer is calculated
  • Enable or disable buttons

These conditions are useful for:

  • Calculations that require a user to select a value first.
  • Performance improvements in charts which display a large amount of data.
  • Enabling or disabling buttons to perform actions or navigation.
  • Hiding or showing layers in maps to show more or less detail.

You can add a calculation condition to most objects (Filter panes and Text & image objects do not support calculation conditions).

You have the ability to control the message that a user sees when a calculation condition hasn't been met.

It is recommended to use aggregation functions when defining the calculation condition. See more details in Use Aggregation functions (Qlik Community).

Show conditions

In some scenarios, there are visualizations or sheets that should not be shown until specific conditions are met.

A show condition can be used in each sheet or visualization added to a container and determines if a sheet, chart or a table is visible or not. This way, you use the same space in the sheet and rotate between two or more visualizations depending on user selections or other actions, using a container and show conditions. You can also keep sheets hidden for specific users or show them only in specific scenarios of the analysis.

This is useful when you want to show different visualizations or sheets based on:

  • Which user is accessing
  • The value of a variable
  • The possible number of values in a field
  • The selections performed by the user

Using variables

Variables are reusable items across the app that control different items in the app and simulate behaviors with dynamic calculations.

They can contain plain text or dynamic expressions or calculations.

Variable values can be defined or changed from:

  • Actions and triggers in sheets or buttons
  • Variable input extension (available in the Qlik Dashboard bundle)
  • Selections in the app (if the variable is using dynamic calculations in its definition)

In the Variables overview F panel, you can create, edit, and delete variables in an unpublished app.

  • When using a variable in an app, any change made to the variable is applied everywhere that variable is used.
  • Variables are defined using the variables overview, or in the script using the Data load editor.
  • Open the Variables overview by clicking Ò at the bottom of the Assets panel when editing a sheet.

Here are some tips when using variables:

  • If the variable contains a calculation use the dollar-sign expansion F, $(variable), to evaluate the variable.
  • You can define variables in an external file to share between other apps. Load the file and work with variables in the Data load editor F.
  • When a variable contains an equal sign in front of the expression, it will be evaluated globally with every selection.
  • There are a variety of other system variables F available for use.
  • Variable state can be saved when creating bookmarksF

    • Qlik Sense Client-Managed: Toggle for saving variable states in bookmarks is not available.
