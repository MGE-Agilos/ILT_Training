Create Visualizations with Qlik Sense

Activity 3: Using properties to define visual aspects

Summary:

  • Work with the visualizations properties.
  • Define specific colors for values and segments.
  • Use sorting and shape features in objects.
  • Apply background options.

Module: Working with other visualizations and properties

Qlik Sense app: ABC Sales Analysis v2

Challenge

In the ABC Sales Analysis v2 app, provide more information and context, and give more insights by including visual aspects in these sheets:

ABC Dashboard

Use color to highlight the comparison between current and last year sales:

  • Highlight the Sales CYTD - LYTD bar chart by coloring the bars red if they are below 0 or green if they are above 0.
  • Use the Sales CYTD - LYTD master measure to set the segment colors.

Product overview

Use a silent legend based on the Product category dimension, and apply it to the relevant visualizations in the sheet.

  • Set the Value colors for the values of the Product category master dimension. Use a different color for each value, but choose similar colors for these category items: Footwear (all values containing footwear), Clothes (all values containing clothes), and Sports (Swimwear and Sportswear).

  • Apply this color library based on Product category to the visualizations in the sheet using this dimension: donut, treemap, and bar chart.

Display each product photo in the tooltip in both bar chart and treemap. The product png images can be found in this location:

https://qt-lss.s3.amazonaws.com/Curriculum/Product-Photos/

The corresponding png file for each product is labeled the same as the product name. If the product name contains spaces, the image file will have the spaces replaced with a "+" symbol. Example: For the product called Bow tie, the image file is called Bow+tie.png

Customer Sales

Use the Customer sales details table to highlight discount rates above the 7.2% threshold with color and compare sales for the current year versus previous year with shape and color indicators, and the picture of each product per row. Include also a chart that allow the user to see the weight of each region in Sales by Customer Type.

  • Values below that threshold must display on a green background, values above that threshold must be displayed on a red background.
  • Add an indicator in this table for the Sales CYTD - LYTD column. The indicator will show up and down arrows (▲ and ▼) in green and red colors, respectively, to represent the Sales CYTD - LYTD variance.
  • Use the photo of each product per row in the table object.
  • Add a master visualization to show sales by each region in the pie chart.

Sales analysis

Include information for the Avg deal size* key measure and guide users on the app selections using buttons:

  • Add an Avg deal size* KPI and use color and shapes to highlight the value in red with a warning symbol when it is under 3.5k.

    Link the KPI with the Customer orders sheet, so by clicking on this KPI, the user can navigate to this sheet.

    Use a title ("Avg Deal Size") and footnote ("Sales / Order volume").

    Display information so users can see the info icon when hovering over the object.

  • Sort the Customer values in the combo chart and the pivot table based on the Avg Deal Size* in descending order.
  • Create two buttons to allow guided selections:
    • The Americas button to toggle selections in the Division field, selecting or clearing selections for the Americas regions (North America and South America). You can use the * wildcard: (*America).
    • The Current Year H1 button to select the current year and the H1 period. In this case use a search: '(>=01/01/2023<=06/30/2023)'.

*The Avg deal size is calculated as the sales amount divided by the number of orders.

  • Insert a custom background to the pivot table using a color and an image.

Hints

  • For the ABC Dashboard sheet, set Segment colors for the master measure called Sales CYTD - LYTD and use the Library colors in the chart properties when using this measure to color the chart.
  • For the Product overview sheet,
    • Master items can be modified when a sheet is open in edit mode. From the Assets panel, you can select the master item you want to modify and use the edit @ button. The color for the measure or dimension can be set from the edit dialog.
    • The Value colors tab is available from the edit dialog of the master dimensions. The Colors and legend section in the Properties panel allows you to set the colors in a chart.
  • For the Customer Sales sheet,
    •  The Background color expression property defines the color of the column in a table. The Discount column already exists in the table but an expression is required to define the color expression. Colors can be defined with the RGB() function. For instance, a red color can be represented as: RGB(170,57,57) and a green color as: RGB(87,149,50). You can modify these colors by altering the 3 parameters in the function.

    • Change colors dynamically, based on a comparison, by using an IF statement. Using a condition as the first parameter determines what expression is evaluated (the first expression is returned if the condition is met and the second one is returned if it is not met). The syntax is the following: IF(condition, expression 1, expression 2).

    • Use an indicator representation in the Sales CYTD - LYTD column. Using a limit (0) to show positive numbers in green with the ▲ symbol, or negative numbers in red with the ▼ symbol. Other Unicode character sets can be found in the following Qlik community post.
    • The Representation option inside each column of the table allows us to show the values as images.
    • Master visualizations can be shown when hover over the charts, using the Tooltip option under the Appearance section.
  • For the Sales Analysis sheet,
    • The Avg deal size has not been created as a master measure. It is possible to add a measure in a visualization using the expression editor. Remember to add a label for the expression.
    • Use Conditional colors and define the segments, adding a limit, to customize color and shape.
    • KPIs can be linked with existing sheets in the app using the Link to sheet feature in the properties panel.

    • In the properties panel, use the Sorting option to sort the dimension by the Sales expression.

    • Use two buttons with a toggle field selection action in the corresponding fields: Division and OrderDate.
    • Background options can be found under Presentation > Styling, both in charts and sheets.
Step-by-step guide
  1. Open the ABC Sales Analysis v2 app and navigate to the Analyze > Sheet area.

    2. ABC Dashboard sheet

  2. Go to the ABC Dashboard sheet and turn the edit mode on.
  3. Using the Assets panel, select Master items > Measures > Sales CYTD - LYTD. Then click Edit@ and OK.
  4. Select the Segment colors tab and set the Segment colors to Custom. Then set the template scheme to Basic, the Segment format to Fix value, and set the default limit to 0.
  5. Press the first segment and set the color to red. Then for the second segment, set the color to green and save the settings.

  6. Select the Sales CYTD - LYTD bar chart in the sheet to review the chart properties. Under Appearance > Colors and Legend, set the Library colors to On and clear the Reverse colors.
  7. In analysis mode, you can make selections in this chart to review that the color result in the bars is the expected one, negative values are red bars and positive values are green bars.

    8. Product overview

  8. Clear selections and open the Product Overview sheet in Edit mode.
  9. From the Assets panel, under Master items > Dimensions, select the Product category master dimension and click the edit @ button in the information pop-up.
  10. Select the Value colors tab from the edit dialog of the master dimension. Then apply different colors to each value as follows:
    • Click on the corresponding color for that value
    • Use the color palette to select a different color

    You can also apply a default color for others and null values.

    You can "group" the different categories using similar colors.

  11. Modify the bar chart, pie chart, and treemap properties: under Appearance > Colors and Legends, set Colors to Custom By Dimension, select Product category, and make sure Library colors is on.

  12. In the bar chart and treemap properties, go to Appearance > Tooltip and set it to Custom. Then under Images, select URL as Type and enter the following expression as URL:

    ='https://qt-lss.s3.amazonaws.com/Curriculum/Product-Photos/' & Replace(Product,' ','+') & '.png'

    13. Customer Sales

  13. Navigate to the Customer Sales sheet and select the Customer sales details table.

  14. In the Properties panel, under the Discount column, set the Background color expression using the following expression:

    If (Avg(Discount)>=0.072, RGB(170,57,57), RGB(87,149,50))

  15. For the Sales CYTD - LYTD column in the table, set the Representation to Indicator. Then add a limit, set it to 0, and set the color and shapes to the limit. Use the ▲ and ▼ icons, to represent negative or positive values.
  16. Set the Indication Position to Right and check the Apply color to value option.
  17. In analysis mode, sort the table by Customer by double clicking on the first column label:
    Note: Once you sort the table or resize columns in analysis mode, the properties of the table change. If you want to edit the properties of the table again (in edit mode), you need to Apply (or Discard) the changes you applied in analysis mode.
  18. Include Product as a new dimension. For this column, set the Representation to Image, and Image setting as Add URL. In the Image URL enter the following expression:

    ='https://qt-lss.s3.amazonaws.com/Curriculum/Product-Photos/' & Replace(Product,' ','+') & '.png'

    .

    19. Sales Analysis

  19. Open the Sales Analysis sheet in edit mode, reduce the size of the container, and add a KPI in a free space on the sheet.
  20. Use the properties panel to add the measure using the following expression:

    [Sales]/[Num Orders]
    .

    Then label the measure as Avg deal size and click Add new, under Master item, to add it as master measure in the app. To save the new master measure, click Create.
  21. In the KPI properties panel, under Appearance > Color, set the Conditional colors to On and the Library colors to Off.
  22. Add a limit of 3500. For the left segment, set the color to red and choose the è warning symbol. For the right segment, set the color to green and choose the m tick symbol. Check the Gradient box.

  23. Under Appearance > Presentation, set Link to sheet to On. Then use the drop-down and select the Customer Orders sheet.
  24. Under Appearance > General, set Title to Avg deal size* and Footnote to *Sales / Order volume. Then set Show details to Show.
  25. Under Appearance > Presentation, clear Show first KPI title.
  26. Let's sort the customers based on this measure. In the existing container, access the combo chart properties. Then, in the properties panel of the combo chart, under Sorting, make sure Customer is the first field on the list.
  27. Set the Sorting to Custom, Sort by expression for this dimension using the Expression:

    [Avg deal size]

    and select Descending. Note that you can add a subtitle in both charts to explain the sort order to the end users.
  28. Repeat the same steps for the pivot table.
  29. From the Assets panel, under Charts, drag and drop a Button into the sheet. Then select it to configure the settings in the properties panel as follows:
    1. Add an action, selecting Toggle field selection as Action, label this action as Americas, use Division as Field, and the following expression as Value:

      '*America'

    2. Under Appearance > General, change the Label to Americas.
    3. Under Appearance > Background, change the color background, then under the Icon property, set Show Icon to On and add a Map icon from the drop-down list.

    Repeat this step again to add a new button, this time using the OrderDate field as Field in the action and the '(>=01/01/2023<=06/30/2023)' as the Value. Label this button Current Year H1 and apply the History icon.

  30. In analysis mode, select EMEA, 2023, using the pivot table (to select EMEA) and the Sales and Sales difference chart in the container (to select the year). Notice how the KPI number changes. Clear selections, click on the Americas button and the Current Year H1 button. Then click the Americas button again to clear the selections in the Division field.
  31. Hover over the KPI and click the ] info button. An info dialog appears with the details about the expression used for this calculation.
  32. Close the Details panel and click on the KPI once to see the details about the sheet that is linked. Then, click again to navigate to the Customer orders sheet.
  33. Come back to the Sales Analysis sheet and click on the pivot table at the bottom of the sheet. Go to Appearance > Presentation > Styling.
  34. Under General section, choose a Single color as the Background color and type: #f9f9f9.
  35. In the Background image option, select Media library. Select Upload media to upload the WorldMap.png file, that can be found in the StudentFiles folder of this module. When the image is already loaded, select In app and choose the file we have just uploaded.
  36. Click Insert. Select Always fit and the bottom-center position.

Solution

Click on the Solution image to open as a pop-up, or right-click on the image for more options. To close, click on the image again.
