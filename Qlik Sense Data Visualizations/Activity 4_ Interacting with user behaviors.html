Create Visualizations with Qlik Sense

Activity 4: Interacting with user behaviors

Summary:

  • Apply dynamic calculations to visualizations and objects in sheets to create interactive text and actions.
  • Use calculation modifiers, trend lines, and reference lines to provide context and insights.
  • Include calculate conditions and show conditions to maximize user attention focus and create a guided analysis.

Module: Working with other visualizations and properties

Qlik Sense app: ABC Sales Analysis v2

Challenge

Apply some dynamic and interactive responses in the ABC Sales Analysis v2 app:

In the ABC Dashboard sheet, modify the sheet title and sheet description to provide more context:

  • For the sheet title, use the basic title but include information for the selected Year and Region values. Always display all the possible or available regions and only the selected years. If there are no regions selected, list all the regions and if there are no selected years, do not display any years. Title examples:
    • With no selections, the title should read: ABC Dashboard - Americas, Asia & Australia, EMEA.
    • With 2022 and 2023 years and Americas region selected, the title should read: ABC Dashboard 2022, 2023 - Americas.
  • For the sheet description, include a sheet description and display the following information: Last order date, Last reload date, and Today's date.

The Product Analysis sheet analyzes relationships between different products correlated by different measures. This sheet should be hidden if a single product is selected.

In the Sales analysis sheet, adjust the container properties to show the waterfall chart only when the data contains a single year:

  • Apply a show condition to the waterfall chart embedded in the container.
  • The condition should show the chart when one single year is selected or available based on other selections.

In the Customer sales sheet, show only relevant information and enable buttons based on the data selected. Additionally, classify customers dynamically in three tiers to allow advanced customer classification based on selections:

  • If the selection has 5 or fewer customers, the table should be rendered with the calculated results.
  • If the selection has more than 5 customers or no customers have been selected, the table should not display the calculated result, and instead, display an appropriate message.
  • When the table is rendered with results and only one Product is selected or available, a new column with information about the customer's country and city in the following format: Country ( City ) should be shown. Otherwise, this column should not be displayed.
  • If there is only one customer, enable the Customer orders button, otherwise disable it.

Hints

  • In order to create dynamic text, you can concatenate static text and expressions using the ampersand (&) symbol. Use single quotes for the static text and expressions for dynamic text. Use the GetFieldSelections() and Concat() functions to get the list of selected or possible values in a field. You can add extra parameters to functions to include separators for the values. Notice that you might need to use Distinct when applying the Concat function.
  • The Year field is an auto-generated field and in order to use it in an expression, you will need to refer to it as [OrderDate.autoCalendar.Year]. Use the Max() function on the OrderDate that returns the maximum value available. This function is applicable to numeric and date fields. When applied to dates, you might need to format the numeric date value again using the Date() function. Use other date functions to represent last reload and today's date: ReloadTime() and Today().
  • A Show condition can be set to each object in a container object. In the container properties, under Content, select the corresponding visualization and use the Show Condition input box to apply the condition.
  • A calculation condition in the table can be set using Add-ons in the properties panel, under the Data handling section. Each table column can also be hidden or displayed based on a condition using the Show column if property, under the Data section in the properties panel.
  • A button can be disabled or enabled based on a calculation condition. In the Enable chart section of the button properties, set the Condition to enable chart option on.
Step-by-step guide
  1. In the ABC Sales Analysis v2 app, open the ABC Dashboard sheet in edit mode.

    2. ABC Dashboard

  2. In the Sheet properties panel, add the following Title Expression:

    ='ABC Dashboard '& GetFieldSelections([OrderDate.autoCalendar.Year],', ')&' - '& Concat(distinct Region,', ')
  3. In the Sheet properties panel, add the following Description:

    ='Last order date '& Date(Max(OrderDate),'MM/DD/YYYY')&chr(10)&

    'Reload date '& Date(ReloadTime(),'MM/DD/YYYY')&chr(10)&

    'Today is '& Date(Today(2),'MM/DD/YYYY')

  4. In analysis mode, select EMEA using the pie chart, then select 2022 and 2023, using one of the line charts. Note that the dynamic title changes. Then go to the Sheets area, hover over the ABC Dashboard sheet, and click on the ] info icon to see the description.

    5. Product analysis

  5. Open the Product analysis sheet in edit mode.
  6. In the Sheet properties, under Show condition, enter the following expression:
    Count(Distinct ProductID)>1
  7. In analysis mode, select one single product using the scatter plot in this sheet and check the results. You can navigate to the app overview and see that the sheet is not available anymore.

    8. Sales analysis

  8. Open the Sales analysis sheet in edit mode, clear selections, and select the existing container object.
  9. In the properties panel, under Content, expand the Sales and Margin composition waterfall chart and under Show condition, type the following expression:
    count(distinct [OrderDate.autoCalendar.Year])=1
  10. Turn analysis mode on, and test the condition by selecting one single year. Then clear selections and turn edit mode on.

    11. Customer sales

    Table conditions
  11. Navigate to the Customer sales sheet and edit the Customer sales details table.

  12. Using the properties panel (under Add-ons > Data handling), add the following expression in the Calculation condition field:

    Count(DISTINCT CustomerID) <= 5
  13. Modify the Displayed message to use the following expression:

    ='There are '&count(Distinct CustomerID)&' customers available.

    Please select 1 to 5 customers to see the customer details'

    When there are more than 5 customers in the data set the table displays like this:

  14. Using the properties panel (under Data > Columns ):
    1. Add a new column (measure) in the table, with the label: Country (City), and with the expression:

      =Country& ' ( '& City &' )'

    2. Define the following expression for the Show column if property:
      count(distinct Product) =1

    So when there is only one product selected or available, the table displays like this:

    15. Button conditions
  15. Select the button in the sheet, then under Enable chart, set the Condition to enable chart to On and add the following Calculation condition:
    count(distinct CustomerID) =1
  16. In analysis mode, clear all the selections and review the disabled button and the message in the table. Then, make selections in the scatter plot to verify the calculated condition works fine. What happens when you select the 3 customers with the highest sales amounts? The table displays. And when you select only one region? A new column appears in the table displaying the country and city information for the customer.
