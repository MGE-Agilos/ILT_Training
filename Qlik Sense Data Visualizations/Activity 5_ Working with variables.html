Create Visualizations with Qlik Sense

Activity 5: Working with variables

Summary:

  • Explore and work with Qlik Sense variables.
  • Understand when variables are evaluated and the differences.
  • Use the Variable input extension (part of the Qlik Dashboard bundle) to modify variable values.

Module: Working with other visualizations and properties

Qlik Sense app: ABC Sales Analysis v2

Dependencies: Before working in this activity, make sure you complete Activity 1.

Challenge

In the ABC Sales Analysis v2 app, use variables to improve the analysis and user experience in these sheets:

ABC Dashboard sheet, allow business users to dynamically change the number of customers and products included in this ranking for the two bar charts representing the top ten customers and products based on sales.

  • Create a variable, vTopNumber, to store the number of bars to display in the charts (originally with a value of 10).
  • Use the new variable in the Dimension limits of the charts to dynamically display the top customers and products specified by the variable. Also include dynamic titles in these charts to represent how many bars are being displayed by the variable.
  • Bonus Point: Use the Variable input extension (part of the Qlik Dashboard bundle) to allow end users to interact with the vTopNumber variable definition.

In the Regional sales sheet, represent the number of customers and the percentage of customers per country and city. Work with variables to simplify calculations:

  • Create a KPI with the total number of customers.

  • In a new column in the pivot table, display the percentage of customers per country and city over the total number of customers.

  • Use the following variables in the KPI and pivot table:
    • vTotalCustomers, with an expression that evaluates the total number of distinct customers.
    • vNumCustomers, with an expression (in plain text) to get the number of distinct customers. This variable will contain the expression but it does not evaluate it until the variable is used.

Hints

  • You can create variables when editing a sheet, using the Ò icon.
  • A variable with an equal sign in front of the value evaluates the expression globally. This can be used to evaluate the total number of customers.
  • Variables can be used in expressions and visualizations using the dollar sign expansion: $(VariableName).
  • Calculate the percentage of customers in each country by dividing the vNumCustomers variable by the vTotalCustomers variable.
Step-by-step guide
  1. In the ABC Sales Analysis v2 app, open the ABC Dashboard sheet in edit mode.

    2. ABC Dashboard

  2. Click the Ò icon on the bottom left corner. The Variables dialog opens.
  3. Click Create new and enter the following variable details:
    • vTopNumber as Name
    • 10 as Definition
  4. Click Create to save the variable and then click Close. The variable is now available to use.

    Note: Variable names are case sensitive in Qlik Sense. In order to calculate and use this variable in an expression, apply the following syntax: $(vTopNumber)

  5. Select the Top 10 selling customers bar chart. Then in the bar chart properties change the following:
    • Under Data > Dimensions, select the dimension and set the Limitation using the expression below:

      $(vTopNumber)

    • Under Appearance > General, set the title to this expression:

      ='Top '&$(vTopNumber)&' selling customers'

  6. Repeat the previous step for the Top 10 selling products bar chart.
  7. Change the vTopNumber variable definition to 5. Then in analysis mode, review the charts.
    8. Bonus Point:

    Use the variable input extension (Qlik Dashboard Bundle)

  8. In edit mode, create some space between the text and image objects and the bar charts and drag a Variable input from the Assets panel > Custom objects (under Qlik Dashboard bundle).
  9. Use the object properties, under Appearance > General, to add a Title, Set the number of top selling items.
  10. Under Appearance > Variable, select the vTopNumber variable (and Show as Input box). Then stop editing the sheet and notice how users can change the variable value using this object and how the bar charts react when the value is changed.

    11. Regional sales

  11. Open the Regional sales sheet in edit mode.

  12. At the bottom of the Assets panel, open the Variables overview and click Create new to create a new variable, vNumCustomers.

  13. Type the following expression as the variable Definition and save the variable by clicking on the @ icon:

    count(distinct CustomerID)
  14. Add a second variable, vTotalCustomers using the following Definition:
    =count(distinct CustomerID)
  15. Save the variable. Now there are two new variables in the app:

  16. Add a new KPI in the sheet next to the pivot table.
  17. Type the following expression as the measure for the new KPI:
    $(vNumCustomers)

    You can include a label as well, Num Customers.

    Note: Variable names are case sensitive in Qlik Sense. In order to calculate a variable, use the following syntax: $(variableName)
  18. Select the pivot table and use the Properties panel to add a new measure:
    $(vTotalCustomers)

    What is displayed in the new column? Can you explain why?

  19. Modify the expression as follows:
    $(vNumCustomers) / $(vTotalCustomers)
  20. Format the new expression as Number (12.3%) and add a label, %Customers.
  21. Disable the edit mode and clear all the selections. What is the new column in the table representing? What is the percentage of customers in Argentina?

  22. Select Argentina in the table. Looking at the new KPI you created, you can see there are 3 customers in this country. But why has the %Customers value changed?
  23. Enable Edit mode in the sheet and open the Variable overview panel.
  24. Modify the vTotalCustomers variable using the following Definition:
    =count(distinct ALL CustomerID)
  25. Disable edit mode and review the results in the table again. Make different selections in the Country field and see how the percentages remain correct.
  26. Note: The ALL qualifier in the expression overwrites the user selections as it defines the whole data set. You can also use set analysis to achieve the same, as this is equivalent to {1} TOTAL. See more info about TOTAL F in the help site.
