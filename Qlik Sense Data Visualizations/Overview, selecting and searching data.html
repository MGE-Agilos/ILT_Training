Qlik Sense platform and Qlik Sense app

Qlik offers a Data Integration and Analytics platform where users can get actionable insights from raw data.

In the Analytics platform, Qlik Sense works with user data from different data sources or the data catalog.

The Qlik Sense platform allows users to:

  • Extract, transform, aggregate, combine, and associate data
  • Visualize, analyze, search, and select data
  • Discover and share data insights
  • Collaborate with others in shared development areas
  • Monitor metrics and charts and send alerts

Hub

The data exploration starts in the hub where users can create and access apps. Depending on the version of the product we are working on, we can find some differences.

Qlik Cloud Analytics Hub

When Qlik Cloud launches, the hub is displayed along with all the apps to which a user has access.

Apps in the hub can be shared and organized into spaces. The spaces types you should know are the following:

  • Personal space— where you edit and upload your existing apps and create new ones.
  • Managed space— where published apps are organized and their access is governed. Users with a determined role and administrators can create managed spaces and invite other members.
  • Shared space— where a group of users can co-develop apps. Users with professional license and administrators can create shared spaces.

Find more info about the Qlik Cloud Analytics Hub here.

Qlik Sense Hub

When we talk about Qlik Sense Enterprise Client-Managed, the Hub has a different name: the Qlik Sense Hub.

  • Users have an area called Work, where they can create and work on their apps.

  • If they publish an app, it will immediately appear in the Published area.

  • When published, apps are organized into streams. Only users with admin rights can create streams and give others permission to read and/or publish apps.

Find more info about the Qlik Sense Hub here.

App overview

Qlik Sense is a web-based platform that has a number of different views to work from.

The App overview displays a summary of the main components in the app (sheets, bookmarks, and stories). From this area, you can also see the App information section and configure global settings for the app.

In addition, the App overview can be used to navigate to different views, or work functions, categorized as:

  • Prepare: Add, transform, and load data, and manage data associations. Additionally, you can define the business logic to improve the Insight Advisor when providing insights or searching the data.
  • Analyze: Navigate to existing sheets. Edit and create sheets for analysis, navigate through them, create bookmarks, share notes and generate insights.
  • Narrate: Create, edit, and play stories.

The view of choice depends on whether you are working with the data and building a data model, exploring and discovering data, or working with data storytelling.

In a new app, only the Prepare drop-down menu is available. Once data has been added into the app, the other toolbar items become available to the user.

Sheet view

In the Sheet view F , you can:

  • Create and edit visualizations (Edit sheet mode)
  • Analyze data
  • Take snapshots of visualizations for building stories
  • Work with bookmarks
  • Configure general app options, such as adding thumbnail image for easy identification
  • Download or export most visualizations as an image, PDF, or data
  • Create and share notes
  • Extract insights from data
  • Subscribe to a chart, monitor it or create alerts.
Storytelling view

In the Storytelling view F , you can:

  • Create stories and share business insights
  • Publish stories to share with other users

  • Allow users to play back story and act on discoveries

Data model viewer

In the Data model viewer F , you can:

  • See an overview of the data structure of your app
  • Display metadata about the tables and fields
  • Create master dimensions and measures from the fields
Data load editor

In the Data load editor F , you can:

  • Connect your app to data sources
  • Create, edit, and run data load scripts to load data
  • Use debugging tool to make corrections to the data load script
Data manager

In the Data manager F , you can:

  • Add data easily from your own data sources or the Data catalog, using the existing data connectors
  • Manage data table associations and apply association recommendations with all possible table pairs
  • Auto-generate data load script, so you can use it in your app without learning a scripting language
  • View all data tables in the app, whether you added them using Add data, or loaded them using script in the Data load editor
  • Add, edit, and delete tables, and add calculated fields to perform various table operations
  • Apply changes and reload data
  • Manage automatically concatenated and joined tables
Business logic (Logical model and vocabulary)

In the Business Logic F, you can work on a Logical model and the Vocabulary sections to:

  • Define the business rules and metadata to customize and guide insight generation and interaction
  • Group fields in data categories and classify each field based on data classes
  • Define the default aggregation for each field
  • Decide to include or exclude data values in the Lookup operation at field level
  • Hide or show fields from the Insight advisor field list and search

  • Create vocabularies for Insight advisor defining terms and values that may be used in queries that are not present in your data model

  • Add example questions to give ideas to your users of results they can obtain with the app

In a Qlik Sense app, we identify the following building blocks:

  • Data load script — connects to a data source and retrieves the data
  • Data model — provides structure for the loaded data
  • Expressions — calculations involving functions, fields, and operators
  • Measures — numerical data fields calculated for use in visualizations
  • Dimensions — named fields for use in visualizations
  • Visualizations — chart objects and other sheet objects that represent data

Basic components in visualizations

Most data visualizations are created using 2 components:

  • Dimensions F
    • Fields that determine how data in a visualization is grouped.
    • Typically found as the slices in a pie chart, on the x-axis of a vertical bar chart, or a row in a table.
  • Measures F
    • Created from expressions composed of aggregation functions, fields, and mathematical operators.
    • Calculations used in a chart are typically represented on the y-axis of a bar chart or a column in a table.
    • Measures are the result of calculations that return a single value summarizing all relevant records.

In order to perform calculations with the data, most charts group the data using one or more dimensions and then perform one or more calculations (using measures).

Most chart types such as line, bar, or pie, use at least one dimension and one measure to represent the data.

Review the different Qlik visualizations F to find out how dimensions and measures are represented in each of them.

Qlik Sense allows business users to change the dimension or measure of a visualization on the fly without the need of editing the visualization by including alternative dimensions and measures in a visualization.

The visual exploration menu also allows users to change other visualization properties.

Basic chart aggregations

Here are some of the common chart aggregation methods:

Function Description
Sum() Calculates the total of the values given by the expression or field across the aggregated data.
Count() Aggregates the number of values, text and numeric. The optional DISTINCT qualifier sets the function to ignore duplicate values.
Avg() Returns the aggregated average of expression or field iterated over the chart dimensions.
Median() Returns the median value of the range of values aggregated in the expression iterated over the chart dimensions.
Max() Returns the maximum numeric value or expression over a number of records.
Min() Returns the minimum numeric value or expression over a number of records.
Note: When a field name is used within an expression, for example, the field Measure in Sum(Measure), spelling is extremely important and it is case sensitive. If you incorrectly spell the expression such as Sum(measure), it will not work.

All chart expressions are classified in different categories such as Mathematical functions F, Field functions F, Counter functions F, Conditional functions F, etc.

Qlik Sense can also perform advanced analytics calculations with K-means functions F to perform clustering and analyze segmentation.

Selections

The filters you apply to represent a chosen subset of data in an app are referred in Qlik as selections.

You can make selections to focus on particular topics and all associated visualizations will update to reflect that selection.

Selections can be performed in analysis mode in different ways:

  • Selecting a value from a filter pane displayed on a sheet

  • Clicking to select parts of visualizations (charts and tables) on a sheet

  • For certain visualization types, using the lasso and/or circle selection tool/s
  • Using the Selections tool
  • Using the Selection bar to change existing selections
  • Using the search or the Insight Advisor

Once a selection is made, users can see the selected values and fields from the Selections bar at the top of the sheet. Additional selections in fields not available in the sheet can be performed using the Selections tool.

By default, when making a selection, all the sheets and visualizations will react to the selection. But, it is possible to define different sets of selections for the same dimensions or fields using Alternate states or using specific syntax in expressions (Set analysis) that can override user selections and operate the calculation with a different set of data. For additional details about Alternate states or Set analysis, please visit Controlling data selections.

When analyzing data, the following operations can be found useful when working with selections:

  • Move forward and backward in your selections and clear selections to restore the initial set.
  • Lock selections, fixing some selections in one or more fields so they cannot be cleared.
  • Set one specific value that will remain selected for a field. This feature keeps a single value selected in the field that cannot be cleared.
  • Select possible, alternative, and excluded values based on a selection set.
  • Search for and select dimensions that weren't initially selected.
  • Review associative insights with the Insight Advisor, finding hidden relationships with the non-associated values.
  • Store the selection as a bookmark.
  • Apply an initial selection when opening the app using a default bookmark.

Selection states

Qlik Sense uses the Associative selection model F when users make selections in an app to represent what data is being filtered and what data is excluded or available to include in the selection.

Filter panes are color-coded to indicate which values are selected, alternative, possible, and excluded, as follows:

Selected values

Indicator: Green background with a checkmark
Values that have been selected to filter the data.
For instance, if you select 1950's in the Decade filter pane, the 1950's is a selected value.

Possible values

Indicator: White background
All the values 'Associated' with selected items. You can refine your selection by selecting one or more of the possible values.
For instance, if you selected 1950's in the Decade filter pane, the movie title called A Lesson in love will be a possible value (if not selected).

Alternative values

Indicator: Light gray background
All the unselected values in a filter pane where a selection is made are considered alternative values.
These are values that would be possible if a selection had not already been made in that field. They are excluded, but they are only excluded by a single selection in the same filter pane.
For instance, if you selected 1951 in the Year filter pane, other years in the Year filter pane will be alternative values.

Excluded values

Indicator: Dark gray background
Values in other filter panes not associated with the selected value.
For instance, if you selected 1950's in the Decade filter pane and then 1951 in the Year filter pane, other values in the Decade filter pane will be excluded.

Selected excluded values

Indicator: Dark gray background with a checkmark
Values that were first selected, and then excluded by a new selection in another field.
For instance, if you selected 1950's and 1960's in the Decade filter pane, and then you select 1951 in the Year filter pane, the value 1960's in the Decade filter pane will be a selected excluded value.

Click on the arrows next to each value (above) or click on the different areas in the image (below) to see more info on each color code.

Associative insights

When applying selections, Qlik's cognitive engine analyzes the data model and can help identify hidden relationships with the excluded values from the selection set.

In the Selection tool, the Associative insight panel will display on the right.

If the Associative insights panel is closed or hidden, click Explore selections to open the panel.

To discover the contributions of your selections and excluded values against your measures, and reveal relationships that you may have missed, follow these steps:

  • Make selections in the app
  • Click Generate insights from the Associative insight panel (in the Selections tool)
  • Analyze the results:
    • Change the measure of analysis using the drop-down menu, if needed.
    • Click on the insight cards to find out more about relationships and contributions to the excluded values.
    • Using the cards, show related data and apply new selections (possible or excluded values).

Bookmarks

Some selections states can be reusable for other business users or can be set as a starting point for user analysis:

  • Common selection states can be defined and stored in the app by the business analyst.
  • Business users can apply these predefined selection states or create new ones for personal use.
  • Selection states stored in the app are known as Bookmarks.

A Qlik bookmark stores a specific set of selections and a location in the app (the sheet where the bookmark is created).

Users can create, modify, and apply bookmarks from the > button within the toolbar of the app and a Qlik Sense app can store many bookmarks.

Using bookmarks is a way to easily:

  • Keep track of a certain selection state on a certain sheet.
  • Guide the user to a relevant sheet and create a solid starting point for further analysis.
  • Land on the sheet you were exploring when you created the bookmark.
  • Allow a connection between storytelling and the data using bookmarks as links.

See more info about Bookmarking selections F in the help site.

A default bookmark can be set in the app to be applied as the initial selection when opening the app. By specifying a default bookmark, the default selections are applied when a user opens the app. See more info about Setting a default bookmark to create an app landing page F in the help site.

After publishing an app, it is not possible to edit bookmarks that were included in the app from the beginning, however, you can create your own private bookmarks instead.

Buttons can apply bookmarks if they include the corresponding bookmark action.

Search types and Insight Advisor

Users can search in the app using different capabilities:

Smart search (Global search)

Smart search is the global search tool in the app.

When you are analyzing data on a sheet, Smart search is available in the selections toolbar to find any relevant value in the entire data set in the app.

If you perform a search with two or more terms, the search generates one query that includes all search terms and then displays the available combination of results, with each combination on a separate row. It is also possible to search for multiple search terms sequentially.

Other related topics:

  • Using Smart search F

List searches (Field search)

List search makes it easier for you to find or filter in filter panes, selection items, and tables, and make selections in the resulting list.

There are different types of list searches:

  • Text search (including wildcards)
  • Numeric search
  • Fuzzy search
  • Expression search

Note that there is a limit of 5,000 characters for the search values on a filter pane.

It's also possible to use these searches when working with set analysis by using Set modifiers with advanced searches F.

Insight Advisor (Visual and natural language search)

Search using natural language or request insights as well as data items in the Insight Advisor F, also known as IA.

In an app you can use the Insight Advisor and perform natural language queries on the app. The search results are returned as a gallery of visualizations in which the search terms are found. Click on a visualization to go directly to the sheet it comes from or to add them to a new or an existing sheet.

You can also use the Qlik Insight Advisor Chat. This allows you to perform natural language searches and conversations with the Insight Advisor over several apps. Use the Ask Insight Advisor F button on the hub.

Qlik Sense Desktop: Insight Advisor is no longer supported from August 2022 release.

Qlik Sense auto-generates the best analytics and insights for you based on the overall data set and search criteria.

  • This feature is powered by the Qlik cognitive engine, which works with the Qlik Associative Engine to provide machine suggestions and automation in combination with user intuition.
  • It finds new and existing insights in the data you have loaded, and suggests charts based on them, allowing you to have a complete view of your data to augment visualizations that already exist in the app.
  • Users can start a search using the Insight Advisor (available from the Insight Advisor button). Here, you can add new visualizations in the Qlik Sense app without having to define the visualization type, expression, or dimensions.

  • You can find the Insight Advisor analysis types F which means that you will be prompted to choose the type of analysis from a variety of predefined available options (ranking, period over period, correlations, etc). It is a more directive approach in case you are looking for a specific analysis, but you can still use the Insight Advisor in your own way.

  • There is a global Insight Advisor option called Ask Insight Advisor. Here, you can converse with the IA by making natural language queries over multiple Qlik Sense apps.

  • Users can search using natural language and generate insights on master items, which can then be used by consumers of published apps. The generated results and filtered calculations can also be stored as master items (reusable items) in the app.

  • In the Business logic sections (under the Prepare area) of each app, there are choices for Logical model and Vocabulary, where you can define aspects of the app metadata and vocabulary for the particular data set to help the Insight Advisor to interpret natural language queries.
