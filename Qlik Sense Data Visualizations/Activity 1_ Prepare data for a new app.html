Create Visualizations with Qlik Sense

Activity 1: Prepare data for a new app

Summary:

  • Create a new Qlik Sense app and add app data from Excel.
  • Use Data manager to prepare the tables for analysis with source filters, field transformations, and new field creations.
  • Gain familiarity with the Visual preparation (Data manager associations) to associate the tables and create the data model.

Module: Preparing data for analysis

Qlik Sense app: None, user creates a new app, ABC Sales.

Data Files: Qlik Sense Data\ExcelFiles\SalesData_MultiTabs.xlsx

Challenge

Create a new app called ABC Sales and add data from the MS Excel file named SalesData_MultiTabs including all the worksheets in this file.

Locate this file under the ExcelFiles folder or download it from the Student Files.

Apply some data transformations as described below and load the data into the app:

Data transformations:
  • In the products table:
    • Apply a filter to the table to remove rows related with the "Free Product Kit" (with ProductID = 90000)
    • Split the Location field into 3 fields using the dot as a separator: Floor.Section.Row
  •  In the customers table:
    • Rename the Contact field to Customer_contact
    • Replace the United Kingdom values in the Country field with UK

  • In the orders table:

    • Format the OrderDate field using the MM/DD/YYYY format
    • Create a new field called Sales using the expression:
      Quantity * UnitPrice * (1- Discount)
    • Bucket the sales into 3 groups creating a new field called Sales Type:
      • Low sales - sales lower than the average value of 1180
      • Medium sales - sales greater than or equal to 1180 and lower than 6000
      • High sales - sales greater than or equal to 6000
  • In the employees table, create EmpName as a calculated field using the expression:
    [First Name] &' '& [Last Name]

Associate the tables using the recommended associations as follows:

Table associations
  • orders and products by ProductID
  • orders and customers by CustomerID
  • orders and employees by EmployeeID and EmployeeKey
  • customers and Regions by Division

Load the data and review the final data model and table structure in the Data model viewer.

Hints

  • Files can be loaded into an app using the Files and other sources option or just by dragging and dropping the file into the app.
  • Prepare the data by filtering the rows, renaming fields and tables, and creating new calculated fields from the Prepare > Data manager > Tables section. You can also associate the tables identifying the key fields in each with the visual data preparation view (Data manager > Associations).
  • The Load data button loads the tables and fields you define into the app.
  • Once the data is loaded in the app, you can review the tables and fields using the Data model viewer.
  • The ▶ Associations view in Data manager allows you to associate the tables, using the visual data preparation. There are different methods to associate tables:
    • The Ñ button shows the recommended associations and allows you to associate all the tables according to Data manager recommendations. You can break undesired associations afterwards by pulling the bubbles apart until the association breaks.
    • Tables can be associated one by one by dragging them on top of one another. If there is a recommended field/s for the association, the tables get associated using the key field. If there are no recommended associations, you choose the fields from each table and create a custom association.
    • Existing associations between tables can be changed by selecting the joint point between tables. You can change the recommended field or create a custom association.

 

Step-by-step guide
  1. From the Hub, create a new analytics app called ABC Sales.

  2. Open the new app and add all the data from the SalesData_MultiTabs Excel file by dragging and dropping the file into Files and other sources area.

    If the data file was loaded automatically, click Cancel to continue with Data manager.

    If the Select data dialog appears, make sure all tables are selected and the data profiling is Enabled. Then click Next.

    Note: You can add Filters when loading the data using the Filters tab in the Select data dialog or when editing the table in Data manager.
  3. In the Data manager, select the Tables ' view and verify that all the tables are displayed in Pending add status.

    4. Creating new fields and filtering product data

  4. Select the products table and click Edit @.
  5. Select the header in the Location column. The values in this field correspond with a 3 number code identifying the Floor, Section, and Row where the product is stored in the shop. In the Split section, make sure both dots are selected and click Create fields.
  6. In the table, rename the new columns as Floor, Section, and Row respectively.
  7. Select the first column in the table that represents the product ID. Notice there are a total of 79 identifiers for products. Click Select data from source on the top right corner of the dialog. Then click Filters (at the top right corner) and select Add filter specifying the following filter: products.ProductID > 90000.
  8. Click Update, notice how the total number is now 78 and close the product table.

    9. Transforming customer data

  9. Select the @ edit icon in the customers table. Then rename the Contact column as Customer_contact.
  10. Notice how the City and Country fields in this table have been identified as geo fields. Select the Country column and review the Summary bar chart at the bottom. How many distinct countries appear in the data?
  11. In the Replace section, select United Kingdom and type UK as the Replacement value. Click Replace. How many distinct countries appear now in the data? Close the customer table edit dialog.

    12. Preparing order data

  12. Edit the orders table and review the OrderDate field. This field has been identified as a date field. From the Date icon (), set the date format for this field from the Display format drop-down (MM/DD/YYYY).

  13. In the orders table, click Add field > Calculated field to add a new field, Sales, using the following expression:

    Quantity * UnitPrice * (1- Discount)

    14. Note: Make sure your expression is the same as above. In Qlik Sense field names are case sensitive.

  14. Select the new calculated field and review the metric below the table. Qlik has identified the Sales field as a recommended measure and displays some statistics based on the data. Use the transformation cards to classify the sales values into 3 groups: High, Medium, and Low.

  15. In the Bucket section, at the bottom, create 3 buckets. The default buckets are split equally across the values.
    1. Select the first bucket and set the Bucket name as Low sales and the Up to value to 1180.
    2. Click the right arrow to move to the second bucket and repeat the process using Medium sales for the bucket name and 6000 for the upper limit.
    3. Click the arrow again to edit the third bucket and set the name to High sales. Then select the left arrow and click Create buckets.

    A new field is created in the table, rename the column as Sales type. Then close the orders table.

    16. Adding new employee data

  16. Edit the employees table and add a new calculated field, EmpName, using the following expression:

    [First Name] &' '& [Last Name]

    Please note the field names are specified using square brackets in this case as these names contain spaces.

    Note: You can concatenate strings or field values using the ampersand (&) character. Plain text or strings can also be added using single quotation marks.
  17. Review the metrics below the definition to see more details about the new field created. Notice the difference between the count of this field and the distinct values. This difference is due to some duplication of rows in the table (example: review in the table the rows for EmployeeID E0003). Then close the employee table.

    18. Associating the data

  18. Navigate to the Associations view to associate all the tables.
  19. Associate the tables need as follows:
    • orders and products are associated using the ProductID field in each table.
    • orders and customers are associated using the CustomerID field in each table.
    • orders and employees are associated using the EmployeeID and EmployeeKey fields from each table.
    • customers and Regions are associated using the Division field in each table.

    The Ñ button shows the recommended associations in a panel on the right, where you can preview them. Click Apply all to apply all the recommended associations and load the tables. Navigate to the Data model viewer and make sure the final table structure matches the solution diagram.

  20. Return to the Data manager (Associations view) and select the association link between employees and orders. Then rename the association name to EmployeeID.
  21. Load data again to update all the changes and close the dialog after the data is reloaded.

    22. Loading and reviewing the data

  22. Click Prepare > Data model viewer. Move the tables so they are visible and do not overlap others. Notice how all the tables are now in the diagram connected to each other.

    Note: Use the options in the toolbar to expand all the tables and see the fields.
    Note: After loading the customers table, new geo fields are created automatically in the table (see fields with _GeoInfo suffix). These fields contain the coordinates and area map for the city and country fields. You can review the data in these fields using the Preview panel.
  23. Select the orders table and the Sales field in this table and review the Preview panel at the bottom to see a sample of data in the orders table.
