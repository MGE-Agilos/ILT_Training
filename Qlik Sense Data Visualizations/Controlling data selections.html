Controlling data selection benefits

In Qlik Sense by default, all the calculations are performed over all the available data set in the app. Users can apply filters by making selections in the data to perform a different analysis or focus the analysis to a specific subset of the data.

The current selections in the app define the selection state or data set that is being evaluated:

  • User selections reduce the data set that is being evaluated and define the selection state
  • By default, all the visualizations and calculations in all the sheets in the app react to the selection state (user selections)

Business users can change or clear selections as they need to review other areas of the data, but this action is not always required. Some metrics require specific calculations that need to operate or evaluate the whole data set, or specific sets of the data not related with the user selections or filters.

Some examples of these cases are:

  • Time and date comparisons: Year on year, CYTD versus LYTD, rolling months, etc.
  • Percentages over all or over an area: % customers per region, % attendees per class, etc.
  • Totals and absolute values: Total sales for current year, Total number of cases, etc.
  • Basket analysis or advanced analysis: Sales based on customers buying X product, Sales based on products with this Y margin, etc.

Working with selections and controlling them can provide business analysts and business users with a very powerful way of performing data analysis in Qlik Sense.

Business users can create their own selections or filters in the data by:

  • Making selections in visualizations, filter panes, or in the app fields by using the Selections tool.
  • Applying predefined selections stored in the app (using bookmarks).
  • Making searches with the global search and the visual search (Insights).
  • Using triggers in buttons or sheets with actions to perform selections, searches, or apply bookmarks.
  • Applying the bookmark or the select parameters in the app URL.

Business analysts can control or define different sets of data in the calculations or visualizations by:

  • Saving predefined selections using bookmarks and using them in calculations.
  • Defining and working with alternate states applying them to sheets, objects in the sheet, or calculations.
  • Setting specific actions or triggers with the data selections to apply when the business user navigates in the app or clicks on a specific button.
  • Using calculations provided by the calendar measures, reference lines and modifiers in measures, or calculations with filter generated by the Insight Advisor.
  • Defining their own data sets or subsets in expressions or use predefined data sets when defining the master measure or calculation. This is achieved using special syntax with the expressions called Set analysis syntax.

Defining alternate selection state

In some cases, business users need to compare different data sets dynamically or need alternative visualizations or areas in the app where they can apply a different set of selections from the current selection state in the app without having to clear the current selections.

Business Analysts can define Alternate states in the app. They are selection states that represent different or similar filters to the current selection state:

  • An alternate state is defined in the app and applied to the sheet objects or to the sheets.
  • Alternate states are defined as master items with a unique name and can be used across all the app in the sheets, objects, and expressions.
  • When defining alternate states in a sheet or object, you can select from:

    • Any alternate state defined in the app.

    • <inherited>, in which case the state defined for the sheet is used.

    • <default state>, which represents the current selection state where no alternate state is applied.

  • Objects or sheets in an alternate state will not react with the default selection but will apply selections based on the alternate state they have assigned.
  • You can combine different states in the same visualization defining alternate states for each expression.
  • When applying an alternate state to an expression, you can use the alternate state unique name as set identifier in the set analysis syntax. Example:
    • Sum( {StateName1} Sales)

      In the expression above, the sum of sales will be performed using all the data set including any selections in any filters made in fields or sheets in the StateName1 alternate state.

For more information about alternate states, search for Using alternate states for comparative analysis F or download this guide.

Defining and working with data sets in expressions

Controlling selections at the expression level in a Qlik Sense app is possible using a specific syntax in the expression, also known as set analysis syntax.

Qlik allows users to analyze sets of data in an app not driven exclusively by user selections, but by other dynamic or static sets defined using set analysis.

Set analysis

Set analysis is a feature in Qlik Sense useful for:

  • Providing power to Qlik Sense apps by controlling the data set that is being evaluated for each expression.
  • Allowing dynamic solutions to cut down on the amount of scripting that might otherwise be required.

  • Improving performance over the use of an IF statement.

Set expressions

A set expression is a set of field values defined in a statement. Use set expressions to work with set analysis.

Set expressions are:

  • Only used in the context of an expression
  • Not available to use in the data load script
  • Composed of three elements, but not all parts have to be present

The three parts of a set statement are:

  • Set identifier (can be omitted in some cases)
  • Set modifier (optional)
  • Set operator (optional)

When using a traditional inner set expression, all these component must be written inside the brackets of the aggregation function in question. For instance, in the expression Sum( {$<Region={EMEA}>} Sales ) the identifier and the modifier are inside the Sum() brackets.

For more information, search Qlik Help pages for Set analysis and set expressions F.

Set identifiers

The set identifier defines how the set of field values in the app relates to the field or expression you are evaluating.

Examples::

Set identifier Example Result
$ Sum({$} Sales) Returns sales for the current selection, so it is the same as Sum(Sales). This identifier can be omitted.
1 Sum({1} Sales)

Returns total sales within the application, disregarding the selection, but not the dimension.
$1 Sum({$1} Sales) Returns sales for the previous selection.

For more information, search Qlik Help pages for Syntax for set expressions F.

You can also use Bookmarks and Alternate states as set identifiers:

  • Use the Alternate state name in set analysis: Sum( {State2} Sales)
  • Use the Bookmark name in set analysis: Sum( {MyBookmark} Sales)

Set modifiers

Modify a set by making an additional or changed selection. The modification is written in the set expression.

The modifier consists of one or several field names, each followed by a selection to be made on the field, all enclosed by angle brackets < and >.

Examples:

Sum( {1<Region= {'USA'} >} Sales )

Returns the sales for the region 'USA', disregarding the current selection.

The curly brackets define the element set, where the elements can be either field values or searches of field values.

Sum( {$<Year = {2016}, Region = {'USA', 'SEW','UK'}>} Sales )

Returns the sales for the current selection, but with the new selections both in Year and in Region.

When multiple field values are listed within curly brackets, the values are separated by commas.

Sum( {$<Year = {">2013<=2016"}, Division = {'North America'}>} Sales )

Numeric search that specifies a defined range applied in numeric fields using double quotes, as in the Year field of this example.

Sum( {<Division= {"*America"}>} Sales )

String search that is applied using double quotes and wildcard characters, such as an asterisk or question mark.

This expression returns the sales for all the divisions containing 'America' at the end of the division name, such as 'North America' or 'South America'.

Notice how the set identifier has been omitted. As a result, the selections in other fields will apply to the calculation.

Sum( {$<Year =- {"<2010"}>} Sales )

Set operators can be applied to sets. In this case, the exclusion (-) is used to exclude all the years before 2010 from the available years.

Sum( {$<Year = {2016}, Month= ,Quarter= >} Sales )

This expression specifies fields in the set modifier without a set; all the values in those fields are included in the calculations. The selections in them are not affecting the result.

Sum( {$<Year = {$(=YEAR(TODAY())-3)}>} Sales )

Returns the Sales for the year: current year - 3.

Dynamic expressions can also be used in set analysis. They must always be enclosed in a dollar-sign expansion syntax and begin with =.

Note: If no set identifier is specified, the $ is assumed. It is a best practice to specify a set identifier.

For more information, search Qlik Help pages for Set modifiers F.

Set operators

Use set operators to create new sets based on the union (+), exclusion (-), intersection (*), and symmetric difference(/), also known as mutual exclusion.

Examples:

Union (+)

Year={">2015"}+{2010}

Any year after 2015 and 2010

Exclusion (-)

Year=-{"<2010"}

Any years except years before 2010

Intersection (*)

Year={">2014"}*{"<=2016"}

Only 2015 and 2016 years

Symmetric difference (/)

Year={">2014"}/{"<=2016"}

Any year after 2016 or before 2015

Other ways of using set analysis

Once you understand how set expressions work, you should know that they can be created in a simpler way. You have the option of applying the app selections to an expression using the expressions editor or you can also write the set expressions outside the aggregation function.

Creating set expressions with expression editor

If you want to define a set expression based on current selections or a bookmark, but you do not want to type it directly into the expression field, you can use the expression editor panel and its different sections: Fields, Functions, Set expressions and Variables.

You only have to select the field and the aggregation function you want to use from the Fields section, and then, check the Set expression box in there.

The set expression created will be based on current selection on the default state.

In case you want it to be based on selections in Alternate states or a Bookmark, you can define it on the Set expressions section.
