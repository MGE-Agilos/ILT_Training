Qlik Stories and its components

The purpose of data storytelling is to connect the data discoveries in a story and emphasize elements that are important in your discussions and decision making.

After analyzing the app, users can compose a presentation (StoryÇ ) to narrate the insights and the data discoveries they have found in the app.

 

In a Qlik Sense app, users can create, view or share stories. The Storytelling view in a Qlik Sense app can be launched from the Narrate section.

  • Users can create new stories from the Narrate > Storytelling view or from the App Overview (under the Stories section).
  • Stories consist of one or more slides that can be played in the order they are arranged in the story.
  • Slides help structure your story using snapshots of your data visualizations and embedded sheets.
  • Stories can be printed or exported to PDF format.

A slide can contain:

  • Objects, such as snapshots (with or without visual effects). These snapshots can represent visualizations or sheet objects at a particular point in time with or without selections.
  • Textual content (with links to external URLs or Bookmarks).
  • Various kinds of visual content, such as shapes (for example rectangles, circles, lines, arrows, and annotations) and images.
  • Embedded sheets from the app where the user can still perform selections.

Snapshots

A snapshot is a static representation of a data object at a certain point in time which you can use to build your story.

Snapshots F allow you to collect the insights you encounter in your visualizations.

  • Manage snapshots in the snapshot library, accessible from the visualization’s shortcut menu.
  • Snapshots can be annotated to help distinguish between the different snapshots when you build your story.
  • Snapshots will not be updated at a data reload.

Prepare the app for others

After you have created and built your app with sheets and visualizations, you need to fine-tune it for others to use in an easy and efficient manner.

  • Create reusable master items (visualizations, dimensions, and measures).
  • Use bookmarks to keep track of important and interesting data selections and set a default bookmark if applicable.
  • Add a description and a thumbnail for the app to make it easier to recognize it from the hub.
  • Add accurate descriptions of sheets, visualizations, measures, and dimensions.
  • Include buttons for navigation, selections, or bookmarks that can guide users in their analysis.
  • Tag dimensions, measures, and fields to help users when searching.
  • Create stories based on the business needs and the insights and findings.
  • Remove all visualizations, measures, or dimensions that are not relevant to your business users.
  • Structure the information and the objects in the sheet to help users in their analysis. You can use an app template.
  • Style and brand your app.

Responsive design

Qlik Sense is designed according to the standards of responsive web design.

  • It has been designed to adapt to different operating systems, screens, and devices, many of them with dedicated screen resolutions.
  • Provides a unique user experience allowing you to discover your data on various screens, operating systems, and devices, without having to install additional clients or plug-ins.
  • App developers can choose the rendering layout for sheets on small screens, such as tablets and mobile phones. List view remains the default option, and only shows the visualization titles.

  • The Grid view shows a small preview of your visualizations, and reduces the amount of white space. This gives developers more flexibility and control over how apps designed for mobile are displayed.

Examples:

See more about Interacting with the user interface F and Changing the small screen layout F in the help site.

By default, Qlik Sense uses a responsive layout for sheets that adjusts the sheet to the dimensions of the user’s screen, but you can customize sheets by changing:

  • The grid sizing, to be able to fit more visualizations on a sheet, or have more control over how visualizations are positioned.
  • The sheet size, from Responsive layout (by default) to Custom (specifying the width and height for the sheet) if you want to use a non-responsive layout.
  • The small screen layout option, from List view (by default) to Grid view. Sheets must be set to responsive layout to display as grid on small screens.

  • The extended sheet option, that extends the area of a sheet 50% more vertically. When a sheet is using a custom size, you can change the grid spacing, but you cannot use Extend sheet to change the size.

KPI objects can also be configured to have a layout behavior for the font of the KPI, from the Properties panel, under the Presentation section. There are three options to set the layout mode in KPIs:

  • Fixed: The font size is independent of the box size and the length of the text.
  • Fluid: The font size adapts to the size of the box.
  • Responsive: The font size adapts to the size of the box and the length of text.

Styling and branding an app

Before publishing the app, you can style it based on your company standards. The style will be applied to all sheets in the app.

In the app settings V, you can:

  • Apply a theme. Theme settings also allow developers to set the font family for text in Qlik Sense charts.
  • Set the sheet's background color.
  • Change the default sheet's title font color.
  • Change the reading order (right to left).
  • Add a logo.
  • Hide the toolbar or even the sheet header.
  • Insert an image as sheet background.

Additionally, you can use the app variables to store color settings or other general settings to include styling and branding.

Themes in Qlik Sense

By default, Qlik Sense comes with three default themes:

  • Sense Classic: provides a compact view of objects, and limits the space between them.
  • Sense Focus: adjusts the padding and spacing around objects as well as provides designated spaces for titles.
  • Sense Breeze: is based on the Sense Focus theme but with different color settings.
  • Sense Horizon: is based on the Sense Focus theme but with different color palette.

You can create custom themes based on your company standards and use them in your apps.

Custom themes allows you to style an app:

  • Change colors
  • Add images and backgrounds
  • Specify font sizes and font colors on a global or granular basis
  • Define color palettes
  • Customize margins, padding and spacing

In order to use new themes in your Qlik Sense apps, these themes must be included in Qlik Sense:

  • In Qlik Sense Desktop, locate the theme folder under the: \Users\[UserName]\Documents\Qlik\Sense\Extensions\ folder.
  • In Qlik Sense Enterprise on Windows, in the Qlik Management Console, under the Extensions section, import the Theme (as a zip folder).
  • In Qlik Sense SaaS editions, access the Management console (https://<your tenant address>/console) and add the new themes in the Themes section.

Publishing and sharing

With Qlik Sense Enterprise on Windows and Qlik Sense SaaS editions, you can publish or share apps, sheets, and stories with others.

  • In Qlik Sense Enterprise on Windows, the Qlik Sense Management Console administrator can determine which users can publish different kinds of content into controlled and managed areas called streams. Users work in their personal Work area and publish apps to streams.
  • In Qlik Sense SaaS editions, the owner of the app can decide to share the app or part of the app (only public sheets) with others, and can locate the app in a shared or managed space. From the SaaS Management console the administrator of the tenant also can control and manage users and spaces.

When you publish or share an app:

  • In QSEoW, all sheets and stories of the app become available to the other users that have access to the stream where the app is published.
  • In Qlik Sense cloud editions, all public sheets and stories of the app become available to the users that have access to the space where the app or moved or published. Additionally, it is possible to share an app with non-space members, but only in case you are the space owner or have Can manage permissions.

  • The private sheets and stories that you create as a part of a published or shared app can also be published or shared, making them available to the users with access to the app.

  • The shared or published app is locked (read-only), meaning you cannot make any changes to the published or shared sheets and stories, but you can use them to analyze your data or duplicate the existing sheets. These options are based on your user rights in the specific space or stream.

  • You can add private sheets to the app. When you edit your private sheets, only the master items and charts in the Assets panel are available.

  • You can choose which stream or space the app appears in. Users will get different rights assigned to streams or spaces. This is controlled by the server or tenant administrator.

Active analysis

Qlik Sense helps users to easily access the relevant information in all the apps and data in the hub using pro-active analysis and alerting.

In the Hub (QSEoW and QS Cloud), users can:

  • Ask business questions using the Insight Advisor chat
  • Define and see alerts based on conditions in the data
  • Monitor charts from different apps in one single screen
  • Create subscription reports based on a visualization or sheet
  • Use notes to collaborate with other users

  • Build automated analytics and data workflows

Insight Advisor chat

The Insight Advisor chat is a natural language chat bot that allows you to ask business questions among different apps in the hub.

  • The Insight for chat in hub property must be enabled in the apps we want the Insight Advisor chat to work with. This property can be enabled in each app from the App settings.
  • Master items, business logic, and vocabulary are applicable when asking questions in natural language.
  • Filters are also applicable when performing the question and the Insight Advisor chat.
  • The Insight Advisor chat provides suggestions based on the query and users can select between apps, measures, and dimensions when searching.
  • The results are presented as narrative insights, auto-generated visualizations, and observations followed by other recommended questions to follow up.

See more about Exploring apps with conversational analytics F in the help site.

Monitor data with alerts

In QSE on Windows or QS Cloud you can include data alerts based on conditions.

Alerts are defined by a set of:

  • Master measures: select one or more master measures in the app to review with the conditions.
  • Data set definition: include an optional dimension or bookmark. By default the current selections in the app will be applied. If a Dimension is added in the alert, a table with the dimension and measures is calculated and the conditions will be evaluated per row.
  • Conditions: one or more logical conditions that include the measures to compare, the operator, and the value to compare with. This value can be static, another measure or an aggregated calculation on the current set.

When defining a data alert, a wizard assists you in defining the alert details and data, conditions, and the distribution.

Alerts can be distributed by email and as a notification in the cloud hub. A SMTP server must be defined to send alerts by email.

Alerts are defined for each user but you can also share with others the alerts you create.

See more about Monitoring data with alerts F in the help site.

In QSEoW, an additional installation for an add-on component, Qlik Alerting, is needed. This add-on includes a Qlik alerting portal and a custom extension to include a button in apps for users to launch the alert wizard. This addition is not needed in Cloud editions of Qlik Sense. See more about Qlik Alerting here.

Monitor charts in the Cloud hub

Users can monitor specific visualizations in the Charts area of their cloud hub.

Monitor charts include the selections that were applied in the app when the user started monitoring them.

From the Home or Catalog areas of the hub, all the monitored charts from different apps are displayed allowing the user to see metrics and perform some analysis without having to open the apps.

The monitored charts are dynamically updated when the app is updated. Users can see the history of the chart changes based on the different updates.

Users cannot make additional selections in the monitored charts but they can open the app by selecting View in app.

See more about Monitoring visualizations F in the help site.

Subscription reports in QS Cloud

Subscriptions allows you to schedule recurring emails. Those will contain an attached PDF or PowerPoint file with current data for your preferred chart or sheet, an image of the visualization, a link to the corresponding app and a link to your subscription management console.

  • Subscriptions must be turned on by a Qlik Sense admin.

  • You need a valid email address configured in Qlik Sense.

  • You need to have view permissions for the app in which you are creating subscriptions.

  • Recipients must have access to Qlik Sense.

  • Filters can be applied to subscription reports.

See more about Scheduling reports with subscriptions F in the help site.

Notes in QS Cloud

This feature lets you create notes, share them with other users and work in a collaborating environment.

  • It is possible to create a note from an app (on Notes menu), from the navigation bar or from an snapshot.

  • You can include snapshots of your preferred charts on the notes, so you can illustrate your comments and make them easier to understand to other users.

  • All notes to which an user has access to can be visualized from the Home and Catalog sections, or from the Notes tab when navigating an app. Notes linked to a specific chart can be viewed by right clicking over the chart and selecting Notes > Find notes related to this chart.

  • Notes are organized into three categories:

    • Notes related to charts in this sheet.

    • Notes related to charts in other sheets.

    • All other notes (they are not related to current sheet or do not contain any snapshot).

  • Members of a shared or managed space can view the notes that you create, and in case they have edit permissions, can write and collaborate on it, too. The note owner and note editors can change permissions on a note.

See more about Working with notes F in the help site.

Qlik Application Automation in QS Cloud

Provides a no-code visual interface to automate analytics or create workflows.

  • The automation editor is available from the Cloud Hub by clicking Add new. An automation is a workflow composed by blocks that you select and drag onto the automation canvas in the creation process.

  • There are templates available when you create a new automation. Although they are preconfigured, you can adapt the template or modify it.

  • Users can access their personal automations from the Catalog section of the Cloud hub.

  • By using Qlik Application Automation, the Qlik Reporting connector allows to create multi-sheet and customized reports, that can be in PDF or PowerPoint format. It is possible to insert sheets from different Qlik Sense apps on these reports.

See more about Qlik Application Automation F in the help site.

When using these features, we may receive numerous notifications through web or email. To temporarily stop the delivery of all your notifications, click Pause notifications in the top right corner of the Profile settings page.
