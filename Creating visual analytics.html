Why visualizations?

Visualizations are graphical representations of numerous data points that allow us to easily understand the data and discover trends, patterns, and insights. See the following examples:

Cholera Map

John Snow created this map in 1854 showing the location of London’s 13 public houses and 578 cholera deaths using stacked bars based on location. In this case, the choice of representing the geodata (location) of the data points, generates very meaningful insights.

See more info here.

Anscombe's Quartet

This table contains four sets of values known as Anscombe's Quartet.

If you plot the data in charts, it is immediately obvious that the sets are very different indeed.

The fact that you immediately can tell that the four charts are different is a result of the efficiency of the human visual system to detect patterns.

If you want to know more about visual concepts and human perception, go to the Other resources section.

Presentation types

There are four basic presentation types that you can use to present your data:

  • Comparison
  • Composition
  • Distribution
  • Relationship

To determine which chart is best suited for each of those presentation types, first you must answer a few questions:

Questions to determine what visualization you should use

 

5. Do you want to better understand the relationship between value sets?
  • Relationship charts are suited for showing how one variable relates to one or a number of different variables. You could use this to show how something positively effects, has no effect, or negatively effects another variable.
  • When trying to establish the relationship between things, use these charts:
    • Scatter Plot
    • Bubble
    • Line

Questions to determine what visualization you should use

 

1. Do you want to compare values?
  • Charts are perfect for comparing one or many value sets, and they can easily show the low and high values in the data sets.
  • To create a comparison chart, use these types of graphs:
    • Column
    • Bar
    • Circular area (Radar)
    • Scatter plot
    • Bullet

Questions to determine what visualization you should use

 

2. Do you want to show the composition of something?
  • Use this type of chart to show how individual parts make up the whole of something, such as the device type used for mobile visitors to your website or total sales broken down by sales rep.
  • To show composition, use these types of graphs:
    • Pie
    • Stacked Bar
    • Stacked Column
    • Area
    • Waterfall

Questions to determine what visualization you should use

 

3. Do you want to understand the distribution of your data?
  • Distribution charts help you to understand outliers, the normal tendency, and the range of information in your values.
  • Use these charts to show distribution:
    • Scatter plot
    • Line
    • Column
    • Bar

Questions to determine what visualization you should use

 

4. Are you interested in analyzing trends in your data set?
  • If you want to know more information about how a data set performed during a specific time period, there are specific chart types that do extremely well.
  • Choose a:
    • Line
    • Dual-Axis Line
    • Column

Guidelines for designers

Here are some design guidelines for information designers, as described in “The Visual Display of Quantitative Information” (Tufte, 2001):

  • Show the data without unnecessary decoration
  • Design so that the viewer sees the data rather than the visual design
  • Avoid distorting what the data has to say
  • Avoid wasting unnecessary space
  • Make large data sets coherent

In order to choose the right visualization, Qlik Sense offers augmented self-service with chart recommendations (Qlik Chart Suggestion wizard). This makes creating visualizations faster and more intuitive than ever before:

  • As you drag fields onto the screen, Qlik automatically understands what type of field it is.
  • If you drag on a measure, the values will be summed and displayed as a KPI.
  • If you then add a dimension, the visualization will automatically change to a recommended optimal chart based on the fields added and the data within the fields.

Additionally, the Qlik Insight Advisor offers advanced visual representation of natural language search results and provides generated insight in the data:

  • Access it from the Insight Advisor button, at the top area of the app.
  • You can generate insights on the data, or perform a search using natural language or master items and fields.
  • The search results display existing or new visualizations that you can include to the app. These visualizations might include filters and advanced expressions based on the complexity of the search.
  • You can customize the business logic, under the Prepare menu, to define field groups and relationship between them. Additionally you can define business terms that do not exists in the data but users can refer to when performing queries. These features can improve the Insight Advisor search.
  • You can use the Insight Advisor analysis types F to create visual analytics easily.

Qlik Sense Visualizations

In a Qlik Sense app, visualizations are created in the app sheets.

  • A sheet is where charts and tables for data visualization are placed.
  • Sheets can also include other objects such as filter panes, text or images, input boxes, etc.
  • An app can include one or several sheets.
  • Sheets provide a way of structuring your ideas and purposes for your app. See more info about Structuring an app using sheets F.

The selections that you make affect visualizations regardless of which sheets they are located on.

Qlik Sense visualizations can be classified in charts, tables and additional objects. We can also find custom objects in the Assets panel, that are divided in different bundles and included in the default installation, but we will discuss them other module.

Qlik Sense charts

Here are the basic charts in Qlik Sense. Click on the images to see more details:

Bar charts are one of the most effective visualizations you can choose for showing comparison.

  • The length of bars is perceived pre-attentively.
  • It takes very little effort to compare different sets of data.
  • It can be dimensionless. You can create a chart with measures only, without dimensions.
  • If your data is continuously changing over time, consider a line graph instead.
  • Avoid comparisons with different units ($ and Quantity) on the same scale.
  • Use a continuous scale to provide a complete and accurate view of time-based data (Time aware chart F).
  • Trend lines can be applied to this chart. See more info about the type of lines in the help site (Trend lines F).

See more info about the Bar chart F in the help site.

Box plots are good for comparing range and distributions for groups of numerical data.

A box plot is represented with a box with whiskers and a center line. You can define these elements in the box plot using one of the following presets:

  • Standard (Tukey) — center line represents the median (2nd quartile), the box start and end represent the 1st and 3rd quartiles. Whisker length can be set to 1, 1.5 or 2 inter-quartile ranges.
  • Percentile-based — center line represents the median (2nd quartile), the box start and end represent the 1st and 3rd quartiles. Whisker length is adjusted based on a percentile.
  • Standard deviation — center line represents the average value, the box start and end represent one standard deviation variance. Whisker length is set to a multiple of standard deviations.

See more info about the Box plot F in the help site. Also in this video you can learn more about this visualization.

Bullet charts are good at comparing and measuring performance with more enriched information than a common gauge.

This chart can be used to visualize and compare performance of a measure to a target value and to a qualitative scale among various areas or sets.

  • The dimension axis shows the areas or sets. The dimension is optional in this chart. If there is no dimension can behave as a simple gauge.
  • The measure axis shows the performance of a measure to a target for each category item. The measure is represented by the length of the bar.
  • Each measure can also have a target value, represented with a vertical line, and use segments to display a qualitative range displayed behind the bar.
  • Bullet charts show the performance of a measure in different areas or groups, and the comparison of the target values within each group.

See more info about the Bullet chart F in the help site.

Combo charts combine lines, markers and bars in one single chart.

  • They represent different measures against the same dimension.
  • They are suitable for comparing two measures that are on different scales, as you can represent them either as bar, marker or line.
  • Use a continuous scale to provide a complete and accurate view of time-based data (Time aware chart F).
  • Reference lines can be applied for both measures and dimensions.

See more info about the Combo chart F in the help site.

A distribution plot is suitable for comparing range and distribution for groups of numerical data.

  • Data is plotted as value points along an axis.
  • Value points plotted along the axis indicate the distribution of the values.
  • A bounding box can be displayed to see the range of values.

See more info about the Distribution plot F in the help site.

Gauges are good at providing a summary of the status of the data.

  • They display a single measure value.
  • They are typically used in dashboards.
  • Gauges use colors and range values to provide additional meaning.

See more info about the Gauge F in the help site.

Histograms can be used to represent the distribution of numerical data.

  • Use buckets to bin the range of values, dividing the values into a series of intervals.
  • The intervals are usually consecutive, non-overlapping and are often of equal size.
  • Each interval is represented with a rectangle when there are values in the interval.
  • The height of the rectangle represents the frequency of the values in that series.

See more info about the Histogram F in the help site.

KPI (Key Performance Indicator) objects are used in dashboards and they provide a summary of the data.

  • Display up to two measure values.
  • Use conditional colors and symbols to convey information quickly.

See more info about the KPI F in the help site.

Line charts are good for analyzing data over some sort of time dimension.

  • They are an effective way to display trends.
  • They make use of the pre-attentive attribute of position and therefore make it easy to identify negative or positive trends.
  • Dimension values should be in sequence and at equal intervals between the data points.
  • Use a continuous scale to provide a complete and accurate view of time-based data (Time aware chart F).
  • Trend lines can be applied to this chart. See more info about the type of lines in the help site (Trend lines F).
  • Line styling includes line types, such as dashed lines, line curve, line thickness, and point size.

  • Number of visible points and lines displayed can be controlled.

See more info about the Line chart F in the help site.

Maps can be used to plot data per region or per location.

You can create a map by using one or more layers. The following layers can be combined:

  • Point layer
  • Area layer
  • Line layer
  • Density layer
  • Chart layer

  • Background layer - custom background images are allowed

Geo data or location data can be defined in the Qlik app, using geometries (from geographic sources like KML), names of locations, such as names of countries, regions, cities, postal codes, or line geometries (from sources like Geo JSON).

Data manager can identify geo-fields based on the location names, airport codes, administrative area names, ZIP codes, etc. The coordinates and area geometries will be created automatically once the values have been identified by Data manager.

See more info about the Map F in the help site.

Mekko charts are good at showing normalized percentage values of grouped data.

This chart can be used to compare groups, while being able to compare category items contained within these groups.

  • The dimension axis shows the groups.
  • The measure axis shows the normalized percentage value for each category item.
  • The size of each group shows its value.

  • Mekko charts show a comparison between groups, and a comparison of the impact of categories within each group.

See more info about the Mekko chart F in the help site.

Pie charts are popular visualizations for displaying the visual distribution of measurements.

  • They are beneficial when showing up to seven items or slices.
  • They combine smaller chart slices into one named “Others” with the Limitation property under Dimensions.
  • Optionally, a second measure can be used to determine the radius of each pie slice. In combination with a Donut representation you can show negative values.
  • There are different parameters available in the pie properties to customize the styling: outline width, outline color, corner radius, and inner radius.

See more info about the Pie chart F in the help site.

The scatter plot is the perfect visualization if you want to display correlation. With a scatter plot:

  • The relationship between two measures is displayed, one on each axis.
  • Each point is a dimension value.
  • A third measure can be reflected in the point size.
  • Regression lines can be applied to this chart.

  • Number of visible bubbles displayed in scatter plots can be controlled.

See more info about the Scatter plot F in the help site.

Treemaps are good at providing an overview of a large amount of hierarchical data in a reduced amount of space.

They display hierarchical data by using nested rectangles.

  • Each rectangle’s area is proportional to the measure it represents.
  • Sorting is automatic according to the size of the rectangle.

See more info about the Treemap F in the help site.

Waterfall charts are good at showing the positive and negative values in a data set.

They illustrate how an initial value is affected by intermediate positive and negative values.

  • The first and final values are represented by whole bars.
  • The intermediate values are represented by floating bars.

  • You can also show subtotals in the chart.

See more info about the Waterfall chart F in the help site.

Qlik Sense tables

If you need to see numeric or textual data without a graphical representation, use a table.

  • Tables are located in the charts section in Qlik Sense.
  • Cell values can be copied individually from each table.
  • There are two types of tables in Qlik Sense: table and pivot table.
  • Table styling can be defined in the properties panel and include text alignment in each individual column, font styles and size, and color.
  • Measure columns in tables can be displayed as sparklines (mini charts).
  • Table columns can also be represented with indicators using dynamic or static limits. Color can also be applied to the specific segments determined by the limits.

Click on the images below to see more details:

  • Presents data in rows and columns.
  • Supports sorting by the business user.
  • Can include trend indicators or mini charts.

See more info about the Table F in the help site.

  • Groups the data based on the hierarchical dimensions.
  • Expands and collapses the dimensions - useful to see subtotals.
  • Supports rearranging of the measures and dimensions (pivoting) to get different views of the data.

See more info about the Pivot table F in the help site.

Additional Qlik Sense objects

There are some basic objects you can include in a sheet. Click on the images below to see more details:

You can add a filter pane to control what data is shown in the visualizations on a sheet. A filter pane:

  • Filters the data of several dimensions at once.
  • Displays multiple dimensions.
  • Each listbox (field) inside the filter pane can be customize in terms of presentation (view modes like compact, checkbox and histogram options, or showing data in single column or grid) and order.

See more info about the Filter pane F in the help site.

The text & image object complements other visualizations by offering options to add text, images, hyperlinks, and calculated measures with expressions.

  • Format and color the text and align the paragraphs.

  • The background image has sizing and positioning options.

  • You can set the responsive behavior for text and images.

See more info about the Text & image F in the help site.

Container objects include several visualizations and allow users to switch between them.

  • Include visualizations from master visualizations or visualizations you drag and drop from a sheet.
  • Switch between visualizations using the container tabs, on the same sheet in the dashboard. Set a default tab.

  • Show or hide the visualizations inside the container based on conditions. This allows you to customize the dashboard per user or based on specific scenarios.

See more info about the Container F in the help site.

Button objects add quick links for easy selection and navigation in your app.

  • Buttons are useful in assisting end users with predefined selections, selection controls, or navigation.

  • A button can include one or more selection actions to be performed when the button is clicked. These actions are performed in the order they are listed under the Actions menu.

  • A button can navigate to another sheet, a story, or a website when the button is clicked.

  • Buttons can be styled using different fonts, icons or background images, for instance. They also can be disabled using a calculation condition.

See more info about the Button F in the help site. See a demo in this video.

Visualizations and object properties

Visualizations and other sheet objects can be customized or edited using the properties panel.

By selecting each object in the sheet, you will see an indicator on the bottom right , displaying the location and size of it (Active Grid and Chart Dimensions). In this video you can learn more about this feature.

In edit mode, the properties panel is available on the right side of the screen. If it is hidden, click h in the lower-right corner.

You can see the different properties you can change or customize in this panel.

If the chart was created with Chart suggestions, the properties panel only shows a subset of the full properties list. Toggling Chart suggestions to off will display all the configuration options for that chart.

From the Properties panel F you can:

  • Add dimensions and measures, and specify how they are displayed. You can also duplicate them with right-click.

  • Style font used for titles, subtitles, and footnotes in most of the objects.

  • Change the way the visualization appears on the sheets.
  • Set the sorting order and sorting criteria.
  • Style and change the appearance of a visualization.
  • Insert background color or images in charts and sheets.
  • Add calculation modifiers for line charts, bar charts, combo charts, and tables. These calculations include accumulation, moving average, differences, etc.
  • Add trend lines in bar charts and line charts to represent the average, linear, polynomial, exponential, logarithmic, or power trends.
  • Add reference lines to the measure and dimension axes.
  • Add custom tooltips in charts.

Simplified Authoring (Cloud only)

This user interface provides fast access of the most common creation elements, including objects properties, so all components will not be visible by default.

Some of the things we can find on this experience are:

  • Fields and selections

    At a glance, it is possible to see a frequency count, a symbol indicating data type, and you can make selections while editing.

  • Master items unfolded

    You can directly unfolded the list of dimensions and measures.

  • Data properties

    The Data section of the traditional Properties panel is displayed directly in this mode, so you can easily add or remove fields, dimensions or measures.

  • Visualization properties

    An autochart is created by default. If preferred, you can use the visualization properties to override and pick another chart.

  • Presentation properties

    Depending on the chart used, it will show the most common settings for sorting, color, labels, styling, tooltips, and axis. It includes font size and family for labels.

  • Data table

    It is a table with all fields at the bottom, for data discovery. The table can be sorted, and you can make selections on field values.

  • Selections

    There are five types of filtering, which can be used to include or exclude field values:

    • Pick dimension values

    • String search for values

    • Numeric comparison of values

    • Numeric comparison based on an expression on another field

    • Clear selection, where the chart ignores the current selection

To learn more about Simplified Authoring experience, you can visit this QCC course.

Use the Advanced options toggle located on the right top, to gain access to the full properties panel.

This course utilizes the Advanced options, so you will be able to practice will the all the components.
